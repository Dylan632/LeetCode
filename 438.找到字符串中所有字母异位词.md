## 思路（必填）
### 问题分析
  - 在字符串 s 中找到所有 p 的异位词（字母相同但顺序不同的字符串）的起始索引
  - 异位词的本质：包含相同的字母且每个字母出现次数相同
  - 约束条件：只包含小写字母，可利用数组索引快速统计
### 算法选择
  选择滑动窗口算法，原因：
  1. 异位词要求连续的子串，窗口天然适合
  2. 窗口大小固定为 p.length()，符合滑动窗口特征
  3. 通过复用 cnt 数组，可以在 O(1) 时间内判断窗口是否为异位词
### 实现步骤
- **初始化：** 遍历目标串 `p`，记录需要的字符数量。(`cnt[char] > 0` 表示还**缺**多少
- **右指针进货（扩张）：** 右指针 `right` 扫到的字符进入窗口，消耗需求。(`cnt[char]--`)
- **左指针退货（收缩）：** 如果某个字符拿多了（导致 `cnt[char] < 0`），说明窗口内出现了不该有的字符（或者目标字符数量超标）。此时必须移动左指针 `left`，把吃进去的字符吐出来 (`cnt[char]++`)，直到刚好不超标为止。
- **结算：** 如果经过退货调整后，当前没有任何字符超标，且窗口长度刚好等于 `p` 的长度，说明找到了一个异位词！

## 关键点

>[!Tip] cnt 数组的巧妙用法
> - **初始化**：记录 p 中各字母需要多少个
> - **窗口内字符**：`cnt[c]--`，消耗需求
> - **窗口外字符**：`cnt[c]++`，退还需求
> - 核心思想：用正负值区分“还需多少”和“多拿了多少”

>[!Success] 窗口有效条件
> 当 `cnt` 中所有值都 ≥ 0 时，窗口恰好是异位词

>[!Info] 单次遍历
> 每个字符最多进出窗口各一次，保证 O(n) 时间复杂度
## 代码

```java
import java.util.ArrayList;  
  
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    public List<Integer> findAnagrams(String s, String p) {  
        // 统计 p 的每种字母的出现次数  
        int[] cnt = new int[26];  
        for (char c : p.toCharArray()) {  
            cnt[c - 'a']++;  
        }  
  
        List<Integer> ans = new ArrayList<>();  
        int left = 0;  
        for (int right = 0; right < s.length(); right++) {  
             int c = s.charAt(right) - 'a';  
            // 右端字母进入窗口  
            cnt[c]--;  
            
            while(cnt[c] < 0) {  
                cnt[s.charAt(left) - 'a']++;  
                left++;  
            }  
            if (right - left + 1 == p.length()) {  
                // 窗口左端点下标加入答案  
                ans.add(left);  
            }  
        }  
        return ans;  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点

>[!Note]
> - **滑动窗口**：维护固定大小窗口，通过左右指针调整范围，适用于子串/子数组问题
> - **数组哈希**：利用数组下标作为哈希键，适用于字符范围有限的情况（如小写字母）
> - **双指针**：left 和 right 协同工作，避免暴力枚举所有子串

## 复杂度

- 时间：O(n)，每个字符最多进出窗口各一次
- 空间：O(1)，cnt 数组固定 26 大小
