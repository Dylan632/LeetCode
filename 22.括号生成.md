## 思路(必填)
### 问题分析
给定 n 对括号,生成所有能组成有效括号的组合。有效括号的条件是：任意前缀中左括号数量 ≥ 右括号数量,且最终左右括号数量相等。
### 算法选择
使用回溯法(深度优先搜索),在构建过程中通过剪枝避免无效组合：
  - 有效条件1：左括号数量 < n 时才能放 `(`
  - 有效条件2：右括号数量 < 左括号数量时才能放 `)`
### 实现步骤
  1. 初始化 `char[2n]` 数组存储当前序列
  2. 递归尝试放置 `(` 和 `)`,用数组下标 left + right 定位当前位置
  3. 当 right == n 时,序列完成,加入结果集
  4. 通过两个 if 条件实现剪枝,只生成有效组合

## 关键点
>[!Tip]
> - 原地回溯：`path[left + right] = '('` 直接覆盖,递归返回后自动”撤销”,无需显式还原
> - 剪枝策略：right < left 确保不会出现 )( 这种无效前缀
> - 终止条件：right == n 而非 left + right == 2n,因为右括号更“慢”,用完时序列一定完成

## 代码

```java
class Solution {
    public List<String> generateParenthesis(int n) {
        List<String> ans = new ArrayList<>();
        char[] path = new char[n * 2]; // 所有括号长度都是一样的 2n
        dfs(0, 0, n, path, ans); // 一开始没有填括号
        return ans;
    }
    // 目前填了 left 个左括号，right 个右括号
    private void dfs(int left, int right, int n, char[] path, List<String> ans) {
        if (right == n) { // 填完 2n 个括号
            ans.add(new String(path));
            return;
        }
        if (left < n) { // 可以填左括号
            path[left + right] = '('; // 直接覆盖
            dfs(left + 1, right, n, path, ans);
        }
        if (right < left) { // 可以填右括号
            path[left + right] = ')'; // 直接覆盖
            dfs(left, right + 1, n, path, ans);
        }
    }
}
```

## 知识点

### 问题本质

我们需要在位置 `0,1,2,…,2n−1` 中选择 `n` 个位置填入左括号,其余 `n` 个位置填入右括号。但需要满足额外约束：**对于括号字符串的任意前缀,右括号的个数不能超过左括号的个数**。例如 `())(` 是不合法的。

### 核心思想

- **枚举策略**：枚举当前位置填左括号还是右括号
- **本质**：「选或不选」的思想——填左括号视为「选」,填右括号视为「不选」

### 递归约束条件

1. **核心原则**：递归过程中,右括号个数不能超过左括号个数
2. **情况一**：右括号个数 == 左括号个数 → 只能填左括号
3. **情况二**：右括号个数 < 左括号个数 → 可以填右括号

### 终止条件

由于左括号个数始终 `≥` 右括号个数,且至多填 `n` 个左括号,所以当我们填了 `n` 个右括号时,也一定填了 `n` 个左括号,此时完成所有 `2n` 个括号的填充。

### 常见问题答疑

> [!Question] 什么时候需要写恢复现场,什么时候不需要写?
> 
> 如果初始化 `path` 为空列表,就需要写恢复现场。
> 
> 本题由于所有括号长度固定为 `2n`,可以预先创建一个长为 `2n` 的 `path` 列表,在递归时**直接写入字符**(而不是插入字符),这样无需写恢复现场。

> [!Question] 代码如何保证 `path[0]` 一定是左括号,`path[2n−1]` 一定是右括号?
> 
> - `path[0]`：初始 `left=right=0`,填右括号的条件不成立,所以只能填左括号
> - `path[2n−1]`：由于只有 `right<left` 时才能填右括号,当 `right=left` 时 `right` 不能再增加,所以始终有 `right≤left`。当填了 `2n−1` 个括号时,唯一解是 `left=n` 且 `right=n−1`,此时填左括号的条件不成立,所以只能填右括号

## 举例 n=2
```
dfs(left=0, right=0)  当前 path: [ , , , ]
└── 满足 left<2，填 '('
    dfs(1, 0)  当前 path: [ (, , , ]
    │
    ├── 满足 left<2，填 '('
    │   dfs(2, 0)  当前 path: [ (, (, , ]
    │   │
    │   └── left已满，满足 right<left，填 ')'
    │       dfs(2, 1)  当前 path: [ (, (, ), ]
    │       │
    │       └── 满足 right<left，填 ')'
    │           dfs(2, 2)  当前 path: [ (, (, ), ) ]
    │           → 触底：right == 2
    │           → 生成 "(())" ✨ 加入 ans 列表！
    │           → return (向上一层回溯)
    │
    └── 回溯退回后，满足 right<left，填 ')'
        dfs(1, 1)  当前 path: [ (, ), , ]  (注意：原位置的 '(' 被覆盖)
        │
        └── 满足 left<2，填 '('
            dfs(2, 1)  当前 path: [ (, ), (, ]
            │
            └── left已满，满足 right<left，填 ')'
                dfs(2, 2)  当前 path: [ (, ), (, ) ]
                → 触底：right == 2
                → 生成 "()()" ✨ 加入 ans 列表！
                → return (向上一层回溯)
```

**值的自动回退**：`left` 和 `right` 作为方法的参数,在递归 `return` 回来时,天然就恢复到了这层最初的值,从而把 `left + right`(也就是修改数组的指针)也自动回退到了正确的位置。
举例：**退回到 `dfs(1, 0)`**, 当程序退回时,局部变量 `left` 依然是 `1`,`right` 依然是 `0`(<font color="#de7802">因为 Java 是值传递,下一层怎么改都不会影响这一层的变量</font>)。 

### 📝 完整的退回路线图

我们把整条**出栈**的路连起来看：
- `dfs(2, 2)` 生成了 `(())`,结束,**退回给 →**
- `dfs(2, 1)` 后面没代码了,结束,**退回给 →**
- `dfs(2, 0)` 后面没代码了,结束,**退回给 →**
- **`dfs(1, 0)` 醒来!** 发现后面还有个 `if` 满足条件,**转身进入 →**
- `dfs(1, 1)` 开启新的探索...

递归中 `return` 的精髓——它不仅是 **“结束”**,更是 **“退回并寻找下一个可能”**!

## 回溯
```java
/* 回溯算法框架 */ 
void backtrack(State state, List<Choice> choices, List<State> res) { 
	// 判断是否为解 
	if (isSolution(state)) { 
		// 记录解 
		recordSolution(state, res); 
		// 不再继续搜索 
		return; 
	} 
	// 遍历所有选择 
	for (Choice choice : choices) { 
		// 剪枝：判断选择是否合法 
		if (isValid(state, choice)) { 
			// 尝试：做出选择，更新状态 
			makeChoice(state, choice); 
			backtrack(state, choices, res); 
			// 回退：撤销选择，恢复到之前的状态 
			undoChoice(state, choice); 
		} 
	} 
}
```
