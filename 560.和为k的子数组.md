## 思路

### 问题分析

- 题目要求：找到和为 k 的连续子数组的个数
- 难点：子数组可以是任意长度，暴力枚举需要 $O(n^2)$ 时间

### 算法选择

当我们遍历到当前前缀和 `prefixSum` 时，只要看看之前有多少个前缀和等于 `prefixSum - k`，就说明有多少个子数组和为 `k`。

- 前缀和 + 哈希表：将二维问题降维为一维
- 数学原理：$\text{sum}(i, j) = \text{prefixSum}[j] - \text{prefixSum}[i-1] = k$
- 等价转化：查找 $\text{prefixSum}[i-1] = \text{prefixSum}[j] - k$ 的出现次数

### 实现步骤

1. 初始化哈希表 `{0: 1}`（前缀和 0 出现 1 次，代表空前缀）
2. 遍历数组，累加计算当前位置的前缀和 `prefixSum`
3. 查找哈希表中 `prefixSum - k` 的出现次数，累加到答案
4. 将当前 `prefixSum` 加入哈希表，更新其出现次数

## 关键点

> [!Tip] 子数组的和，就等于**两个前缀和的差**。

- ✅ 先查找再更新：避免子数组长度为 0 的错误（当 k=0 时）
- ✅ 初始哈希表包含 `{0: 1}`：正确处理从数组开头到当前位置的子数组

## 代码

```java
class Solution {
    public int subarraySum(int[] nums, int k) {
        // key: 前缀和, value: 该前缀和出现的次数
        Map<Integer, Integer> cnt = new HashMap<>(nums.length + 1, 1);
        cnt.put(0, 1); // 默认前缀和为 0 的子数组出现 1 次
        int prefixSum = 0;
        int ans = 0;
        for (int x : nums) {
            prefixSum += x;
            ans += cnt.getOrDefault(prefixSum - k, 0);
            cnt.merge(prefixSum, 1, Integer::sum);
        }
        return ans;
    }
}
```

## 知识点

### Java 8 语法糖 `merge`
merge（A，B，C）
1. 第一项：是哈希表的 Key（键），不一定是数字
2. 第二项：是准备存入的 新 Value（值），也不一定是数字
3. 第三项：是一个`BiFunction`（双入参函数），可以把它理解为 **“新老数据合并规则”**。只要是能接收两个参数并返回一个结果的逻辑，都可以放进去。
```java
cnt.merge(prefixSum, 1, Integer::sum);
```

一句话搞定“<font color="#ff0000">没有就存 1，有就加 1</font>”。
如果prefixSum已经存在了，就把旧的值（prefixSum）<font color="#ff0000">出现的次数</font>和新的值（1）加起来，如果prefixSum不存在，就把 （prefixSum，1）存入哈希表。

**`merge` 方法原理：**

```java
// 等价于
if (cnt.containsKey(prefixSum)) {
    cnt.put(prefixSum, cnt.get(prefixSum) + 1);
} else {
    cnt.put(prefixSum, 1);
}
```

**`Integer::sum` 是什么？**

```java
public static int sum(int a, int b) {
    return a + b;  // 就是简单的加法
}
```

`::` 是方法引用语法，`Integer::sum` 相当于把加法操作作为参数传入。当键已存在时，`merge` 会用这个函数合并旧值和新值。

### getOrDefault
```java
ans += cnt.getOrDefault(prefixSum - k, 0);
```
这个 `0` 是**程序员自己设计的**，Java 的哈希表**默认并不是 0**，它的默认是没有（`null`）。
**详细解释：**

- 在 Java 中，如果你用普通的 `cnt.get(key)` 去哈希表里找东西，如果找不到，Java 不会自作主张给你返回 0，而是会**返回 `null`**（空）。
- 你的 `ans` 是一个 `int` 类型的基本数字。如果你执行 `ans += null`，Java 遇到这种“数字加空气”的情况，会当场崩溃，报出 `NullPointerException`（空指针异常）。
- **为什么我们要写 `0`？** `getOrDefault(key, defaultValue)` 的设计初衷就是为了防止空指针。我们在括号里写 `0`，是在告诉 Java：“如果这个前缀和在历史记录里没出现过，请不要给我 `null`，**请默认给我 `0`**。”
- **业务逻辑上的意义**：在我们的题目里，没找到对应的前缀和，就意味着 **“有 0 条路径符合条件”**。所以加上 0，结果 `ans` 不变，不仅逻辑上完美契合，代码也安全运行了。
