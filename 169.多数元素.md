## 题目
![[附件/169.png]]

## 思路

### 算法选择
**摩尔投票法**：核心理念为 **票数正负抵消**。

### 实现步骤
1. **初始化**：票数统计 `votes = 0`，众数候选 `target`
2. **循环**：遍历数组 `nums` 中的每个数字 `num`
   - 当票数 `votes` 等于 0，则假设当前数字 `num` 是众数
   - 当 `num == target` 时，票数 `votes` 自增 1；当 `num != target` 时，票数 `votes` 自减 1
3. **返回值**：返回 `target` 即可

## 关键点

> [!Tip]
> 题目保证多数元素存在，所以不需要二次验证。如果题目不保证，需要再遍历一次验证结果。

## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)
class Solution {
    public int majorityElement(int[] nums) {
        int target = 0;
        int votes = 0;
        for (int num : nums) {
            if (votes == 0) {
                target = num;
            }
            votes += num == target ? 1 : -1;
        }
        return target;
    }
}
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点

**摩尔投票法**的核心是 **票数正负抵消**，时间和空间复杂度分别为 O(N) 和 O(1)，为本题的最佳解法。

### 推论

设输入数组 `nums` 的众数为 `target`，数组长度为 `n`。

**推论一：** 若记众数的票数为 +1，非众数的票数为 -1，则所有数字的票数和 > 0。

**推论二：** 若数组的前 `a` 个数字的票数和 = 0，则数组剩余 `(n-a)` 个数字的票数和一定仍 > 0，即后 `(n-a)` 个数字的众数仍为 `target`。

![[附件/169_1.png]]

根据以上推论，记数组首个元素为 `nums[0]`，众数为 `target`，遍历并统计票数。当发生票数和 = 0 时，剩余数组的众数一定不变：

- 当 `nums[0] == target`：抵消的所有数字中，有一半是众数 `target`
- 当 `nums[0] != target`：抵消的所有数字中，众数 `target` 的数量最少为 0 个，最多为一半

利用此特性，每轮假设发生票数和 = 0 都可以缩小剩余数组区间。当遍历完成时，最后一轮假设的数字即为众数。

## 复杂度

- Time: O(n)
- Space: O(1)
