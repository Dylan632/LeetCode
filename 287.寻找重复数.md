## 题目
![[287.png]]

## 思路（必填）
### 问题分析
题目给定一个包含 `n + 1` 个整数的数组 `nums`，其数字都在 `[1, n]` 范围内。根据鸽巢原理（抽屉原理），必然至少存在一个重复的数字。难点在于进阶要求：**不能修改原数组**，且只能使用 **O(1)** 的额外空间。
### 算法选择
选择了 **快慢指针（Floyd 判圈算法）**。因为数字范围是 `1` 到 `n`，且数组大小是 `n + 1`，我们可以把数组下标当作当前节点，把 `nums[i]` 的值当作 `next` 指针指向的下一个节点。由于存在重复数字，意味着有多个下标指向了同一个值，这就天然构成了一个**多入度的节点**，即**链表中的环的入口**。
### 实现步骤
1. **构建虚拟链表**：将数组转化为 `i -> nums[i]` 的关系。由于所有的值都在 `1` 到 `n`，因此从下标 `0` 出发，永远不会越界，且下标 `0` 本身不会被任何节点指向（因为没有值为 `0`），非常适合作为起点。
2. **快慢指针找相遇点**：`slow` 每次走一步（`nums[slow]`），`fast` 每次走两步（`nums[nums[fast]]`）。由于存在环，两人必将在环内某个节点相遇。
3. **寻找入环点**：相遇后，将一个新指针 `head` 放在起点 `0`。`head` 和 `slow` 同时每次走一步，它们最终相交的节点，就是环的入口，也就是那个导致多入度的**重复数字**。

## 关键点
>[!Tip] 
>根据弗洛伊德算法的推导，当快慢指针相遇时，存在一个固定的距离关系： **从链表起点（0）到环入口的距离，等于从“相遇点”继续走到环入口的距离。**

## 代码

```java
class Solution {  
    public int findDuplicate(int[] nums) {  
        // 0 一定不在环上，适合作为起点  
        int slow = 0;  
        int fast = 0;  
        while (true) {  
            // 等价于 slow = slow.next            
            slow = nums[slow];  
            // 等价于 fast = fast.next.next            
            fast = nums[nums[fast]];  
            // 快慢指针移动到同一个节点  
            if (fast == slow) {  
                break;  
            }  
        }  
        // 再用一个指针，从起点出发  
        int head = 0;  
        while (slow != head) {  
            slow = nums[slow];  
            head = nums[head];  
        }  
        // 入环口即重复元素  
        return slow;  
    }  
}
```

## 知识点

> [!Question] 如何证明 nums 中至少存在一个重复的数字？
> 
> 视作有 n+1 个球和 n 个抽屉，第 i 个球放入第 `nums[i]` 个抽屉。根据抽屉原理（鸽巢原理），存在一个抽屉至少有两个球，这个抽屉的编号即重复元素。

## 用基环树理解
想象有一个 n+1 个节点的图，节点编号从 0 到 n。
对 i=0,1,2,…,n，连一条从 i 到 `nums[i]` 的有向边，可以得到一个**有向图**。
![[287_1.png]]
每个节点的入度，就是这个节点在 nums 中的出现次数。重复元素的入度大于 1。上图中的节点 2 的入度为 2，所以 2 是重复元素。

如何找到入度大于 1 的节点？

在每个节点的出度都是 1 的情况下，n+1 个点 n+1 条边的有向图，又叫内向基环森林，每个连通块都恰好有一个环。

由于 `nums[i]≥1`，所以节点 0 的入度是 0，不在环上。从节点 0 出发，进入基环，基环的入环口，就是入度大于 1 的节点。

⚠注意：如果从非 0 节点出发，可能无法找到答案。如下图，如果从节点 3 出发，无法找到入度大于 1 的节点。
![[287_2.png]]

考察包含节点 0 的连通块，其形状和 [[142.环形链表II]] 是一样的，所以都可以用**快慢指针**解决。
两道题目的联系如下，注意 i 和 `nums[i]` 都是节点。

|       | **142. 环形链表 II** | **287. 寻找重复数** |
| ----- | ---------------- | -------------- |
| 链表节点  | node             | i              |
| 下一个节点 | node.next        | `nums[i]`      |
| 头节点   | head             | 0              |
| 入环口   |                  | 重复元素           |
在链表中，我们的移动方式是从节点 node 移动到下一个节点 node.next。
在数组中，我们的移动方式是从节点 i 移动到节点 `nums[i]`。由于数组长度是 n+1，而 `nums[i]<n+1`，所以不会下标越界。

## 复杂度

- Time: O(n)，其中 n 是 nums 的长度。
- Space: O(1)
