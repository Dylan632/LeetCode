## 思路（必填）
### 问题分析：
  - 输入：k 个升序链表数组
  - 输出：合并后的一个升序链表
  - 约束：时间效率要高，空间尽量优化
### 算法选择
  选择**优先队列(最小堆)** 的原因：
  1. 堆可以高效维护 k 个元素的最小值,O(log k) 时间获取和更新
  2. 每次取出最小节点后,只需将其后继节点入堆,保证堆中始终有所有链表的当前候选节点
  3. 时间复杂度 O(N log k) 优于暴力合并的 O(Nk)
### 实现步骤
  1. 初始化最小堆，将所有非空链表的头节点入堆
  2. 创建虚拟头节点 dummy 用于构建结果链表
  3. 循环处理直到堆为空：
    - 取出堆顶（当前所有链表中的最小节点）
    - 若该节点有后继，将后继入堆
    - 将该节点连接到结果链表末尾
  4. 返回 dummy.next（真正的头节点）

## 关键点
> [!Tip]
> - 堆中始终维护 k 个候选节点（各链表的当前最小值）
> - 每个节点只入堆、出堆各一次

## 代码

```java
class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);
        for (ListNode head : lists) {
            if (head != null) {
                // 把所有非空链表的头节点入堆
                pq.offer(head);
            }
        }

        ListNode dummy = new ListNode();
        ListNode cur = dummy;
        // 循环直到堆为空
        while (!pq.isEmpty()) {
            // 剩余节点中值最小的节点出堆
            ListNode node = pq.poll();
            // 下一个节点不为空
            if (node.next != null) {
                // 下一个节点有可能是最小节点，入堆
                pq.offer(node.next);
            }
            // 把 node 添加到新链表的末尾
            cur.next = node;
            // 准备合并下一个节点
            cur = cur.next;
        }
        return dummy.next;
    }
}
```

## 知识点

### PriorityQueue（优先队列）

普通队列是“先来后到”（FIFO），而优先队列按**优先级**出队——优先级最高的元素最先出队。Java 的 `PriorityQueue` 默认是**小顶堆**，即数值越小优先级越高，`poll()` 总是返回最小元素。

```java
PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);
```
如果把这段自定义比较器 `(a, b) -> a.val - b.val` 删掉，程序在运行时就会报错（通常是抛出 `ClassCastException` 异常）。

原因如下：
- **默认排序的条件：** 默认的小顶堆依赖于元素的**自然排序**，这就要求放入队列的元素类必须实现了 `Comparable` 接口。
- **`ListNode` 的局限：** LeetCode 提供的自定义类 `ListNode` 并没有实现 `Comparable` 接口。因此，当试图把两个 `ListNode` 放入优先队列时，Java 并不知道该依据什么来判断谁大谁小（是比较内存地址？还是比较里面的 `val`？）。
- **比较器的作用：** `(a, b) -> a.val - b.val` 就是明确告诉 `PriorityQueue`：**请根据节点里面的 `val` 属性的值从小到大进行比较。**
简单来说，默认是小顶堆没错，但你得告诉它“根据什么指标来排这个小顶堆”，因为 `ListNode` 自身没有提供比较规则。


**底层结构**：二叉堆用数组存储，逻辑上是一棵完全二叉树。通过索引公式可以在数组中快速定位父子节点：

| 节点 | 索引公式 |
|------|----------|
| 左子节点 | $2i + 1$ |
| 右子节点 | $2i + 2$ |
| 父节点 | $(i - 1) / 2$ |

**核心操作**：

- `offer(e)`：$O(\log n)$
  将新元素放到数组末尾，然后**向上冒泡**（Sift Up）——与父节点比较，若更小则交换，直到到达正确位置。

- `poll()`：$O(\log n)$
  将堆顶（最小值）与末尾元素交换后删除，再把末尾元素**向下沉降**（Sift Down）——与较小的子节点比较，若更大则交换，直到恢复堆性质。

- `peek()`：$O(1)$
  直接返回数组首元素。

### 自定义比较器

`PriorityQueue` 默认只支持基本类型。自定义对象需传入 `Comparator`：

```java
// 小顶堆：a.val - b.val（最小元素在堆顶）
PriorityQueue<ListNode> minHeap = new PriorityQueue<>((a, b) -> a.val - b.val);

// 大顶堆：b.val - a.val（最大元素在堆顶）
PriorityQueue<ListNode> maxHeap = new PriorityQueue<>((a, b) -> b.val - a.val);
```

<font color="#de7802">a 和 b 都是 ListNode 对象，a.val 和 b.val 是访问节点的值字段，需要访问 .val 字段来比较</font>  

> [!Tip] 比较器返回值含义
> - 负数：a 优先级更高，排前面
> - 正数：b 优先级更高
> - 零：两者优先级相等
>
> **为什么 `a.val - b.val` 是小顶堆？**
> 当 $a.val < b.val$ 时，$a.val - b.val$ 为负数，表示 a 优先级更高，会排在堆顶。所以值越小的元素优先级越高，最终堆顶就是最小元素。
>
> **举例**：a=3, b=5 → `3-5=-2`（负数）→ a 优先级高，3 排前面。反之 a=8, b=4 → `8-4=4`（正数）→ b 优先级高，4 排前面。无论哪种情况，较小的数都会被推到前面。

## 复杂度

- Time: O(N log k)，其中 N 是所有链表的节点总数，k 是链表数量
- Space: O(k)，堆中最多存储 k 个节点
