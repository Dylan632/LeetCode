## 思路（必填）
### 问题分析
题目要求设计一种数据结构，能够高效地插入字符串、查询字符串是否完全存在，以及查询是否存在以特定前缀开头的字符串。
### 算法选择
选择使用**字典树（Trie）**。这种数据结构利用字符串的公共前缀来减少查询时间，最大限度地减少无谓的字符串比较，查询效率非常高（与目标字符串长度成正比，与库中字符串总数无关）。
### 实现步骤
1. **定义节点**：创建一个 `Node` 类，包含一个大小为 26 的 `Node` 数组（映射 a-z 字母路径）和一个布尔值 `end`（标记是否为单词结尾）。
2. **插入操作**：从根节点开始，逐字符向下遍历。如果当前字符对应的子节点不存在，则创建一个新节点。遍历到单词最后一个字符后，将该节点的 `end` 标记为 `true`。
3. **查询抽象**：封装一个核心的 `find` 方法，模拟指针在树中的游走。一旦遇到空路径（字符对应节点不存在），直接返回 0（未找到）。如果成功走完路径，根据所在节点的 `end` 标识，返回 2（是完整单词）或 1（仅仅是前缀）。
4. **功能路由**：`search` 检查返回值是否为 2；`startsWith` 检查返回值是否不为 0。

## 关键点
>[!Tip] 
>使用数组下标 `c - 'a'` 替代哈希表进行字符到节点的映射，以空间换取极致的寻址时间；巧妙复用 `find` 逻辑处理了两种不同的查询诉求。

## 代码

```java
class Trie {  
    private static class Node {  
        Node[] son = new Node[26];  
        boolean end = false;  
    }  
  
    private final Node root = new Node();  
  
    public void insert(String word) {  
        Node cur = root;  
        for (char c : word.toCharArray()) {  
            c -= 'a';  
            // 无路可走？  
            if (cur.son[c] == null) {  
                // new 出来！  
                cur.son[c] = new Node();  
            }  
            cur = cur.son[c];  
        }  
        cur.end = true;  
    }  
  
    public boolean search(String word) {  
        return find(word) == 2;  
    }  
  
    public boolean startsWith(String prefix) {  
        return find(prefix) != 0;  
    }  
  
    private int find(String word) {  
        Node cur = root;  
        for (char c : word.toCharArray()) {  
            c -= 'a';  
            // 道不同，不相为谋  
            if (cur.son[c] == null) {  
                return 0;  
            }  
            cur = cur.son[c];  
        }  
        // 走过同样的路（2=完全匹配，1=前缀匹配）  
        return cur.end ? 2 : 1;  
    }  
}
/**  
* Your Trie object will be instantiated and called as such: 
* Trie obj = new Trie(); 
* obj.insert(word); 
* boolean param_2 = obj.search(word); 
* boolean param_3 = obj.startsWith(prefix); 
* */
```

## 知识点
### 思路
- “若我俩都走过同样的路，则我俩心意相通。”

假设字符串里面只有 a 和 b 两种字符。

从左到右遍历字符串，把 a 视作往左走，把 b 视作往右走。怎么记录路径？怎么区分不同的路径？用二叉树记录。

- insert：例如先插入字符串 aab，相当于生成了一条移动方向为「左-左-右」的路径。标记最后一个节点为终止节点。再插入字符串 aabb，相当于生成了一条移动方向为「左-左-右-右」的路径。标记最后一个节点为终止节点。
- search：例如查找字符串 aab，相当于查找二叉树中是否存在一条移动方向为「左-左-右」的路径，且最后一个节点是终止节点。
- startsWith：例如查找前缀 aa，相当于查找二叉树中是否存在一条移动方向为「左-左」的路径，无其他要求。
![[208_1.png]]

推广到 26 种字母，其实就是一棵 26 叉树，对于 26 叉树的每个节点，可以用哈希表，或者长为 26 的数组来存储子节点。

### 算法
- 初始化：创建一棵 26 叉树，一开始只有一个根节点 `root`。26 叉树的每个节点包含一个长为 26 的儿子节点列表 `son`，以及一个布尔值 `end`，表示是否为终止节点
- insert：
	1. 遍历字符串 `word`，同时用一个变量 `cur` 表示当前在 26 叉树的哪个节点，初始值为 `root`。
	2. 如果 `word[i]` 不是 `cur` 的儿子，那么创建一个新的节点 `node` 作为 `cur` 的儿子。如果 `word[i]=a`，那么把 `node` 记录到 `cur` 的 `son[0]` 中。如果 `word[i]=b`，那么把 `node` 记录到 `cur` 的 `son[1]` 中。依此类推。
	3. 更新 `cur` 为儿子列表中的相应节点。
	4. 遍历结束，把 `cur` 的 `end` 标记为 `true`。

- search 和 startsWith 可以复用同一个函数 find：
	1. 遍历字符串 `word`，同时用一个变量 `cur` 表示当前在 26 叉树的哪个节点，初始值为 `root`。
	2. 如果 `word[i]` 不是 `cur` 的儿子，返回 0。`search` 和 `startsWith` 收到 0 之后返回 `false`。
	3. 更新 `cur` 为儿子列表中的相应节点。
	4. 遍历结束，如果 `cur` 的 `end` 是 `false`，返回 1，否则返回 2。`search` 如果收到的是 2，返回 `true`，否则返回 `false`。`startsWith` 如果收到的是非 0 数字，返回 `true`，否则返回 `false`。

### 关系拆解

- **`Node` (类定义/图纸)**： 它是房间的**设计图**。设计图规定了每个房间都有 26 扇门（`son` 数组），并且地板上有一个开关（`end` 标志），用来标记这是否是一个终点。
- **`root` (根节点/大堂)**： 它是这座大楼的**入口大堂**。它本身也是一个房间（`Node` 实例），但它是所有单词出发的起点。
- **`son` (子节点数组/26扇门)**： 这是房间里的 **26 扇门**。
    - 注意：`son` 本身是一个**容器（数组）**，它里面装着 **26 个位置**。
    - 重点：这些位置起初是空的（`null`）。当你执行 `new Node()` 并放进 `son[c]` 时，就相当于在这一扇门后**新建了一个一模一样的房间**。
- **`cur` (当前指针/探险者)**： 它不是一个房间，它是**你自己**。
    - `Node cur = root;` 表示你现在站在大堂。
    - `cur = cur.son[c];` 表示你推开了 `c` 这扇门，走进了门后的那个新房间。现在，你的位置更新了。

### 为什么可以用.son
可以用 `.son`，**纯粹是因为在最开始的 `Node` 类（图纸）里，我们自己定义了它。**
在 Java（以及很多面向对象的编程语言）中，`.`（点号）的作用翻译成大白话就是 **“的”**
```java
private static class Node {
    Node[] son = new Node[26]; // <--- 注意这里！
    boolean end = false;       // <--- 还有这里！
}
```
这段代码是一张**设计图**。它规定了：只要你创建了一个 `Node` 对象，这个对象就**天生自带两个属性**：

1. 一个叫 `son` 的数组（用来存 26 扇门）。
2. 一个叫 `end` 的布尔值（用来标记是不是终点）。

### `cur.son` 到底是什么意思？

在代码的后面，我们写了 `Node cur = root;`，这说明 `cur` 是一个具体的 `Node` 对象（就像按照图纸建好的一个具体的房间）。

- 当你写 **`cur.son`** 时，机器的理解是：获取 `cur` **的** `son` 属性。（拿到当前这个房间的 26 扇门）。
- 同理，当你写 **`cur.end`** 时，意思是：获取 `cur` **的** `end` 属性。（看看当前房间地上有没有终点标记）。
### 为什么不能直接写 `son[c]`？

假设在迷宫里有成百上千个房间（成百上千个 `Node`），**每一个房间都有自己独立的一套 `son` 数组**（都有自己的 26 扇门）。
如果你只写一句 `son[c] = new Node();`，计算机会立刻罢工并报错，因为它会问你：**“你要推开的是哪一个房间的门？”**
加上 `cur.` 前缀，就是明确告诉计算机： “我要找的是 **`cur`（当前所在位置）的** 那套 `son` 数组，去操作它里面的第 `c` 扇门。”

> **总结：** `.` 操作符是用来访问对象内部数据的钥匙。因为 `cur` 是一头“节点兽”（Node），而图纸规定了每头“节点兽”肚子里都有一个叫 `son` 的器官，所以你才能用 `cur.son` 把它掏出来用。
