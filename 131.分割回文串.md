## 思路（必填）
### 问题分析
给定字符串 s，要求将其分割成若干子串，使得每个子串都是回文串，返回所有可能的分割方案。
### 算法选择
采用回溯算法（深度优先搜索）
- 每次决策：选择下一个回文子串的结束位置
- 状态：当前已分割的部分 path，以及剩余待分割的起始位置 i
### 实现步骤
1. 从位置 i 开始，枚举所有可能的结束位置 j
2. 若 `s[i..j]` 是回文，则加入 path，递归处理 `s[j+1..]`
3. 当 i 到达字符串末尾，说明找到一个完整分割方案
4. 回溯：撤销当前选择，继续枚举其他可能

## 关键点
>[!Tip] 
>- 回溯模板：做选择 → 递归 → 撤销选择
  >- 注意复制 path 到结果集，避免引用传递导致错误

## 代码

```java
class Solution {  
    public List<List<String>> partition(String s) {  
        List<List<String>> ans = new ArrayList<>();  
        List<String> path = new ArrayList<>();  
        dfs(0, s, path, ans);  
        return ans;  
    }  
  
    // 现在 s 未被分割的部分为 [i, n-1]
    // 枚举下一刀切在哪
    private void dfs(int i, String s, List<String> path, List<List<String>> ans) {
        if (i == s.length()) { // s 分割完毕
            ans.add(new ArrayList<>(path)); // 复制 path
            return;
        }
        // 枚举子串的结束位置
        for (int j = i; j < s.length(); j++) {
            // 判断 [i, j] 是不是回文串
            if (isPalindrome(s, i, j)) {
                // 分割！
                path.add(s.substring(i, j + 1));
                // 现在 s 未被分割的部分为 [j+1, n-1]
                dfs(j + 1, s, path, ans);
                path.removeLast();
            }
        }
    }  
  
    private boolean isPalindrome(String s, int left, int right) {  
        while (left < right) {  
            if (s.charAt(left++) != s.charAt(right--)) {  
                return false;  
            }  
        }  
        return true;  
    }  
}
```

## 知识点
### 一、 核心思想：“切一刀，看一看”

解决此类分割问题的基本套路是 **DFS + 回溯**。
- **动作：** 从当前剩余字符串的头部切下一段。
- **判断：** 如果切下的是回文串，把它存入当前路径 `path`，然后递归去切剩下的部分。
- **剪枝：** 如果切下的不是回文串，直接放弃这种切法，换个位置切。
- **回溯：** 一条路走到黑（或者碰壁）后，必须“拔出刀”（从 `path` 中移除刚加进来的子串），恢复现场，去尝试下一种可能。

### 🔥 二、 重点突破 1：终止条件 `i == s.length()` 的含义

> **疑惑点：** 为什么 `i` 等于字符串长度时，就说明完美分割了？
- **`i` 的定义：** `i` 是我们当前准备下刀的 **“光标”** 位置（即剩下未分割字符串的起始索引）。
- **成功通关：** 我们只有在前面切下的每一刀**都是回文串**的情况下，才会继续调用更深层的 `dfs`。
- **完美越界：** 当光标 `i` 一路畅通无阻，最终等于 `s.length()` 时，意味着光标已经越过了最后一个字符，**没有任何“边角料”剩下**。
- **结论：** 这说明这条路线上的每一次切割都是合法的，并且刚好把整条字符串切得干干净净。此时必须立刻将 `path` 拷贝并收集到结果集 `ans` 中。

### 🔥 三、 重点突破 2：回溯中的 `for` 循环与 `j++`

> **疑惑点：** 在 `dfs(1)` 中，经历了后续的深层递归后，为什么 `j` 会自然地从 1 变成 2？
这是理解**函数调用栈**的核心：**“冻结”与“解冻”**。
1. **冻结：** 当在 `dfs(1)` 且 `j=1` 的循环中，我们调用了更深层的 `dfs(2)` 时，当前的 `dfs(1)` 并没有结束，而是**停在原地被“冻结”** 了。它牢牢记着自己此时的状态是 `i=1, j=1`。
2. **解冻与恢复现场：** 当 `dfs(2)` 及更深层的代码执行完毕并 `return` 后，程序回到了那个被冻结的 `dfs(1)`。它紧接着执行下一行代码 `path.removeLast()`，把之前试探性放入的子串拿出来，这就叫**恢复现场**。
3. **循环的自然推进：** 恢复现场完毕，意味着 `if` 语句结束，这一轮的循环体也就彻底执行完了。按照 `for` 循环的铁律，此时会执行循环头部的 **`j++`**，所以 `j` 自然而然就从 1 变成了 2。
4. **结论：** 并非神奇的跳跃，而是因为去探索“平行宇宙”的分身回来了，本体继续执行本层循环的下一次迭代。

## 复杂度

- Time: $O(n \cdot 2^n)$，其中 $n$ 为 $s$ 的长度。答案的长度至多为逗号子集的个数，即 $O(2^n)$，因此会递归 $O(2^n)$ 次，再算上判断回文和加入答案时需要 $O(n)$ 的时间，所以时间复杂度为 $O(n \cdot 2^n)$。

- Space: $O(n)$，返回值的空间不计入。
