## 思路（必填）
### 问题分析
给定一个无重复元素的整数数组 `candidates` 和一个目标整数 `target`，找出 `candidates` 中可以使数字和为目标数的所有组合。同一个数字可以被无限制重复选取。
### 算法选择
使用回溯算法（深度优先搜索），因为：
- 需要枚举所有可能的组合（组合问题典型解法）
- 每个元素可以重复使用
- 需要收集所有满足条件的解
### 实现步骤
1. 预处理：对数组排序，使剪枝条件可以生效
2. 定义状态：`path` 存储当前路径，`ans` 存储所有有效解
3. 递归搜索：从 `i` 开始尝试每个候选数
4. 做选择：将当前数加入路径，更新剩余目标值 `left`
5. 递归：由于数字可重复使用，下次仍从当前索引 `j` 开始
6. 撤销选择：回溯，恢复现场，尝试其他可能

## 关键点
>[!Tip] 
>  - 剪枝优化：排序后，当 `candidates[j]` > `left` 时直接终止循环
  >- 避免重复组合：通过 `i`/`j` 控制，保证组合按非递减顺序生成

## 代码

```java
class Solution {  
    public List<List<Integer>> combinationSum(int[] candidates, int target) {  
        Arrays.sort(candidates);  
        List<List<Integer>> ans = new ArrayList<>();  
        List<Integer> path = new ArrayList<>();  
        dfs(0, target, candidates, ans, path);  
        return ans;  
    }  
  
    private void dfs(int i, int left, int[] candidates, List<List<Integer>> ans, List<Integer> path) {  
        if (left == 0) {  
            // 找到一个合法组合  
            ans.add(new ArrayList<>(path));  
            return;  
        }  
        for (int j = i; j < candidates.length && candidates[j] <= left; j++) {  
            path.add(candidates[j]);  
            dfs(j, left - candidates[j], candidates, ans, path);  
            // 恢复现场  
            path.removeLast();  
        }  
    }  
}
```

## 知识点
### 回溯保存技巧
```java
if (left == 0) {
    ans.add(new ArrayList<>(path));
    return;
}
```
**重点**：这里必须用 `new ArrayList<>(path)` 进行**深拷贝**。因为在整个递归过程中，大家共用的是同一个 `path` 内存地址。如果我们直接写 `ans.add(path)`，后续 `path` 里面的数字被清空或修改时，`ans` 里保存的结果也会跟着全被清空。所以必须新建一个列表把当前的值“快照”保存下来。
### 回溯中的 `int j = i` (去重技巧)
**1. 解决的痛点：避免排列重复**
- 在“组合”问题中，`[2, 3]` 和 `[3, 2]` 是同一个答案。
- 如果每次递归都从头开始选（`j = 0`），不仅会搜出 `[2, 3]`，还会“回头”搜出 `[3, 2]`，导致结果重复。

**2. 核心原理：传参机制（单向挡板）**
- **当前层选择：** 当前层 `for` 循环遍历到索引 `j`，选中了 `candidates[j]`。
- **向下传参：** 调用下一层递归时，将当前的 `j` 作为下一层的起始索引 `i` 传进去：`dfs(j, ...)`。
- **下一层限制：** 下一层的 `for` 循环接收到 `i = j`，循环就变成了 `for (int j = i; ...)`。这相当于在数组 `j` 的位置插了一块 **“只能向右滑动的挡板”**。

**3. 最终效果：强制非递减**
- 下一层（以及更深层）在挑数字时，**只能看当前挡板位置及右边的数字，彻底屏蔽了挡板左边（前面）的数字**。
- 结果：选出的数字序列被**强制规定为“从小到大（或相等）”的非递减顺序**（例如只产生 `[2, 3]`）。既然所有合法答案都只有这一种统一的升序格式，就从根源上完美消灭了 `[3, 2]` 这种乱序的重复组合。

**💡 一句话总结：** <font color="#de7802">把当前的 j 传给下一层当起点，就是告诉下一层“不许走回头路去选前面的数字”，从而保证组合序列永远是非递减的，实现完美去重。</font>

## 回溯过程

![[39_1.png]]
为了去除重复子集，**一种直接的思路是对结果列表进行去重**。但这个方法效率很低，有两方面原因：
- 当数组元素较多，尤其是当 `target` 较大时，搜索过程会产生大量的重复子集。
- 比较子集（数组）的异同非常耗时，需要先排序数组，再比较数组中每个元素的异同。
<font color="#de7802">我们考虑在搜索过程中通过剪枝进行去重。</font>
![[39_2.png]]

总结来看，给定输入数组 $[x_1, x_2, \ldots, x_n]$，设搜索过程中的选择序列为 $[x_{i_1}, x_{i_2}, \ldots, x_{i_m}]$，则该选择序列需要满足 $i_1 \leq i_2 \leq \cdots \leq i_m$，**不满足该条件的选择序列都会造成重复，应当剪枝。**
为实现该剪枝，我们初始化变量 `start`，用于指示遍历起始点。当做出选择 $x_i$ 后，设定下一轮从索引 $i$ 开始遍历。这样做就可以让选择序列满足 $i_1 \leq i_2 \leq \cdots \leq i_m$，从而保证子集唯一。除此之外，我们还对代码进行了以下两项优化：

- **优化1**：在开启搜索前，先将数组 `nums` 排序。在遍历所有选择时，当子集和超过 `target` 时直接结束循环，因为后边的元素更大，其子集和一定超过 `target`。
- **优化2**：省去元素和变量 `total`，**通过在 `target` 上执行减法来统计元素和**，当 `target` 等于 $0$ 时记录解。
