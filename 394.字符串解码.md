## 思路（必填）
### 问题分析
  - 题目要求解码形如 `k[encoded_string]` 的字符串，其中 encoded_string 可能嵌套
  - 例如：`3[a2[c]]` → `accaccacc`
  - 核心难点：处理嵌套结构，需要从内向外展开
### 算法选择
  - 栈模拟递归：将递归过程显式化，使用栈保存每一层的上下文（前缀字符串和重复次数）
  - 为什么选择栈？因为嵌套结构符合“后进先出”（LIFO）的特性
### 实现步骤
  1. 遍历字符串的每个字符
  2. 遇到字母：直接拼接到当前结果字符串
  3. 遇到数字：累积计算重复次数 k（处理多位数）
  4. 遇到 `[`：
    - 保存当前状态（前缀字符串 + 重复次数）到栈
    - 重置当前结果和重复次数，开始处理内层
  5. 遇到 `]`：
    - 从栈中恢复前缀字符串和重复次数
    - 将当前结果重复 k 次并拼接到前缀后
  6. 返回最终结果

## 关键点
>[!Tip]
>- 使用栈保存”递归前”的状态（前缀字符串和重复次数）
>- 通过 `res.setLength(0)` 清空 StringBuilder，避免频繁创建对象
>- 处理多位数字时，使用 `k = k * 10 + (c - '0')` 累积计算

## 代码

```java
class Solution {  
    private record Pair(String s, int k) {  
    }  
  
    public String decodeString(String s) {  
        Deque<Pair> stack = new ArrayDeque<>(); // 用于模拟计算机的递归  
        StringBuilder res = new StringBuilder();  
        int k = 0;  
        for (char c : s.toCharArray()) {  
            if (Character.isLetter(c)) {  
                res.append(c);  
            } else if (Character.isDigit(c)) {  
                k = k * 10 + (c - '0');  
            } else if (c == '[') {  
                // 模拟递归  
                // 在递归之前，把当前递归函数中的局部变量 res 和 k 保存到栈中  
                stack.push(new Pair(res.toString(), k));  
                // 递归，初始化 res 和 k                
                res.setLength(0);  
                k = 0;  
            } else { // ']'  
                // 递归结束，从栈中恢复递归之前保存的局部变量  
                Pair p = stack.pop();  
                // 此时 res 是下层递归的返回值，将其重复 p.k 次，拼接到递归前的 p.s 之后  
                res = new StringBuilder(p.s).repeat(res, p.k);  
            }  
        }  
        return res.toString();  
    }  
}
```

## 知识点
### 一、核心解题思想：为什么要用栈？

处理类似 `3[a2[c]]` 这种具有**嵌套结构**的数据时，最自然的方式是递归。在计算机底层，递归是通过“调用栈”实现的。

本题采用**手动维护一个栈 (Stack)** 的方式来模拟递归过程：

- 遇到 `[`：相当于进入下一层函数，需要把”进入前”的状态（外层拼接好的字符串和倍数）保存（压栈）。
- 遇到 `]`：相当于当前层函数执行完毕，需要把保存的”历史状态”取出来（出栈），与当前层的结果进行合并。

这种解法避免了系统栈溢出的风险，且逻辑分明。

### 二、关键变量说明

- **`record Pair(String s, int k)`**: Java 14 引入的数据载体语法。用于将**遇到 `[` 之前的历史字符串 `s`** 和 **即将用于重复的倍数 `k`** 绑定打包。
- **`Deque<Pair> stack`**: 充当调用栈，用于保存历史状态。
- **`StringBuilder res`**: 当前层级正在构建的字符串（相当于当前函数的局部变量）。
- **`int k`**: 当前正在解析的数字倍数。

### 三、四大分支逻辑 (遍历字符串字符 `c`)

1. **遇到字母 (`Character.isLetter`)**：普通字符，直接追加到当前的 `res` 尾部。
2. **遇到数字 (`Character.isDigit`)**：数字可能有多位（如 `12`），需要累乘计算：`k = k * 10 + (c - '0');`。
3. **遇到左括号 `[` (核心：保存状态，进入深层)**：
    - 将当前的 `res` 和 `k` 打包压入栈中：`stack.push(new Pair(res.toString(), k));`
    - 清空当前的 `res` 和 `k`，准备解析括号内的新内容。
4. **遇到右括号 `]` (核心：取出状态，合并结果)**：
    - 从栈中弹出历史状态。
    - 将当前括号内解析出的 `res` 重复 `k` 次，拼接到历史字符串 `s` 之后。

### 四、重点代码与方法深度剖析

#### 1. 为什么用 `StringBuilder` 而不是 `String`？
- **原理**：Java 中的普通 `String` 是**不可变**的。如果在循环中用 `+` 号拼接 `String`，底层会不断创建新对象，极度消耗内存且拖慢速度。
- **优势**：`StringBuilder` 是**可变**的字符序列。使用 `append()` 方法可以直接在原对象上追加字符，效率极高。

#### 2. `res.setLength(0);`
- **作用**：在遇到 `[` 准备解析新一层内容时，清空当前的 `StringBuilder`。
- **为什么不用 `res = new StringBuilder();`**：`new` 关键字会在内存中开辟一块全新空间（买一块新白板），消耗资源。而 `setLength(0)` 只是把内部的“书写指针”拉回原点（把白板擦干净），**复用了原有的内存空间**，是最高效的清空方式。

#### 3. `Pair p = stack.pop();`
- **作用**：遇到 `]` 时，说明当前括号的内容处理完了。`pop()` 方法利用栈 **“后进先出 (LIFO)”** 的特性（像从水桶最上面拿盘子），精准地取出进入当前括号前保存的那个状态包 `p`。
- `p.s` 就是进括号前，我们已经拼好的旧字符串。
- `p.k` 就是当前这个括号里的内容需要重复的次数。

#### 4. `res = new StringBuilder(p.s).repeat(res, p.k);`
- **作用**：将历史前缀与当前重复后的结果进行终极拼装（使用了 Java 21 引入的新特性）。
- **执行顺序拆解**：
    1. `new StringBuilder(p.s)`：拿出一块新白板，先写上历史字符串 `p.s`（前缀）。
    2. `.repeat(res, p.k)`：将当前括号内的解析结果 `res` 连续抄写 `p.k` 遍，追加到白板后面。
    3. `res = ...`：将这个完美拼装好的全新字符串，重新赋值给 `res`，使其成为最新的外层结果，继续参与后续遍历。

5. **关于返回类型：** 方法要求返回 `String`，所以必须用 `.toString()` 将可变的 `StringBuilder` 转换为不可变的 `String`。
### 五、动态演示 (以输入 `3[a2[c]]` 为例)
1. **初始状态**: `stack` = `[]`, `res` = `""`, `k` = `0`
2. **读到 `3`**: `k` 变成 `3`。
3. **读到 `[`**:
    - 压栈：`stack.push(("", 3))`
    - 清空：`res` = `""`, `k` = `0`
4. **读到 `a`**: `res` 追加字母，变成 `"a"`。
5. **读到 `2`**: `k` 变成 `2`。
6. **读到 `[`**:
    - 压栈：`stack.push(("a", 2))`
    - 清空：`res` = `""`, `k` = `0`
7. **读到 `c`**: `res` 追加字母，变成 `"c"`。
    - 此时栈的状态：自底向上是 `("", 3)` -> `("a", 2)`_
8. **读到 `]`**:
    - 出栈：拿到 `p = ("a", 2)`。
    - 计算：用 `p.s` ("a") 加上当前 `res` ("c") 重复 `p.k` (2) 次。
    - 新的 `res` = `"a" + "cc" = "acc"`。
9. **读到 `]`**:
    - 出栈：拿到 `p = ("", 3)`。
    - 计算：用 `p.s` ("") 加上当前 `res` ("acc") 重复 `p.k` (3) 次。
    - 新的 `res` = `"" + "acc" + "acc" + "acc" = "accaccacc"`。

遍历结束，返回 `res.toString()`，得到最终答案 `"accaccacc"`。
