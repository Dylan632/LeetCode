## 题目
![[79.png]]
**提示：**

- `m == board.length`
- `n = board[i].length`
- `1 <= m, n <= 6`
- `1 <= word.length <= 15`
- `board` 和 `word` 仅由大小写英文字母组成
## 思路（必填）
### 问题分析
在二维字符网格中搜索一个单词，单词必须按照字母顺序通过相邻单元格（上下左右）构成，且同一单元格不能重复使用。
### 算法选择
DFS + 回溯
- 从每个格子出发进行深度优先搜索
- 用回溯标记访问状态，避免重复使用

### 实现步骤
1. 遍历 board 每个格子作为起点
2. DFS 搜索：检查当前字符是否匹配 `word[k]`
3. 若匹配，标记为已访问，递归搜索四个相邻格子
4. 若任一分支找到完整单词，返回 true
5. 恢复现场（回溯），尝试其他路径

## 关键点
>[!Tip] 
>- 使用 `board[i][j] = '0'` 标记访问，避免重复访问同一个格子
>  - 递归返回后必须恢复现场，保证其他路径的正确性

## 代码

```java
class Solution {  
    private static final int[][] DIRS = {{0, -1}, {0, 1}, {-1, 0}, {1, 0}};  
  
    public boolean exist(char[][] board, String word) {  
        char[] w = word.toCharArray();  
        for (int i = 0; i < board.length; i++) {  
            for (int j = 0; j < board[i].length; j++) {  
                if (dfs(i, j, 0, board, w)) {  
                    return true; // 搜到了！  
                }  
            }  
        }  
        return false; // 没搜到  
    }  
  
    private boolean dfs(int i, int j, int k, char[][] board, char[] word) {  
        if (board[i][j] != word[k]) {  
            // 匹配失败  
            return false;  
        }  
        if (k == word.length - 1) {  
            // 匹配成功！  
            return true;  
        }  
        // 标记访问过  
        board[i][j] = 0;  
        // 相邻格子  
        for (int[] d : DIRS) {  
            int x = i + d[0];  
            int y = j + d[1];  
            if (0 <= x && x < board.length && 0 <= y && y < board[x].length && dfs(x, y, k + 1, board, word)) {  
                // 搜到了！  
                return true;  
            }  
        }  
        // 恢复现场  
        board[i][j] = word[k];  
        // 没搜到  
        return false;  
    }  
}
```

## 知识点
### 一、 核心解题思路

本质是**深度优先搜索 (DFS)** 结合 **回溯 (Backtracking)**。
通过双重循环将网格中的每一个点都作为潜在的起点，然后通过 DFS 像"接力寻宝"一样向四周试探。如果某条路走不通，就退回上一步（回溯），换个方向继续试。

### 二、 代码模块拆解

#### 1. 全局方向数组 `DIRS`

```java
private static final int[][] DIRS = {{0, -1}, {0, 1}, {-1, 0}, {1, 0}};
```

- **严格执行顺序**：向左 $\rightarrow$ 向右 $\rightarrow$ 向上 $\rightarrow$ 向下。
- **作用**：配合循环，优雅地计算相邻格子的坐标，避免写大量冗余的 `if-else`。

#### 2. 主入口 `exist`：起点的遍历与性能优化

- **参数优化技巧**：为了避免在 DFS 中频繁调用 `String.charAt()` 影响性能，提前将 `String word` 转换为 `char[] w`。
- **变量名作用域**：`exist` 中的 `w` 和 `dfs` 参数里接收的 `word`，虽然名字不同，但**底层指向的是同一块内存地址（同一个字符数组）**。这只是参数在不同方法中的命名占位符，传递的是引用。
- **双重循环**：遍历 `board` 的每一个格子 `(i, j)` 作为起点，只要有一次 DFS 返回 `true`，就说明找到了单词。

#### 3. 核心递归 `dfs`：四大步曲

这是算法的灵魂，严格遵循以下四个步骤：
- **第一步：边界与终止条件（剪枝与冲线）**
    - `if (board[i][j] != word[k])`：当前字母对不上，直接返回 `false`（剪枝）。
    - `if (k == word.length - 1)`：字母对上了，且已经是单词最后一个字母，匹配成功，返回 `true`（冲线）。
- **第二步：标记当前节点为"已访问"**
    - `board[i][j] = 0`：将当前字符修改为一个非字母字符（如 `0`），防止在同一条寻找路径中"走回头路"导致死循环。这是一种极其高效的**原地修改**技巧，省去了额外的 `visited` 二维数组。
- **第三步：向四周深搜（严格顺序与短路机制）**
    - 遍历 `DIRS` 寻找下一个字母 `k + 1`。
    - **极度关键的短路机制**：按照 **左 $\rightarrow$ 右 $\rightarrow$ 上 $\rightarrow$ 下** 的顺序执行。如果"向左"递归调用返回了 `true`，`if` 条件成立，代码会直接执行 `return true;` 结束当前层。**排在后面的右、上、下方向会被直接"短路"（跳过），根本不会执行。** 只有前面的方向返回 `false` 碰壁时，才会去尝试后续方向。
- **第四步：恢复现场（回溯）**
    - `board[i][j] = word[k]`：如果四个方向都试过了且都返回了 `false`，说明这条路彻底不通。在返回 `false` 给上一层之前，**必须**把修改成 `0` 的字符恢复成原本的字母。因为当前格子在"这条路线"上没用，但在"别的路线"上可能还会被访问到。

## 复杂度

- Time: $O(mn \cdot 3^k)$，其中 $m$ 和 $n$ 分别为 grid 的行数和列数，$k$ 是 word 的长度。除了递归入口，其余递归至多有 3 个分支（因为至少有一个方向是之前走过的），所以每次递归（回溯）的时间复杂度为 $O(3^k)$，一共回溯 $O(mn)$ 次。
- Space: $O(|\Sigma|+k)$，其中 $|\Sigma|=52$ 是字符集合的大小。递归需要 $O(k)$ 的栈空间。部分语言用的数组代替哈希表，可以视作 $|\Sigma|=128$。
