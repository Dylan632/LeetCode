## 思路（必填）
### 问题分析
要求将矩阵中 0 所在的行和列都清零。约束条件是不能在遍历过程中边发现边修改（否则新改出来的 0 会污染后续的判断）。
### 算法选择
选择了**两次遍历 + 额外数组标记法**。这种做法逻辑严密，在空间复杂度和实现难度之间取得了极好的平衡。
### 实现步骤
1. 扫描整个矩阵，当发现 `matrix[i][j] == 0` 时，在标记数组中记下“第 i 行有 0” (`rowHasZero[i] = true`) 和“第 j 列有 0” (`colHasZero[j] = true`)。
2. 再次完整扫描矩阵，对于每一个元素 `matrix[i][j]`，只要它所在的行或列在刚才的标记数组中为 `true`，就将其赋值为 0。

## 关键点
>[!Tip] 
>状态分离。将“寻找靶点”和“执行清零”这两个动作分在了两次循环中独立完成。

## 代码

```java
class Solution {  
    public void setZeroes(int[][] matrix) {  
        int m = matrix.length;  
        int n = matrix[0].length;  
        boolean[] rowHasZero = new boolean[m];  
        boolean[] colHasZero = new boolean[n];  
  
        for (int i = 0; i < m; i++) {  
            for (int j = 0; j < n; j++) {  
                if (matrix[i][j] == 0) {  
                    rowHasZero[i] = colHasZero[j] = true;  
                }  
            }  
        }  
  
        for (int i = 0; i < m; i++) {  
            for (int j = 0; j < n; j++) {  
                if (rowHasZero[i] || colHasZero[j]) {  
                    matrix[i][j] = 0;  
                }  
            }  
        }  
    }  
}
```

## 知识点
问：能不能在发现 `matrix[i][j]=0` 时，直接把 i 行 j 列全变成 0？

答：这样做是错误的。如示例 1，如果直接全变成 0，那么继续遍历，发现 `matrix[1][2]=0`，我们会误以为 `matrix[1][2]` 一开始就是 0，然后把 2 列（最右边那一列）全变成 0。
