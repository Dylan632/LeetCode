## 题目
![[139.png]]
![[139_1.png]]

**提示：**

- `1 <= s.length <= 300`
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 20`
- `s` 和 `wordDict[i]` 仅由小写英文字母组成
- `wordDict` 中的所有字符串**互不相同**

## 思路（必填）
### 算法选择

动态规划。dp[i] 表示 `s[0..i)` 是否可被拆分。对每个位置 i，枚举可能的分割点 j，检查“前缀可拆分”且“后缀在字典中”。

### 实现步骤
1. 字典转 HashSet，获得 O(1) 查找能力
2. 预计算 maxLen，限制内层遍历范围
3. dp[0] = true，空串可拆分
4. 对 i 从 1 到 n，遍历分割点 j（范围 [max(0, i-maxLen), i)）
5. 先查 dp[j]（O(1)），再查 wordSet.contains(s.substring(j, i))
6. 两者都为 true 则 dp[i] = true，break
7. 返回 dp[n]

### 状态转移图解

以 s = “leetcode”，字典 = [“leet”, “code”] 为例：

```
索引:  0  1  2  3  4  5  6  7  8
字符:     l  e  e  t  c  o  d  e
dp:   T  F  F  F  T  F  F  F  T
                  ↑           ↑
             "leet"在字典中   "code"在字典中
             且dp[0]=T       且dp[4]=T
```

- dp[4]：j=0 时，dp[0]=true 且 “leet” 在字典中 → dp[4]=true
- dp[8]：j=4 时，dp[4]=true 且 “code” 在字典中 → dp[8]=true

## 关键点

> [!Tip]
> - maxLen 剪枝是核心优化——子串长度不可能超过最长单词，大幅缩小搜索空间
> - 短路顺序是隐藏优化——dp[j] 放前面，为 false 时跳过昂贵的字符串操作

## 代码

```java
class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        Set<String> wordSet = new HashSet<>(wordDict);
        int n = s.length();

        // 记录字典中单词最大长度，限制内层遍历范围
        int maxLen = 0;
        for (String word : wordDict) {
            maxLen = Math.max(maxLen, word.length());
        }

        boolean[] dp = new boolean[n + 1];
        dp[0] = true;

        for (int i = 1; i <= n; i++) {
            // 遍历分割点，利用 HashSet O(1) 查找
            for (int j = Math.max(0, i - maxLen); j < i; j++) {
                if (dp[j] && wordSet.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[n];
    }
}
```
