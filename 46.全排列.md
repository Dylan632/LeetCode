## 思路（必填）

### 问题分析

- 输入：一个不含重复元素的数组 nums
- 输出：所有可能的全排列
- 约束：必须使用所有元素，每个元素只能使用一次

### 算法选择

选择回溯算法，因为：

1. 需要枚举所有可能的解 → 搜索问题
2. 解空间呈树状结构 → 深度优先搜索
3. 需要记录路径并回退 → 回溯框架

### 实现步骤

1. 初始化状态空间：
   - path：记录当前排列，长度固定为 n
   - onPath：标记哪些元素已被使用，避免重复选择
2. 递归填充位置：
   - 位置 i 从 0 开始，依次填入数字
   - 对于每个位置，遍历所有未被使用的元素，尝试填入
3. 递归终止：
   - 当 i == n 时，所有位置已填满，得到一个完整排列
   - 将当前 path 的副本加入结果集
4. 回溯恢复：
   - 递归返回后，恢复 `onPath[j] = false`
   - path 无需恢复，因为后续会直接覆盖

## 关键点

> [!Tip] 核心问题
> - **为什么需要 onPath 数组？**
>   因为排列要求每个元素只能使用一次，onPath 防止重复选择
> - **为什么 path 无需恢复现场？**
>   因为排列长度固定为 n，path[i] 会被直接覆盖，不像组合问题需要动态增删
> - **为什么结果要添加副本 new ArrayList<>(path)？**
>   因为 path 是引用，后续递归会修改其内容，必须深拷贝

## 回溯框架

```java
/* 回溯算法框架 */ 
void backtrack(State state, List<Choice> choices, List<State> res) { 
	// 判断是否为解 
	if (isSolution(state)) { 
		// 记录解 
		recordSolution(state, res); 
		// 不再继续搜索 
		return; 
	} 
	// 遍历所有选择 
	for (Choice choice : choices) { 
		// 剪枝：判断选择是否合法 
		if (isValid(state, choice)) { 
			// 尝试：做出选择，更新状态 
			makeChoice(state, choice); 
			backtrack(state, choices, res); 
			// 回退：撤销选择，恢复到之前的状态 
			undoChoice(state, choice); 
		} 
	} 
}
```

## 代码

```java
class Solution {
    public List<List<Integer>> permute(int[] nums) {
        int n = nums.length;
        // 所有排列的长度都是一样的 n
        List<Integer> path = Arrays.asList(new Integer[n]);
        boolean[] onPath = new boolean[n];
        List<List<Integer>> ans = new ArrayList<>();
        dfs(0, nums, ans, path, onPath);
        return ans;
    }

    // 枚举 path[i] 填 nums 的哪个数
    private void dfs(int i, int[] nums, List<List<Integer>> ans, List<Integer> path, boolean[] onPath) {
        if (i == nums.length) {
            ans.add(new ArrayList<>(path));
            return;
        }
        for (int j = 0; j < nums.length; j++) {
            if (!onPath[j]) {
                path.set(i, nums[j]); // 从没有选的数字中选一个
                onPath[j] = true; // 已选上
                dfs(i + 1, nums, ans, path, onPath);
                onPath[j] = false; // 恢复现场
                // 注意 path 无需恢复现场，因为排列长度固定，直接覆盖就行
            }
        }
    }
}
```

为了实现每个元素只被选择一次，我们考虑引入一个布尔型数组 `onPath` ，其中 `onPath[i]` 表示 `choices[i]` 是否已被选择，并基于它实现以下剪枝操作。 
- 在做出选择 `choice[i]` 后，我们就将 `onPath[i]` 赋值为 `True` ，代表它已被选择。
- 遍历选择列表 `choices` 时，跳过所有已被选择的节点，即剪枝。

如下图所示，假设我们第一轮选择 1 ，第二轮选择 3 ，第三轮选择 2 ，则需要在第二轮剪掉元素 1 的分支，在第三轮剪掉元素 1 和元素 3 的分支。

## 代码（交换法）

```java
class Solution {
    List<Integer> nums;
    List<List<Integer>> res;

    void swap(int a, int b) {
        int tmp = nums.get(a);
        nums.set(a, nums.get(b));
        nums.set(b, tmp);
    }

    void dfs(int x) {
        if (x == nums.size() - 1) {
            res.add(new ArrayList<>(nums));
            return;
        }

        for (int i = x; i < nums.size(); i++) {
            swap(i, x);
            dfs(x + 1);
            swap(i, x);
        }
    }

    public List<List<Integer>> permute(int[] nums) {
        this.res = new ArrayList<>();
        this.nums = new ArrayList<>();
        for (int num : nums) {
            this.nums.add(num);
        }
        dfs(0);
        return res;
    }
}
```

## 知识点

写在同一个 `class Solution` 里面的方法，可以直接调用 `swap()`。**同一个类里的方法之间，可以直接互相调用。**

### 1. 回溯算法三要素

- 路径：已做出的选择（path）
- 选择列表：当前可以做的选择（未使用的元素）
- 结束条件：到达决策树底层（i == n）

### 2. 回溯算法模板

```
void backtrack(路径, 选择列表):
    if 满足结束条件:
        result.add(路径副本)
        return

    for 选择 in 选择列表:
        做选择
        backtrack(路径, 选择列表)
        撤销选择
```

### 3. 排列 vs 组合的区别

| 特性         | 排列                   | 组合                      |
| ------------ | ---------------------- | ------------------------- |
| 顺序是否重要 | 是                     | 否                        |
| 元素可否重复 | 否                     | 否                        |
| 回溯特点     | 需要 onPath 标记       | 需要 start 参数避免重复   |
