## 思路（必填）
### 问题分析

这是 LeetCode 第 48 题“旋转图像”。题目要求将一个 n x n 的二维矩阵顺时针旋转 90 度，且拥有硬性约束：必须 **原地（in-place）** 修改，不能分配另一个二维矩阵来进行旋转。

### 算法选择
选择了 **数学规律映射法** 。如果不使用额外空间，直接进行“外圈到内圈的四点旋转”虽然可行，但下标边界计算极为繁琐且容易越界。利用矩阵转置的特性可以巧妙地避开复杂的边界控制。

### 实现步骤
1. **主对角线转置**：遍历矩阵的左下半部分（条件为 `j < i`），将 `matrix[i][j]` 与 `matrix[j][i]` 互换。这一步相当于将矩阵沿着左上到右下的对角线进行翻转，使原来的行变成了列。
2. **水平翻转**：遍历转置后矩阵的每一行，将行内的元素左右对称互换（`row[j]` 与 `row[n - 1 - j]`）。 经过这两步，矩阵刚好完成了顺时针 90 度的视觉旋转。

## 关键点
>[!Tip] 
>将一步到位、容易出错的“旋转动作”，巧妙拆解为了两个简单的“翻转动作”（转置 + 镜像翻转），极大地降低了编码心智负担。

## 代码

```java
class Solution {  
    public void rotate(int[][] matrix) {  
        int n = matrix.length;  
        for (int i = 0; i < n; i++) {  
            for (int j = 0; j < i; j++) {  
                int tmp = matrix[i][j];  
                matrix[i][j] = matrix[j][i];  
                matrix[j][i] = tmp;  
            }  
        }  
        for (int[] row : matrix) {  
            for (int j = 0; j < n / 2; j++) {  
                int tmp = row[j];  
                row[j] = row[n - 1 - j];  
                row[n - 1 - j] = tmp;  
            }  
        }  
    }  
}
```

## 知识点
顺时针旋转 90 度的终极目标公式：
原始坐标 $(i, j) \rightarrow$ 目标坐标 $(j, n-1-i)$
看到这个目标，你会怎么拆解它？
- **第一步：发现 `i` 和 `j` 交换了位置。**
    在矩阵操作中，什么动作能让行下标和列下标互换？显然是**转置（Transpose）**，即沿主对角线对折。
    转置后的坐标变化是：$(i, j) \rightarrow (j, i)$
- **第二步：发现列下标变成了 $n-1-i$。**
    现在的坐标是 $(j, i)$，我们需要把右边的 $i$ 变成 $n-1-i$。
    在一个长度为 $n$ 的一维数组中，把索引 $x$ 变成 $n-1-x$，代表什么操作？代表**中心对称翻转（Reverse）**。既然是对列下标进行翻转，那就是在每一行内部做左右翻转。
    经过这一步：$(j, i) \rightarrow (j, n-1-i)$

## 复杂度

- Time: O(n²)
- Space: O(1)
`n` 为矩阵的边长。转置矩阵需要遍历约一半的元素，反转每一行也需要遍历一半的元素，总操作次数与矩阵元素总数成线性关系。原地修改，仅使用了常数级的额外空间 `tmp`。
