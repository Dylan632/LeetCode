## 思路（必填）
### 问题分析
给定一个包含非负整数的 $m \times n$ 网格，要求找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。每次只能向下或者向右移动一步。
### 算法选择
动态规划（Dynamic Programming）。因为当前格子的最小路径和仅仅取决于其“上方”格子和“左侧”格子的最小路径和，符合最优子结构和无后效性。
### 实现步骤
1. **状态定义**：使用一维滚动数组 `dp` 记录上一行的最优解以及当前行正在计算的最优解。
2. **巧妙初始化**：`dp` 数组长度设为列数加1。`dp[0]` 固定为 `MAX_VALUE`（阻断越界访问的可能）。`dp[1] = 0` 是为了让第一个数字 `grid[0][0]` 计算时，能够正确推导为 `0 + grid[0][0]`。
3. **状态转移**：遍历每一行每一列，公式为 `dp[j+1] = min(左侧路径和 dp[j], 上方路径和 dp[j+1]) + 当前格子值`。

## 关键点
>[!Tip] 
>利用 `dp[0] = Integer.MAX_VALUE` 充当了一堵永远不可跨越的“左墙”，不仅免去了对每一行第一列的特殊 `if` 判断，还保持了核心循环内的代码高度简洁。

## 代码

```java
class Solution {  
    public int minPathSum(int[][] grid) {  
        int n = grid[0].length;  
        int[] dp = new int[n + 1];  
        // dp[0] 是那堵永远是无穷大的“左墙”  
        Arrays.fill(dp, Integer.MAX_VALUE);  
        dp[1] = 0;  
        for (int[] row : grid) {  
            for (int j = 0; j < n; j++) {  
	            // dp[j] 代表当前行左边格子的路径和 
	            // dp[j+1] 在被覆盖前，代表上一行同列格子的路径和
                dp[j + 1] = Math.min(dp[j], dp[j + 1]) + row[j];  
            }  
        }  
        return dp[n];  
    }  
}
```

## 知识点


## 复杂度

- Time: O(mn)
- Space: O(m)
需要遍历整个二维网格 $m$ 行 $n$ 列；使用了一个长度为 $n+1$ 的一维数组进行状态转移，将原本 $O(m \times n)$ 的空间压缩到了 $O(n)$。
