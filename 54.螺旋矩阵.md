## 题目
![[54.png]]

## 思路（必填）
### 问题分析
  - 给定一个 m × n 矩阵，要求按顺时针螺旋顺序返回所有元素
  - 关键在于：如何优雅地处理方向的转换和边界的收缩
### 算法选择
  - 使用方向数组 + 步数控制的方法，避免复杂的边界判断
  - 方向数组将四个方向的移动统一为 (i + dx, j + dy) 的模式
  - 步数规律：每走完两个方向，步数减 1（如：右走 n 步，下走 m-1 步，左走 n-1 步，上走 m-2 步...）
### 实现步骤

1. 从 `(0, -1)` 开始（巧妙设计）
2. 按照 **右→下→左→上** 的顺序循环
3. 每走完一个方向，更新步数为下一个维度的剩余数减 1
4. 直到收集完所有元素

![[54_1.png]]
示例 2 这 12 个数字，可以分为以下 5 组：
```
1→2→3→4
8→12
11→10→9
5
6→7
```
其中第 1,3,5 组都是向右或者向左走的，长度依次为 4,3,2，这是一个从 n=4 开始的逐渐递减的序列。其中第 2,4 组都是向下或者向上走的，长度依次为 2,1，这是一个从 m−1=2 开始的逐渐递减的序列。由于走的步数是有规律的，我们可以精确地控制在每个方向上要走多少步，无需判断是否出界、是否重复访问：

从 (0,−1) 开始。
一开始，向右走 n 步，<font color="#ffc000">每次先走一步，再把数字加入答案</font>。走 n 步即 1→2→3→4，矩阵第一排的数都加入了答案。
然后向下走 m−1 步，即 8→12。
然后向左走 n−1 步，即 11→10→9。
然后向上走 m−2 步，即 5。
然后向右走 n−2 步，即 6→7。
重复上述过程，直到答案的长度等于 mn。

代码实现时，可以这样简化代码：
一开始走 n 步。
把 n,m 分别更新为 m−1,n，这样下一轮循环又可以走 n 步（相当于走了 m−1 步），无需修改其他逻辑。
把 n,m 分别更新为 m−1,n，这样下一轮循环又可以走 n 步（相当于走了 n−1 步）。
把 n,m 分别更新为 m−1,n，这样下一轮循环又可以走 n 步（相当于走了 m−2 步）。
依此类推，每次只需把 n,m 分别更新为 m−1,n 即可。

## 关键点
> [!Tip]
> - 方向切换：`di = (di + 1) % 4` 循环切换右→下→左→上
> - 变量复用：`m` 和 `n` 交替更新，表示当前这一轮要走的步数

## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    // 方向数组DIRS,表示右下左上  
    private static final int[][] DIRS = {{0, 1}, {1, 0}, {0, -1}, {-1, 0}}; 
    public List<Integer> spiralOrder(int[][] matrix) {  
        int m = matrix.length; // 当前剩余区域的“高”  
        int n = matrix[0].length; // 当前这一轮要走的步数  
        int size = m * n; // 总元素个数  
        List<Integer> ans = new ArrayList<>(m * n);  
        int i = 0;  
        // 从矩阵左边一格开始，这样第一步向右就能到 (0,0)        
        int j = -1;  
        // di 表示方向下标：0右 1下 2左 3上  
        for (int di = 0; ans.size() < size; di = (di + 1) % 4) {  
            // 朝当前方向走 n 步  
            for (int k = 0; k < n; k++) {  
                i += DIRS[di][0];  
                j += DIRS[di][1];  
                ans.add(matrix[i][j]);  
            }  
            // 更新下一轮的步数  
            // 步数规律：n, m-1, n-1, m-2, ...  
            int tmp = n;  
            n = m - 1;  
            m = tmp;  
        }  
        return ans;  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点

### 1. 方向数组 DIRS

```java
private static final int[][] DIRS = {{0, 1}, {1, 0}, {0, -1}, {-1, 0}}; 
// 右下左上
```

- `{0, 1}`：向右走，行不变，列 +1
- `{1, 0}`：向下走，行 +1，列不变
- `{0, -1}`：向左走，行不变，列 -1
- `{-1, 0}`：向上走，行 -1，列不变

### 2. 起点 `(0, -1)` 的设计

代码采用 **“先迈步，再记录”** 的逻辑：
- 如果从 `(0, 0)` 开始，第一步向右会变成 `(0, 1)`，漏掉第一个元素
- 从 `(0, -1)` 开始，第一步向右正好到 `(0, 0)`，逻辑统一

### 3. 循环结构

外层循环：
```java
for (int di = 0; ans.size() < size; di = (di + 1) % 4)
```
- `di` 表示方向：0右、1下、2左、3上
- `(di + 1) % 4` 实现方向循环切换：右 → 下 → 左 → 上 → 右 → ...
内层循环：朝当前方向走 `n` 步，每一步更新坐标并加入答案

### 4. 步数更新的精髓

```java
int tmp = n;
n = m - 1;
m = tmp;
```

螺旋走的步数规律：`n, m-1, n-1, m-2, ...`
- 横着走时，长度与“列数”有关
- 竖着走时，长度与“行数”有关
- 每走完一条边，下一方向的可走长度少 1
这三句通过交换 `m`、`n` 并减 1，自动生成上述序列。


## 复杂度

- Time: O(mn)
- Space: O(mn)
