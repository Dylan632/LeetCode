## 题目
![[1.png]]

## 思路（必填）
![[1_1.png]]![[1_2.png]]
1. 创建 `HashMap`，用于存储已遍历过的 `数值 → 下标` 映射
2. 遍历数组，对当前元素 `nums[i]` 计算互补数 `complement = target - nums[i]`
3. 若 `complement` 已在 `map` 中，说明找到了配对，返回两个下标
4. 否则将当前元素加入 `map`，继续遍历

## 关键点
> [!Tip] 先查后存——遍历到 i 时 map 中只有 0...i-1 的元素，天然避免了”使用同一元素两次”的问题。

## 代码
```java
class Solution {  
    public int[] twoSum(int[] nums, int target) {  
        // key: 数值, value: 该数值在数组中的下标  
        Map<Integer, Integer> map = new HashMap<>();  
        for (int i = 0; i < nums.length; i++) {  
            int complement = target - nums[i];  
            // 遍历当前元素，并在map中寻找是否有匹配的key  
            if (map.containsKey(complement)) {  
                return new int[]{map.get(complement), i};  
            }  
            // 如果没找到匹配对，就把访问过的元素和下标加入到map中  
            map.put(nums[i], i);  
        }  
        // 对于这道题来说，题目保证一定有解，所以这行代码实际上永远不会执行。它存在的唯一原因是  
        //  Java 编译器要求方法的所有路径都必须有 return 语句，不写的话编译不过。  
        return new int[0];  
    }  
}
```

## 复杂度

- Time: O(n)
- Space: O(1)
