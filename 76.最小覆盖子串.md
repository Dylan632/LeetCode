## 思路（必填）

### 问题分析
- 在字符串 `s` 中找到包含字符串 `t` 所有字符（包括重复字符）的最短子串
- `t` 中可能有重复字符，窗口中对应字符的数量必须 `>=` `t` 中的数量
- 要求返回最短的符合条件的子串

### 算法选择
- 滑动窗口：适合处理“连续区间”、“最短/最长子串”问题
- 使用 `diff` 数组技巧：`diff[i]` = 窗口中字符 `i` 的个数 - `t` 中字符 `i` 的个数

| `diff[c]` 的值 | 状态含义                  | 动作影响                                                                          |
| :----------: | :-------------------- | :---------------------------------------------------------------------------- |
| **负数** `< 0` | **欠账状态**：窗口里还缺这个字符    | 右指针遇到该字符时，`diff[c]++`，填补欠账                                                    |
| **零** = = 0  | **刚好状态**：该字符数量不多不少刚刚好 | **关键触发点**：<br>1. 从 `<0` 变成 `0`：`geCnt++`<br>2. 从 `0` 变成 `<0`（左指针踢出）：`geCnt--` |
| **正数** `> 0` | **冗余状态**：窗口里这个字符多余了   | 左指针遇到该字符时，直接踢出，不影响 `geCnt`                                                    |

### 实现步骤

**前期准备**：遍历 `t`，统计需要的 `kinds`，并把需要的字符在 `diff` 中记为负数（欠账）。

**步骤 1：右指针进窗口（扩张）**

- 每次把 `sArr[right]` 纳入窗口，`diff[c]++`。
- 如果填平了某项欠账（`diff[c] == 0`），满足的种类数加一（`geCnt++`）。

**步骤 2：左指针出窗口（收缩）**

- **触发条件**：`while (geCnt == kinds)`。只要窗口合法，就一直尝试收缩。
- 先记录/更新当前的最短子串答案。
- 尝试踢出 `sArr[left]`，即 `diff[x]--`。
- 如果踢出前该字符是“刚好”状态（`diff[x] == 0`），一旦踢出就会重新欠账，导致 `geCnt--`，从而打破 `while` 循环，让右指针继续干活。

## 关键点

>[!Tip]
>- 使用 `diff` 数组避免每次比较窗口和 `t` 的字符统计，时间复杂度从 O(26mn) 优化到 O(m + n)
>- `geCnt` 变量：记录窗口中有多少种字符的数量已经满足要求，避免每次遍历 `diff` 数组
>- `diff[c] == 0` 的判断：这是判断“字符数量刚好满足”的关键条件
>- `substring(ansLeft, ansRight + 1)` 是左闭右开区间，右边界记得 `+1`。如果 `ansLeft` 一直是初始值 `-1`，记得返回 `""` 而不是去截取

## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)
class Solution {
    public String minWindow(String s, String t) {
        // 窗口每种字母个数 - t 每种字母个数
        int[] diff = new int[128];
        int kinds = 0;
        for (char c : t.toCharArray()) {
            if (diff[c] == 0) {
                // 统计 t 有多少个不同的字母
                kinds++;
            }
            diff[c]--;
        }

        char[] sArr = s.toCharArray();
        int m = sArr.length;
        int ansLeft = -1;
        int ansRight = m;
        // 窗口内有 geCnt 种字母的出现次数 >= t 中相应字母的出现次数
        int geCnt = 0;
        int left = 0;

        // 移动子串右端点
        for (int right = 0; right < m; right++) {
            // 右端点字母
            char c = sArr[right];
            // 右端点字母移入子串
            diff[c]++;
            // 原来窗口内 c 的出现次数比 t 的少，现在一样多
            if (diff[c] == 0) {
                // 从 < 变成 >=
                geCnt++;
            }

            // 涵盖：所有字母的出现次数都是 >=
            while (geCnt == kinds) {
                // 找到更短的子串
                if (right - left < ansRight - ansLeft) {
                    // 记录此时的左右端点
                    ansLeft = left;
                    ansRight = right;
                }

                // 左端点字母
                char x = sArr[left];
                if (diff[x] == 0) {
                    // x 移出窗口之前，检查出现次数，
                    // 如果窗口内 x 的出现次数和 t 一样，
                    // 那么 x 移出窗口后，窗口内 x 的出现次数比 t 的少
                    geCnt--; // 从 >= 变成 <
                }
                // 左端点字母移出子串
                diff[x]--;
                left++;
            }
        }
        return ansLeft < 0 ? "" : s.substring(ansLeft, ansRight + 1);
    }
}
//leetcode submit region end(Prohibit modification and deletion)
```

## 🔑 灵魂变量解析

- **`int[] diff = new int[128]`**：记录当前窗口与目标字符串 `t` 的字符数量差值（利用 ASCII 码隐式转换，如 `diff['A']` 就是 `diff[65]`）。
- **`int kinds`**：目标字符串 `t` 中包含的**字符种类数**（去重后的数量）。
- **`int geCnt`**：当前窗口内，数量已经达标的**字符种类数**。当 `geCnt == kinds` 时，说明当前窗口合法，可以开始收缩左边界。

## 复杂度

- Time: O(m+n)
- Space: O(1)

双指针各遍历一次，`diff` 数组固定 128 个槽位。
