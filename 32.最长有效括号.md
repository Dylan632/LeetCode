## 思路（必填）
### 算法选择

本题求最长有效括号子串，有三种经典解法：

| 方法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| **栈** | O(n) | O(n) | 直观易理解，用哨兵处理边界 |
| **动态规划** | O(n) | O(n) | 状态转移清晰，需处理两种情况 |
| **双指针** | O(n) | O(1) | 空间最优，需正反两遍扫描 |

本文采用**栈**方法，核心思想：
- 栈存储**下标**而非字符，方便计算长度
- 用哨兵（初始 -1）统一处理边界情况
- 栈顶元素始终表示当前有效括号的左边界之前的位置

### 实现步骤

1. **初始化**：栈底先放 `-1`，作为初始哨兵（虚拟红线）。
2. **遍历字符串**：
   - 遇到 `(`：把当前下标入栈（相当于埋下"炸弹"）。
   - 遇到 `)`：
     - 如果栈中有可匹配的 `(`（栈大小 > 1）：弹出栈顶，用 `当前下标 - 新栈顶` 计算有效长度。
     - 如果没有可匹配的 `(`（栈大小 = 1）：当前 `)` 成为新的哨兵，替换栈底。
3. **返回结果**：遍历结束后返回记录的最大长度。

## 关键点
>[!Tip]
> - 栈里存的是**下标**，不是字符，因为最后要算长度。
> - 弹栈之后新的栈顶，代表当前有效括号段左边界之前的位置。
> - 哨兵的作用是统一处理开头就遇到 `)` 或中间断开的情况。

## 代码

```java
class Solution {  
    public int longestValidParentheses(String s) {  
        List<Integer> stk = new ArrayList<>();  
        stk.add(-1); // 虚拟红线（哨兵）  
        int ans = 0;  
        for (int i = 0; i < s.length(); i++) {  
            if (s.charAt(i) == '(') { // 炸弹  
                stk.add(i); // 记录炸弹下标  
            } else if (stk.size() > 1) { // 栈顶是炸弹  
                stk.removeLast(); // 拆弹  
                // 右端点为 i 时，左端点最小是 stk.getLast() + 1                
                ans = Math.max(ans, i - stk.getLast());  
            } else {  // 栈只有一个数，是红线，s[i] 无法拆弹，成为新的红线  
                stk.set(0,i); // 替换栈底  
            }  
        }  
        return ans;  
    }  
}
```
## 举例实现步骤
假设字符串是 `s = "(()"`
- **初始**：栈为 `[-1]` （虚拟红线）
- `i = 0`，遇到 `(` ：入栈，栈变成 `[-1, 0]`
- `i = 1`，遇到 `(` ：入栈，栈变成 `[-1, 0, 1]`
- `i = 2`，遇到 `)` ：此时发生配对！
    1. **拆弹**：先把 `i=1` 的那个左括号出栈，栈变回 `[-1, 0]`。
    2. **找边界**：现在的栈顶是 `0`（代表第 0 个位置的那个 `(` 还没有被匹配掉）。
    3. **算端点**：对于当前的右端点 `i=2`，阻碍它往左延伸的边界是下标 `0`。所以，当前的有效子串只能从 `0 + 1 = 1` 开始。
    4. **计算长度**：这段有效子串（即中间的 `()`）的长度是 `2 - 0 = 2`。

## 深入理解：为什么 `i - stk.getLast()` 能算出长度

弹栈后，栈顶元素表示**当前有效括号段的左边界之前的位置**，它可能是：
- 初始哨兵 `-1`：前面全部匹配成功
- 多余的 `)` 的下标：该位置发生断层
- 未匹配的 `(` 的下标：外层还有未闭合的左括号

因此，有效子串的范围是 `[stk.getLast() + 1, i]`，长度为：
```
i - (stk.getLast() + 1) + 1 = i - stk.getLast()
```

- Time: O(n)
- Space: O(n)
