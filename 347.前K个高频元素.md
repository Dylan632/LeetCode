## 题目
![[347.png]]

## 思路（必填）
### 问题分析
  - 给定一个整数数组 nums 和一个整数 k
  - 要求返回出现频率前 k 高的元素
  - 可以按任意顺序返回答案
  - 题目保证答案唯一（即恰好有 k 个不同的高频元素）
### 算法选择
  选择桶排序的原因：
  1. 频率范围有限：频率的最大值最多为 nums.length，适合作为桶的索引
  2. 时间复杂度优：桶排序可以在 O(n) 时间内完成，优于优先队列的 O(n log k)
  3. 空间换时间：通过额外的空间存储桶，实现线性时间复杂度
### 实现步骤
1. 既然答案与元素的出现次数有关，那么先用一个哈希表 `cnt` 统计每个元素的出现次数。哈希表的 `key` 是元素值，`value` 是 `key` 在数组中的出现次数。此时问题变成：返回一个列表，包含前 `k` 大的出现次数对应的元素值。
2. 把出现次数相同的元素，放到同一个桶中。同一个桶内的元素出现次数相同，无需对桶内元素排序。设出现次数的最大值为 `maxCnt`。创建一个大小为 `maxCnt+1` 的列表 `buckets`，其中 `buckets[c]` 存储出现次数为 `c` 的元素。（每个 `buckets[c]` 都是一个列表）。注意 `maxCnt`≤`n`，这保证了时间复杂度是 O(n) 的。遍历 `cnt`，把出现次数为 `c` 的元素 `x` 添加到 `buckets[c]` 中。
3. 倒序遍历 `buckets`，把 `buckets[c]` 中的元素加到答案中。一旦答案的长度等于 `k`，就立刻返回答案。

注 1：题目保证答案唯一，所以一定会出现答案长度恰好等于 k 的情况。
注 2：可以按任意顺序返回答案。比如示例 1 返回 `[1,2]` 还是 `[2,1]` 都是正确的。

## 关键点
> [!Tip]
> - 桶的索引设计：桶的索引直接表示频率，巧妙利用了“频率一定是正整数”的特性
> - 倒序遍历：从最大频率开始倒序遍历，确保先收集高频元素
> - 边界条件：虽然题目保证答案唯一，但代码中应加上 `i >= 0` 的判断以防万一

## 代码

```java
class Solution {  
    public int[] topKFrequent(int[] nums, int k) {  
        // 第一步：统计每个元素的出现次数  
        Map<Integer, Integer> cnt = new HashMap<>();  
        for (int x : nums) {  
            cnt.merge(x, 1, Integer::sum);  
        }  
        int maxCnt = Collections.max(cnt.values());  
  
        // 第二步：把出现次数相同的元素，放到同一个桶中  
        List<Integer>[] buckets = new ArrayList[maxCnt + 1];  
        Arrays.setAll(buckets, _ -> new ArrayList<>());  
        for (Map.Entry<Integer, Integer> e : cnt.entrySet()) {  
            buckets[e.getValue()].add(e.getKey());  
        }  
  
        // 第三步：倒序遍历 buckets，把出现次数前 k 大的元素加入答案  
        int[] ans = new int[k];  
        int j = 0;  
        for (int i = maxCnt; j < k; i--) {  
            // 注意题目保证答案唯一，一定会出现某次循环结束后 j 恰好等于 k 的情况  
            for (int x : buckets[i]) {  
                ans[j++] = x;  
            }  
        }  
        return ans;  
    }  
}
```

## 知识点
### 1. 简写合并方法
```java
cnt.merge(x, 1, Integer::sum);
```
- **`merge` 方法**：Java 8 引入的高效更新 Map 的手段。
    - 如果 Key 不存在：存入 `(x, 1)`。
    - 如果 Key 已存在：执行第三个参数的逻辑更新 Value。
- **`Integer::sum`（方法引用）**：
    - 这是 Lambda 表达式 `(oldV, newV) -> oldV + newV` 的极简缩写。
    - 它告诉程序：如果数字已存在，就把旧次数和 1 相加。
- **作用**：统计数组中每个元素的出现次数。如果数字 `x` 是第一次出现，记录次数为 1；如果已经存在，则将旧次数与 1 相加（`Integer::sum`）。
- **内部状态变化（以 `[1, 1, 1, 2, 2, 3]` 为例）**：
    - 初始：`{}`
    - 遍历完所有 `1`：`{1: 3}`
    - 遍历完所有 `2`：`{1: 3, 2: 2}`
    - 遍历结束：`{1: 3, 2: 2, 3: 1}` （即：数字 1 出现了 3 次，数字 2 出现了 2 次...）

### 2. 获取最大频率
```java
int maxCnt = Collections.max(cnt.values());
```
- **`cnt.values()`**：剥离哈希表中的“键”，只提取所有的“值”（即所有元素的出现次数集合，如 `[3, 2, 1]`）。
- **`Collections.max()`**：找出这个集合中的最大值（例如 3）。
- **设计目的**：为了知道**最多需要建多少个“桶”**。最高频率是多少，我们的桶数组下标就得能取到多少（所以数组长度设定为 `maxCnt + 1`）。

### 3.初始化桶与Lambda 表达式
```java
Arrays.setAll(buckets, _ -> new ArrayList<>());
```
**Lambda 表达式 `_ -> ...`**：
- `_`（下划线）：代表“忽略该参数”。在初始化时我们只需要 new 对象，不需要用到当前的索引值。
- **作用**：相比传统的 `for` 循环赋值，这种写法更加声明式，代码极简。

### 4. 理解键值对对象：`Map.Entry`
```java
for (Map.Entry<Integer, Integer> e : cnt.entrySet()) {
    buckets[e.getValue()].add(e.getKey());
}
```
- **概念**：`Entry` 的意思是“条目”或“记录”。在 Java 中，它代表哈希表里的**一个完整的键值对 (Key-Value Pair)**。
- **`entrySet()`**：将字典里所有的“记录条目”打包成一个集合返回。`[1=3, 2=2, 3=1]`
- **类比**：如果 `Map` 是一本《英汉词典》，那么 `Entry` 就是词典里的一条完整解释（如：Apple -> 苹果）。
- **用法提取**：
    - `e.getKey()`：提取数字本身。
    - `e.getValue()`：提取该数字的出现频率。
- **代码实战**：通过 `for (Map.Entry<Integer, Integer> e : cnt.entrySet())`，我们可以在一次循环中同时拿到数字及其频率，方便地执行 `buckets[频率].add(数字)` 的装桶操作。
