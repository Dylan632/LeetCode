## 思路（必填）
### 问题分析
题目要求实现获取 **下一个排列** 的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。如果已经是最大排列（整体单调递减），则重排为最小排列（整体单调递增）。必须原地修改且只允许常量额外空间。
### 算法选择
为了找到“刚好大于”当前排列的字典序，我们需要尽可能保持高位不变，寻找低位中能变大的可能。因此从后向前查找是唯一的选择（基于数学/字典序规律）。
### 实现步骤
1. **寻找尽头（确定交换位置 `i`）**：从右往左遍历，找到第一个“破坏”单调递增规律的元素 `nums[i]`（即 `nums[i] < nums[i+1]`）。此时，`nums[i]` 右侧的子序列是完全单调递减的，意味着右侧这部分已经达到了它们能组成的最大字典序。
2. **寻找替补（确定交换对象 `j`）**：想要让序列变大，我们需要把 `nums[i]` 换成一个稍微大一点点的数。因为 `nums[i]` 右侧是递减的，所以我们再次从右向左找，找到第一个严格大于 `nums[i]` 的元素 `nums[j]`，将二者交换。
3. **重置尾部（反转）**：交换后，`nums[i]` 右侧依然是降序的（即局部最大）。为了得到“刚好大一点”的字典序，我们需要把这一局部最大段变成局部最小段，因此直接反转右侧降序区间 `[i+1, n-1]` 即可。

## 关键点
>[!Tip] 
>只要找到了拐点 `i`，就能确定可优化的最高位；而通过从右向左找 `j` 以及最终的无脑反转 `reverse` 极大地简化了重排序的操作过程。

## 代码

```java
class Solution {  
    public void nextPermutation(int[] nums) {  
        int n = nums.length;  
  
        // 第一步：从右向左找到第一个小于右侧相邻数字的数 nums[i]        
        int i = n - 2;  
        while (i >= 0 && nums[i] >= nums[i + 1]) {  
            i--;  
        }  
  
        //如果找到了，进入第二步；否则跳过第二步，反转整个数组  
        if (i >= 0) {  
            // 第二步：从右向左找到 nums[i] 右边最小的大于 nums[i] 的数 nums[j]   
            int j = n - 1;  
            while (nums[j] <= nums[i]) {  
                j--;  
            }  
            // 交换 nums[i] 和 nums[j]            
            swap(nums, i, j);  
        }  
        // 第三步：反转 [i+1, n-1]（如果上面跳过第二步，此时 i = -1）  
        reverse(nums, i + 1, n - 1);  
    }  
  
    private void swap(int[] nums, int i, int j) {  
        int tmp = nums[i];  
        nums[i] = nums[j];  
        nums[j] = tmp;  
    }  
  
    private void reverse(int[] nums, int left, int right) {  
        while (left < right) {  
            swap(nums, left++, right--);  
        }  
    }  
}
```

## 复杂度

- Time: O(n)
- Space: O(1)
其中 n 是 nums 的长度。最坏情况下需要遍历整个 nums 数组。
