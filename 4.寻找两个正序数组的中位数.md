## 思路（必填）
### 问题分析
题目要求在两个已排序的数组中找到中位数，并且强制要求时间复杂度为 $O(\log(m+n))$。看到 $\log$，且数组有序，必须想到**二分查找**。
### 算法选择
为了找到中位数，实际上是要找到一个**分割线**，将两个数组分别划分为左右两部分，使得：
1. **数量完美平衡**：左右两部分的总元素个数相等（或左边多一个）。
2. **数值完全有序**：左边部分的最大值 $\le$ 右边部分的最小值。
### 实现步骤
1. **预处理**：保证 `nums1` 是长度较短的数组，从而减少二分查找的区间，并防止计算 `nums2` 分割点时越界。
2. **初始化二分区间**：设定开区间 `left = -1` 和 `right = m`，在短数组 `nums1` 的索引区间中进行二分。
3. **二分查找分割线**：
    - 计算 `nums1` 的分割点 `i` 和对应 `nums2` 的分割点 `j`。
    - 比较交叉边界：如果 `nums1[i] < nums2[j + 1]`，说明 `nums1` 的分割线还需要往右移，更新左边界 `left = i`。
    - 否则说明 `nums1` 分割线右侧的元素太小（或左侧太大），更新右边界 `right = i`。
4. **处理边界并返回结果**：二分结束后，分割线确定为 `left`。取出分割线两侧的 4 个临界值（用系统极值处理越界），根据总长度的奇偶性计算最终的中位数。

## 关键点
>[!Tip] 
>这道题最难的不是想到二分，而是如何**处理奇数和偶数长度的统一性**以及**越界（某一个数组的元素全部被划分到左边或右边）**。通过统一的数学公式 `(m + n + 1) / 2` 以及 `Integer.MIN_VALUE / MAX_VALUE` 完美化解了这两个痛点。

## 代码

```java
class Solution {  
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {  
        if (nums1.length > nums2.length) {  
            int[] tmp = nums1;  
            nums1 = nums2;  
            nums2 = tmp;  
        }  
  
        int m = nums1.length;  
        int n = nums2.length;  
        // 循环不变量：nums1[left] <= nums2[j + 1]  
        // 循环不变量：nums1[right] > nums2[j + 1]  
        int left = -1;  
        int right = m;  
        while (left + 1 < right) {  
            // 开区间 (left, right) 不为空  
            int i = left + (right - left) / 2;  
            int j = (m + n + 1) / 2 - i - 2;  
            if (nums1[i] < nums2[j + 1]) {  
                // 缩小二分区间为 (i, right)                
                left = i;  
            } else {  
                // 缩小二分区间为 (left, i)                
                right = i;  
            }  
        }  
        // 此时 left 等于 right-1        
        // nums1[left] <= nums2[j+1] 且 nums1[right] > nums2[(j-1)+1] = nums2[j]，所以答案是 i=left        
        int i = left;  
        int j = (m + n + 1) / 2 - i - 2;  
        int nums1LeftMax = i >= 0 ? nums1[i] : Integer.MIN_VALUE;  
        int nums2LeftMax = j >= 0 ? nums2[j] : Integer.MIN_VALUE;  
        int nums1RightMin = i + 1 < m ? nums1[i + 1] : Integer.MAX_VALUE;  
        int nums2RightMin = j + 1 < n ? nums2[j + 1] : Integer.MAX_VALUE;  
        int max1 = Math.max(nums1LeftMax, nums2LeftMax);  
        int min2 = Math.min(nums1RightMin, nums2RightMin);  
        return (m + n) % 2 > 0 ? max1 : (max1 + min2) / 2.0;  
    }  
}
```

## 知识点
### 二分查找寻找切分点 `i`
开区间写法 `(-1, length)`
```Java
int left = -1;  
int right = m;  
while (left + 1 < right) {  
    int i = left + (right - left) / 2;  
    int j = (m + n + 1) / 2 - i - 2;  
    if (nums1[i] < nums2[j + 1]) {  
        left = i;  
    } else {  
        right = i;  
    }  
}  
```
- **`i` 和 `j` 的含义**：
    - `i` 代表 `nums1` 中属于**左半边**的最后一个元素的下标。这意味着 `nums1` 贡献了 $i + 1$ 个元素给左半边。
    - `j` 代表 `nums2` 中属于**左半边**的最后一个元素的下标。

- **神仙公式 `j = (m + n + 1) / 2 - i - 2`**：
    - 左半边总共需要 $\lfloor\frac{m + n + 1}{2}\rfloor$ 个元素。
    - `nums1` 出了 $i + 1$ 个，`nums2` 就必须出 $\lfloor\frac{m + n + 1}{2}\rfloor - (i + 1)$ 个。
    - 既然 `nums2` 出了这么多元素，那么它左半边最后一个元素的下标 `j` 就是：个数 $- 1$，所以 $j = \lfloor\frac{m + n + 1}{2}\rfloor - i - 2$。

- **判断逻辑**：
    - `nums1[i] < nums2[j + 1]`：如果 `nums1` 左边的最大值，竟然比 `nums2` 右边的最小值还要小，说明 `nums1` 在左半边贡献的元素**太小或太少了**。我们需要把切分线往右移，所以 `left = i`。
    - 反之说明 `nums1` 贡献的元素太大了，切分线需要左移，所以 `right = i`。


## 复杂度

- Time: O($log(\min(m,n)$)
- Space: O(1)
$O(1)$每次循环将二分查找区间缩小一半，且只在较短的数组上进行二分查找。只使用了几个基本类型变量，无额外空间开销。
