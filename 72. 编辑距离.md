## 题目
![[72.png]]
**提示：**

- `0 <= word1.length, word2.length <= 500`
- `word1` 和 `word2` 由小写英文字母组成
## 思路（必填）
### 问题分析
  - 给定两个单词 word1 和 word2，计算将 word1 转换为 word2 的最少操作数
  - 允许的操作：插入、删除、替换（每个操作算 1 步）
  - 这是经典的编辑距离问题

### 算法选择
  - 选择动态规划，因为问题具有最优子结构和重叠子问题特性
  - 定义状态 `dp[i][j] `表示 `word1[0..i-1]` 转换为 `word2[0..j-1]` 的最少操作数
### 实现步骤
  1. 状态定义：`dp[i][j]` 表示 `word1` 前 `i` 个字符转换为 `word2` 前 `j` 个字符的最少操作数
  2. 初始化：
    - `dp[0][j] = j`：空串转换为长度为 `j` 的串需要 `j` 次插入
    - `dp[i][0] = i`：长度为 `i` 的串转换为空串需要 `i` 次删除
  3. 状态转移：
    - 如果 `word1[i-1] == word2[j-1]`：`dp[i][j] = dp[i-1][j-1]`（无需操作）
    - 否则取三种操作的最小值 + 1：
        - 删除：`dp[i-1][j] + 1`（删除 `word1` 的第 `i` 个字符）
        - 插入：`dp[i][j-1] + 1`（插入 `word2` 的第 `j` 个字符）
        - 替换：`dp[i-1][j-1] + 1`（将 `word1[i-1]` 替换为 `word2[j-1]`）
  4. 结果：`dp[len1][len2]`

## 关键点

>[!Tip]
> `dp[i][j]` 的含义：`word1` 前 `i` 个字符转成 `word2` 前 `j` 个字符的最小编辑距离
>
> - 理解状态转移方程中删除、插入、替换的含义
> - 删除：从 `dp[i-1][j]` 转移而来，表示 `word1` 已经处理到 `i-1`，现在删除第 `i` 个字符
> - 插入：从 `dp[i][j-1]` 转移而来，表示在 `word1` 位置 `i` 插入一个字符匹配 `word2[j-1]`
> - 替换：从 `dp[i-1][j-1]` 转移而来，表示将 `word1[i-1]` 替换为 `word2[j-1]`
## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    public int minDistance(String word1, String word2) {  
        char[] chars1 = word1.toCharArray();  
        char[] chars2 = word2.toCharArray();  
        int len1 = chars1.length;  
        int len2 = chars2.length;  
        int[][] dp = new int[len1 + 1][len2 + 1];  
        for (int j = 0; j < len2; j++) {  
            dp[0][j + 1] = j + 1;  
        }  
        for (int i = 0; i < len1; i++) {  
            dp[i + 1][0] = i + 1;  
            for (int j = 0; j < len2; j++) {  
                // 三种操作取最小：  
                // 1. 删除：dp[i][j+1]  
                // 2. 插入：dp[i+1][j]  
                // 3. 替换：dp[i][j]  
                dp[i + 1][j + 1] = chars1[i] == chars2[j] ? dp[i][j] : Math.min(Math.min(dp[i][j + 1], dp[i + 1][j]), dp[i][j]) + 1;  
            }  
        }  
        return dp[len1][len2];  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点
### 最关键的一行：状态转移

```java
dp[i + 1][j + 1] = chars1[i] == chars2[j] ? dp[i][j] :
        Math.min(Math.min(dp[i][j + 1], dp[i + 1][j]), dp[i][j]) + 1;
```
这行是整个题目的核心。我们分两种情况讲。

#### 情况 1：当前字符相同

```java
chars1[i] == chars2[j]
```
比如 `chars1[i] = 'a'`，`chars2[j] = 'a'`，最后一个字符已经匹配上了，不需要额外操作。
```java
dp[i + 1][j + 1] = dp[i][j]
```
意思是：前面怎么变，最后这个相同字符直接对上就行。

#### 情况 2：当前字符不同

```java
chars1[i] != chars2[j]
```
必须做一次操作，有三种选择：

**删除**：`dp[i][j + 1]` → 先把 `word1` 前 `i` 个字符变成 `word2` 前 `j+1` 个字符，然后删掉 `chars1[i]`。
```java
dp[i][j + 1] + 1
```

**插入**：`dp[i + 1][j]` → 先把 `word1` 前 `i+1` 个字符变成 `word2` 前 `j` 个字符，然后插入 `chars2[j]`。
```java
dp[i + 1][j] + 1
```

**替换**：`dp[i][j]` → 先把 `word1` 前 `i` 个字符变成 `word2` 前 `j` 个字符，然后把 `chars1[i]` 替换成 `chars2[j]`。
```java
dp[i][j] + 1
```

三种取最小：
```java
dp[i + 1][j + 1] = min(
    dp[i][j + 1],   // 删除
    dp[i + 1][j],   // 插入
    dp[i][j]        // 替换
) + 1;
```

也就是代码里的：
```java
Math.min(Math.min(dp[i][j + 1], dp[i + 1][j]), dp[i][j]) + 1
```

### LeetCode 经典：编辑距离 (Edit Distance) - DP 求解

**💡 核心思路**

动态规划（Dynamic Programming）。将长字符串的转换问题，拆解为较短字符串转换的子问题。

#### 1. 状态定义 (最关键的一步)

- 定义二维数组 `dp[i][j]`。

- **含义**：表示将 `word1` 的前 `i` 个字符，转换成 `word2` 的前 `j` 个字符，所需要的**最少操作次数**。

- **技巧**：数组大小设为 `[len1+1][len2+1]`。多出来的 `0` 索引代表**空字符串**，这样可以极其方便地处理边界情况，不用写一堆 `if-else`。

#### 2. 边界初始化 (Base Cases)

处理其中一个字符串为空的情况：

- `dp[i][0] = i`：把长度为 `i` 的 `word1` 变成空串，只能操作**删除** `i` 次。

- `dp[0][j] = j`：把空串变成长度为 `j` 的 `word2`，只能操作**插入** `j` 次。

#### 3. 状态转移方程 (核心逻辑)

遍历两个字符串的字符。假设当前对比的是 `word1` 的第 `i` 个字符和 `word2` 的第 `j` 个字符：

- **情况 A：当前字符相同 (`chars1[i] == chars2[j]`)**

    - **做法**：什么都不用做，直接继承它们前面部分的步数。

    - **方程**：`dp[i][j] = dp[i-1][j-1]`

- **情况 B：当前字符不同 (`chars1[i] != chars2[j]`)**

    - **做法**：需要操作 1 次（必然是插入、删除、替换中的一种）。我们找前面状态中**步数最少**的一个，然后 `+ 1`。

    - **三种操作对应的历史状态**：

        1. **替换**：`dp[i-1][j-1]`（前缀已经相同了，最后这俩不同的字符替换一下即可）

        2. **删除**：`dp[i-1][j]`（删掉 `word1` 的当前字符，说明 `word1` 的前 `i-1` 个字符已经匹配了 `word2` 的前 `j` 个）

        3. **插入**：`dp[i][j-1]`（在 `word1` 末尾插入一个字符，说明 `word1` 的前 `i` 个字符已经匹配了 `word2` 的前 `j-1` 个）

    - **方程**：`dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1`

(注：代码中由于有空串偏移，索引写成 `dp[i+1][j+1]`，但逻辑与上述完全一致)_

#### 4. 返回结果与复杂度

- **最终结果**：返回矩阵右下角的值 `dp[len1][len2]`。

- **时间复杂度**：$O(n \times m)$（需要填满整个二维表格）。

- **空间复杂度**：$O(n \times m)$（开辟了二维数组，进阶可使用滚动数组优化至一维）。

  - 二维 DP → 所有涉及两个序列匹配的问题
  - 状态定义技巧 → 用 i 和 j 分别表示两个序列的处理进度
  - 空间优化思想 → 只依赖上一行时可降维
