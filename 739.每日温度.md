## 思路（必填）
### 问题分析
给定每日温度数组，求每一天需要等多少天才能遇到更高的温度。如果没有则填 0。
### 算法选择
单调递减栈。栈中存储索引，对应的温度值单调递减。
### 实现步骤
  1. 遍历每一天的温度
  2. 当栈非空且当前温度 > 栈顶索引对应的温度时：
    - 弹出栈顶索引 j
    - 计算等待天数：i - j，存入 `ans[j]`
  3. 将当前索引入栈，等待后续清算
  4. 栈中剩余的索引对应的日子永远等不到更高温，保持 ans 默认值 0

## 关键点
>[!tip] 
> - 单调栈的本质：维护一个「待办清单」，存储还没找到答案的索引
> - 清算时机：当遇到比栈顶更大的元素时，说明栈顶元素的答案确定了
> - <font color="#de7802">及时去掉无用数据，保证栈中数据有序</font>

## 代码

```java
class Solution {  
    public int[] dailyTemperatures(int[] temperatures) {  
        int n = temperatures.length;  
        int[] ans = new int[n];  
        // todolist  
        Deque<Integer> st = new ArrayDeque<>();  
        for (int i = 0; i < n; i++) {  
            int t = temperatures[i];  
            // 核心逻辑：清算“小本本（todolist）”  
            // 当栈不为空，并且“今天的温度 t”大于“栈顶所记录的那天的温度”时：  
            // 说明我们终于找到了栈顶那一天一直在等的高温！  
            while (!st.isEmpty() && t > temperatures[st.peek()]) {  
                // 把过去那一天（索引 j）从栈里弹出来  
                int j = st.pop();  
                // 计算等待的天数：今天 (i) 减去 过去那一天 (j)
                ans[j] = i - j;  
            }  
            // 将今天加入待办事项  
            // 无论今天有没有帮以前的日子“清算”完，今天本身的“下一个更高温度”还是未知的。  
            // 所以必须把今天的索引 i 压入栈中，留给未来的日子去清算。  
            st.push(i);  
        }  
        return ans;  
    }  
}
```

## 知识点
### 动态图文推演（Dry Run）

为了让你更直观地理解，我们用一个简化的例子跑一遍代码。 假设温度数组是：`[73, 74, 75, 71, 69, 72]`

- **第 0 天 (73度)：** 栈是空的，直接进栈。
    - 栈：`[0]`
- **第 1 天 (74度)：** 74度比栈顶（第0天的73度）高。
    - 触发 `while` 循环，弹出 `0`。
    - 计算答案：`ans[0] = 1 - 0 = 1`（等了1天）。
    - 第 1 天进栈。
    - 栈：`[1]`，结果：`[1, 0, 0, 0, 0, 0]`
- **第 2 天 (75度)：** 75度比栈顶（第1天的74度）高。
    - 弹出 `1`，计算 `ans[1] = 2 - 1 = 1`。
    - 第 2 天进栈。
    - 栈：`[2]`，结果：`[1, 1, 0, 0, 0, 0]`
- **第 3 天 (71度)：** 71度比栈顶（第2天的75度）低，无法清算，只能进栈等。
    - 栈：`[2, 3]`
- **第 4 天 (69度)：** 69度比栈顶（第3天的71度）低，继续进栈等。
    - 栈：`[2, 3, 4]`
- **第 5 天 (72度)：** 开始连环清算！
    - 比栈顶（第4天69度）高：弹出 `4`，`ans[4] = 5 - 4 = 1`。
    - 此时栈顶变成第3天(71度)，72度比它也高：弹出 `3`，`ans[3] = 5 - 3 = 2`。
    - 此时栈顶变成第2天(75度)，72度比它低，循环停止。
    - 第 5 天进栈。
    - 栈：`[2, 5]`，结果：`[1, 1, 0, 2, 1, 0]`

遍历结束，栈里剩下的 `2` 和 `5` 代表第 2 天和第 5 天后面再也没有比它们热的日子了。因为 `ans` 数组初始化为 0，所以不需要管它们，结果已经是正确的 0。
