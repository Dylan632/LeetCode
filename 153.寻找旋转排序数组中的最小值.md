## 题目
![[153.png]]

## 思路（必填）
### 问题分析
在一个旋转过的升序排序数组中查找最小值。例如 `[3,4,5,1,2]` 的最小值是 1。数组特性：至少有一部分是有序的。
### 算法选择
  选择二分查找而非线性扫描，因为：
  - 数组具有部分有序性质
  - 可以通过比较中间元素与最右元素判断最小值在哪一侧
  - 时间复杂度从 O(n) 优化到 O(log n)
### 实现步骤
  1. 初始化开区间 `left = -1`, `right = n-1`
  2. 计算 `mid = left + (right - left) / 2` 防止溢出
  3. 判断 `nums[mid]` 与 `nums[n-1]` 的关系：
    - 若 `nums[mid] < nums[n-1]`，说明 `mid` 在右半段有序部分，最小值在左侧（含 `mid`），`right = mid`
    - 否则 `mid` 在左半段有序部分，最小值在右侧，`left = mid`
  4. 当开区间为空时（`left + 1 >= right`），`right` 就是最小值位置

## 关键点

> [!Tip] 核心技巧
> - 使用开区间 $(left, right)$ 简化边界判断
> - 与最后一个元素比较，巧妙判断当前位置属于哪个有序段
> - 循环终止条件 `left + 1 < right` 确保找到精确解

## 代码

```java
class Solution {  
    public int findMin(int[] nums) {  
        int n = nums.length;  
        int left = -1;  
        int right = n - 1; // 开区间 (-1, n-1)        
        while (left + 1 < right) { // 开区间不为空  
            int mid = left + (right - left) / 2;  
            if (nums[mid] < nums[n - 1]) {  
                right = mid;  
            } else {  
                left = mid;  
            }  
        }  
        return nums[right];  
    }  
}
```

## 知识点

> [!Summary] 核心思想
> 通过比较 `mid` 与 `nums[n-1]` 的大小，间接判断 `mid` 与最小值的位置关系。

### 比较策略

设 $x = \text{nums}[mid]$，比较 $x$ 与 $\text{nums}[n-1]$：

| 情况 | 结论 |
|------|------|
| $x > \text{nums}[n-1]$ | $x$ 在第一段 → 最小值在 $x$ 右边 |
| $x \le \text{nums}[n-1]$ | $x$ 在第二段 → 最小值在 $x$ 左边或就是 $x$ |

### 情况分析

1. **$x > \text{nums}[n-1]$**（在第一段）
   - 数组被分成两个递增段
   - 第一段所有元素 > 第二段所有元素
   - $x$ 一定在最小值左边

2. **$x \le \text{nums}[n-1]$**（在第二段）
   - $x$ 要么是最小值，要么在最小值右边

### Q&A

> [!Question] 能否与 $\text{nums}[0]$ 比大小？
> 可以，但需额外处理边界情况：
> - 在 $(0, n)$ 中二分
> - 若结果等于 $n$，说明数组只有一段，答案为 $\text{nums}[0]$


## 细节

### 二分范围选择

- 使用开区间 $(-1, n-1)$，等价于闭区间 $[0, n-2]$
- 这种写法无需特殊处理边界

> [!Tip] 为什么可以排除 $n-1$？
> $right$ 一开始就指定了 $n-1$，若 $\text{nums}[n-1]$ 是最小值，每次比较 $x$ 与 $\text{nums}[n-1]$ 都有 $x > \text{nums}[n-1]$，只会更新 $left$。循环结束后答案自然落在 $n-1$。

### 概念区分

- **二分范围**：代码实际搜索的区间
- **答案范围**：答案可能出现的所有位置

> [!Note] 
> $\text{nums}[n-1]$ 无需在二分区间内，因为我们已经知道它一定是「蓝色」（最小值或其右侧）。


## 复杂度

- Time: O(log n)
- Space: O(1)
