## 思路（必填）

### 问题分析

- 题目要求在有序数组中找到目标值的索引或插入位置
- 必须满足 `O(log n)` 时间复杂度（暴力遍历 `O(n)` 不符合要求）
- 数组元素互不相同且升序排列

### 算法选择

选择二分查找的 `lower_bound`（找第一个 `>= target` 的元素），原因：

1. 有序数组天然适合二分查找
2. 需要找到第一个 `>= target` 的位置，而非精确匹配
3. 二分查找能在 `O(log n)` 时间内完成

### 实现步骤

1. 初始化开区间：`left = -1`, `right = n`，形成开区间 `(-1, n)`
2. 循环条件：`left + 1 < right` 确保区间不为空
3. 计算中点：`mid = left + (right - left) / 2` 防止整数溢出
4. 分支判断：
   - 若 `nums[mid] < target`，说明目标在右侧，收缩左边界 `left = mid`
   - 否则目标在左侧或当前位置，收缩右边界 `right = mid`
5. 返回结果：循环结束时，`right` 即为答案

## 关键点

- 开区间设计：避免闭区间的边界条件讨论（如 `left <= right` 还是 `left < right`）
- 循环不变量：始终保持 `nums[left] < target` 且 `nums[right] >= target`
- 防溢出：`left + (right - left) / 2` 比 `(left + right) / 2` 更安全

## 代码

```java
// 开区间
class Solution {  
    public int searchInsert(int[] nums, int target) {  
        return lowerBound(nums, target);  
    }  
  
    private int lowerBound(int[] nums, int target) {  
        int left = -1;  
        int right = nums.length; // 开区间 (left, right)        
        while (left + 1 < right) { // 区间不为空  
            // 循环不变量：  
            // nums[left] < target  
            // nums[right] >= target            
            int mid = left + (right - left) / 2;  
            if (nums[mid] < target) {  
                left = mid; // 范围缩小到 (mid, right)            
            } else {  
                right = mid; // 范围缩小到 (left, mid)            
            }  
        }  
        return right;  
    }  
}
```

```java
class Solution {  
    public int searchInsert(int[] nums, int target) {  
        int left = 0, right = nums.length - 1;  
        while (left <= right) {  
            int mid = left + (right - left) / 2;  
            if (nums[mid] == target) {
                // 1. 目标值等于数组中某一个元素
                return mid;
            } else if (nums[mid] > target) {  
                right = mid - 1;  
            } else {  
                left = mid + 1;  
            }  
        }  
        // 2. 目标值在数组所有元素之前
        // 3. 目标值插入数组中
        // 4. 目标值在数组所有元素之后
        return left; // 等价于 right + 1
    }  
}
```

闭区间写法说明：循环结束时 `left` 即为插入位置。

示例 2：输入 `[1, 3, 5, 6]`，`target = 2`

把二分查找想象成在找一条**分界线**。循环结束时，`left` 和 `right` 错开了，它们中间的那条缝，就是你要插入的地方。

![[35二分插入.png]]

- **`right`** 在左边守门（所有比 `target` 小的都在 `right` 及其左边）
- **`left`** 在右边守门（所有比 `target` 大的都在 `left` 及其右边）

数学上，当二分查找的 `while (left <= right)` 循环结束时，一定满足：`left = right + 1`

## 知识点

> [!Tip] 🔍 `lower_bound` 详细解释

### 定义

`lower_bound`：在有序数组中找到第一个大于等于目标值的位置

“lower” 表示下界，即第一个不小于（`≥`）`target` 的位置

`lower_bound` 的“变种”体现在：

1. 目标改变：从“精确匹配”变成“找边界”
2. 返回值改变：从“找到返回索引，找不到返回 `-1`”变成“总是返回一个有效位置”
3. 应用场景扩展：不仅用于查找，还用于插入、范围查询等

### 与标准二分的区别

| 对比项 | 标准二分查找 | `lower_bound` |
| ------ | ------------ | ------------- |
| 查找目标 | 等于 `target` | 第一个 `≥ target` |
| 找不到时 | 返回 `-1` | 返回应该插入的位置 |
| 返回值含义 | 精确匹配的索引 | 边界位置（可能不存在） |
| 应用场景 | 检查元素是否存在 | 插入位置、范围查询 |

### 举例说明

数组：`[1, 3, 5, 7, 9]`

**`target = 5`**

- 标准二分：返回 `2`（找到元素）
- `lower_bound`：返回 `2`（第一个 `>= 5` 的位置）

**`target = 6`**

- 标准二分：返回 `-1`（找不到）
- `lower_bound`：返回 `3`（数字 `7` 的位置，第一个 `>= 6`）

**`target = 10`**

- 标准二分：返回 `-1`
- `lower_bound`：返回 `5`（数组末尾，应该插入的位置）

### 🎯 为什么叫“变种”？

标准二分查找只判断三种情况：

```java
if (nums[mid] == target) return mid;    // 找到了
else if (nums[mid] < target) left = mid + 1;
else right = mid - 1;
```

`lower_bound` 变种的改动：

```java
if (nums[mid] < target) {
    left = mid;  // 小于 target，向右找
} else {
    right = mid;     // >= target，向左找边界（关键！）
}
```

核心区别：

- 标准：找到 == target 就返回
- `lower_bound`：即使 == target 也继续向左找，找第一个符合条件的位置

> [!Question] 为什么不在二分的过程中，找到 target 就立刻返回？
> 
> 这种写法适用范围很窄。在有多个等于 target 的数的情况下，如果在二分的过程中立刻返回，我们得到的可能不是第一个等于 target 的数的下标。为什么返回第一个等于 target 的数的下标更好呢？因为这可以解决更复杂的题目，比如给你一个有序数组 `[1,1,3,3,3,3,5]`，让你计算有多少个数小于 3。我们可以二分查找第一个等于 3 的数的下标 2，那么下标小于 2 的数就是元素值小于 3 的数，这有 2 个。对比可以发现，如果在二分中途就立刻返回，我们不一定找到的是第一个等于 3 的数的下标，所以不一定能算出正确答案 2。

> [!Question] 为什么代码没有特判所有数都小于 target 的情况？
> 
> 如果所有数都小于 target，那么循环中更新的只有 left，无论下面哪种二分写法，最后都一定会返回数组长度，所以无需特判这种情况。
