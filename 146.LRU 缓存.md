## 题目

![[146.png]]

## 思路（必填）

### 算法选择

- HashMap：实现 O(1) 的键值查找
- 双向链表：维护访问顺序，支持 O(1) 的节点移动和删除

### 实现步骤

![[146_1.png]]

1. 数据结构设计：
   - 内部类 Node 存储键值对和前后指针
   - HashMap<Integer, Node> 存储 key 到节点的映射
   - 双向循环链表维护访问顺序（头部最近使用，尾部最久未使用）
2. get 操作：
   - 通过 HashMap 查找节点
   - 若存在，将节点移到链表头部（表示最近使用）
   - 返回节点的值或 -1
3. put 操作：
   - 若 key 已存在：更新 value 并移到头部
   - 若 key 不存在：
     - 创建新节点并插入头部
     - 若超出容量，删除尾部节点（最久未使用）
4. 链表操作：
   - remove(Node)：断开节点的前后连接
   - pushFront(Node)：将节点插入到哨兵节点之后

## 关键点

> [!Tip]
> - 双向链表 + HashMap 组合：这是实现 LRU 缓存的经典方案
> - 哨兵节点技巧：避免了空指针判断，统一了链表操作逻辑
> - 维护两个索引：HashMap 提供 key→Node 的快速查找，链表维护访问顺序

## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)
class LRUCache {
    private static class Node {
        int key, value;
        Node prev, next;
        Node (int k, int v) {
            key = k;
            value = v;
        }
    }

    private final int capacity;
    private final Node dummy = new Node(0, 0);
    private final Map<Integer, Node> keyToNode = new HashMap<>();

    public LRUCache(int capacity) {
        this.capacity = capacity;
        dummy.prev = dummy;
        dummy.next = dummy;
    }

    public int get(int key) {
        // getNode 会把对应节点移到链表头部
        Node node = getNode(key);
        return node != null ? node.value : -1;
    }

    public void put(int key, int value) {
        // getNode 会把对应节点移到链表头部
        Node node = getNode(key);
        // 有这本书
        if (node != null) {
            // 更新 value
            node.value = value;
            return;
        }
        // 新书
        node = new Node(key, value);
        keyToNode.put(key, node);
        // 放在最上面
        pushFront(node);
        // 如果放完之后书架满了，就要扔掉最旧的书
        if (keyToNode.size() > capacity) {
            // 书太多了
            Node backNode = dummy.prev;
            // 从目录里划掉（这就是为什么 Node 里必须存 key）
            keyToNode.remove(backNode.key);
            // 去掉最后一本书
            remove(backNode);
        }
    }

    // 获取 key 对应的 Node，并把这个 Node 移到链表头部
    private Node getNode(int key) {
        // 没有这本书
        if (!keyToNode.containsKey(key)) {
            return null;
        }
        // 有这本书
        Node node = keyToNode.get(key);
        // 把这本书抽出来
        remove(node);
        // 放在最上面
        pushFront(node);
        return node;
    }

    // 删除一个节点（抽出一本书）
    private void remove(Node x) {
        x.prev.next = x.next;
        x.next.prev = x.prev;
    }

    // 在链表头添加一个节点（把一本书放在最上面）
    private void pushFront(Node x) {
        x.prev = dummy;
        x.next = dummy.next;
        x.prev.next = x;
        x.next.prev = x;
    }
}

/**
 * Your LRUCache object will be instantiated and called as such:
 * LRUCache obj = new LRUCache(capacity);
 * int param_1 = obj.get(key);
 * obj.put(key,value);
 */
```

## 知识点

核心概念详解：

1. LRU（Least Recently Used）策略：
   - 最近访问的数据更可能被再次访问（局部性原理）
   - 淘汰最久未使用的数据
2. 为什么选择双向链表：
   - 单向链表删除节点需要知道前驱节点，需要 O(n) 查找
   - 双向链表可以直接获取前驱，实现 O(1) 删除
3. 为什么需要哨兵节点：
   - 避免处理链表为空的特殊情况
   - 避免处理头部、尾部节点的边界判断
   - 统一所有节点的操作逻辑
4. 时间复杂度保证：
   - HashMap 查找/插入/删除：平均 O(1)
   - 双向链表节点移动：O(1)
   - 组合后：所有操作 O(1)

## 复杂度

- Time: O(1) - get 和 put 操作均为常数时间
- Space: O(capacity) - 存储容量受限的键值对
