## 题目
![[236.png]]
**提示：**

- 树中节点数目在范围 `[2, 105]` 内。
- `-109 <= Node.val <= 109`
- 所有 `Node.val` `互不相同` 。
- `p != q`
- `p` 和 `q` 均存在于给定的二叉树中。

## 思路
采用**后序遍历**的递归方法。从底向上回溯，当节点 `p`、`q` 在节点 `root` 的异侧时，节点 `root` 即为最近公共祖先。

## 问题分析
- 给定一个二叉树和两个节点 p、q，找到它们的最近公共祖先（LCA）
- 节点本身也可以是自己的祖先
- 树中节点值唯一，p 和 q 一定存在于树中

## 算法选择
- 选择后序遍历（DFS），因为需要先知道左右子树的信息才能判断当前节点是否是 LCA
- 时间复杂度 O(n)，空间复杂度 O(h)（递归栈）

## 实现步骤
1. 递归终止条件：当前节点为空或等于 p 或 q 时返回当前节点
2. 后序遍历：先递归处理左子树和右子树
3. 判断逻辑：
   - 如果左右子树都返回非空，说明 p 和 q 分布在当前节点两侧，当前节点就是 LCA
   - 如果只有一边返回非空，说明 p 和 q 都在该子树中，返回该子树的结果
   - 如果两边都返回空，说明当前子树不包含 p 和 q，返回 null

## 关键点
>[!Tip] 递归返回值的含义
>返回值表示「最近公共祖先的候选项」，可能是：
>- 真正的 LCA（左右子树都找到了目标节点）
>- 节点 `p` 或 `q`（找到一个目标节点，等待上层判断）
>- `null`（子树内没找到任何目标节点）
>
>利用后序遍历的自底向上特性，找到的最深的满足条件的节点就是 LCA。通过返回值传递子树查找结果，避免全局变量。

## 代码

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) {
            return root; // 找到 p 或 q 就不往下递归了
        }

        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);
        if (left != null && right != null) { // 左右都找到
            return root; // 当前节点是最近公共祖先
        }
        // 如果只有左子树找到，就返回左子树的返回值
        // 如果只有右子树找到，就返回右子树的返回值
        // 如果左右子树都没有找到，就返回 null（注意此时 right = null）
        return left != null ? left : right;
    }
}
```

## 知识点

若 root 是 p,q 的最近公共祖先，则只可能为以下情况之一：
- p 和 q 在 root 的子树中，且分列 root 的异侧（即分别在左、右子树中）
- p = root，且 q 在 root 的左或右子树中
- q = root，且 p 在 root 的左或右子树中

![[236_1.png]]

**问**：lowestCommonAncestor 函数的返回值是什么意思？

**答**：返回值的准确含义是「最近公共祖先的候选项」。对于最外层的递归调用者来说，返回值是最近公共祖先的意思。但是在递归过程中，返回值可能是最近公共祖先，也可能是空节点（表示子树内没找到任何有用信息）、节点 p 或者节点 q（可能成为最近公共祖先，或者用来辅助判断上面的某个节点是否为最近公共祖先）。

**问**：为什么发现当前节点是 p 或者 q 就不再往下递归了？万一下面有 q 或者 p 呢？

**答**：如果下面有 q 或者 p，那么当前节点就是最近公共祖先，直接返回当前节点。如果下面没有，那既然都没有要找的节点了，也不需要递归，直接返回当前节点。
## 记忆

![[236_2.png]]

后序递归。先递归找左子树，再递归找右子树。
- 如果当前节点是 `null`，直接返回 `null`
- 如果当前节点是 `p` 或 `q`，直接返回当前节点
- 然后递归调用，看左右子树的返回结果：
  - 左右都非空，说明 `p` 和 `q` 分别在当前节点两侧，当前节点就是最近公共祖先
  - 如果只有一边非空，就把那一边继续往上返回
  - 如果都为空，就返回 `null`

## 复杂度

- 时间：O(n)，其中 n 为树节点数量，最坏情况需要遍历所有节点
- 空间：O(n)，递归深度取决于树的高度，最坏情况为链状树
