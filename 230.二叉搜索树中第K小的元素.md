## 题目

![[230.png]]

**提示**：

- 树中的节点数为 `n`
- `1 <= k <= n <= 10^4`
- `0 <= Node.val <= 10^4`

## 思路（必填）

### 问题分析

- 给定一个 BST，找到其中第 k 小的元素
- BST 的性质：中序遍历结果是有序序列
- 约束条件：1 ≤ k ≤ n（节点总数）

### 算法选择

- 选择中序遍历，利用 BST 的有序性
- 在遍历过程中计数，找到第 k 个元素即停止
- 时间复杂度 O(k)，空间复杂度 O(h)

### 实现步骤

1. 递归边界：如果当前节点是空节点，返回 -1，表示没有找到。注意题目保证节点值非负。
2. 执行中序遍历，先递归左子树。
3. 判断左子树的返回值 leftRes 是否为 -1。如果不是 -1，说明我们在左子树中找到了答案，返回 leftRes。如果是 -1，说明尚未找到答案，继续下一步。
4. 把 k 减少 1。如果 k=0，那么答案就是当前节点值，返回当前节点值。
5. 现在，答案要么在当前节点的右子树中，要么在除了当前子树的其余节点中。递归右子树，如果答案在右子树中，那么直接返回答案；如果答案不在右子树中，那么右子树也会返回 -1，由于当前子树搜索完毕，所以当前子树没有找到答案，返回 -1。综上所述，可以直接返回右子树的返回值。

## 关键点

> [!Tip]
> - 提前终止：一旦找到答案立即返回，避免无效遍历
> - 计数传递：需要可变计数，Java 中可用数组或 AtomicInteger
> - BST 性质：中序遍历“左-根-右”恰好从小到大访问

## 代码

```java
class Solution {  
    private int k;  
  
    public int kthSmallest(TreeNode root, int k) {  
        this.k = k;  
        return dfs(root);  
    }  
  
    private int dfs(TreeNode node) {  
        if (node == null) {  
            // 题目保证节点值非负，用 -1 表示没有找到  
            return -1;  
        }  
        int leftRes = dfs(node.left);  
        if (leftRes != -1) {  
            // 答案在左子树中
            // 如果左子树返回的不是 -1，说明在左子树里已经找到了第 k 小的节点！ 
            // 此时直接"剪枝"，把答案一层层向上返回，不需要再执行后面的代码了。  
            return leftRes;  
        }  
        if (--k == 0) {  
            // 答案就是当前节点 
            // 此时左边的节点已经全部遍历完了，轮到当前节点了。 
            // 我们将计数器减 1。如果 k 恰好等于 0，说明当前节点就是第 k 小的节点。 
            return node.val;  
        }  
        // 右子树会返回答案或者 -1
        // 左子树和当前节点都不是答案（k还没减到0），那答案肯定在比当前节点更大的右子树里。        
        return dfs(node.right);  
    }  
}
```

## 递归树执行图

假设我们有一棵二叉搜索树，我们要找第 $k = 3$ 小的元素：

```
       5
      / \
     3   6
    / \
   2   4
  /
 1
```

1. 程序一直向左走，先到了 `1` 的左子节点（`null`），返回 `-1`。
2. 回到节点 `1`，执行 `--k`，此时 `k=2`。`1` 返回找它的右子节点（`null`），返回 `-1`。
3. 回退到节点 `2`，执行 `--k`，此时 `k=1`。
4. 回退到节点 `3`，执行 `--k`，此时 `k=0`！触发 `if (--k == 0)` 条件。
5. 节点 `3` 直接返回自己的值 `3`。
6. 值 `3` 赋值给上一层的 `leftRes`，因为 `3 != -1`，所以上一层直接把 `3` 返回去，一路退回给主函数。
7. 右边的 `4`、`5`、`6` 根本就不会被遍历到（这就是代码中提前 `return leftRes` 的妙处，节省了时间）。

```
kthSmallest(root=5, k=3)
│
└── dfs(5)                                     ← 开始中序遍历 (初始 k = 3)
    │
    ├── dfs(3)                                 ← 递归左子树
    │   │
    │   ├── dfs(2)                             ← 递归左子树
    │   │   │
    │   │   ├── dfs(1)                         ← 递归左子树
    │   │   │   │
    │   │   │   ├── dfs(null) → return -1      ← 左子节点为空，返回 -1
    │   │   │   ├── leftRes = -1               ← 左边没找到
    │   │   │   ├── 💥 --k (k=3 变为 2)         ← 遍历到节点 1 (第 1 小的数)
    │   │   │   ├── k != 0                     ← 不是目标，继续
    │   │   │   ├── dfs(null) → return -1      ← 右子节点为空，返回 -1
    │   │   │   └── return -1                  ← 节点 1 处理完毕，未找到
    │   │   │
    │   │   ├── leftRes = -1                   ← 接收节点 1 的结果：没找到
    │   │   ├── 💥 --k (k=2 变为 1)             ← 遍历到节点 2 (第 2 小的数)
    │   │   ├── k != 0                         ← 不是目标，继续
    │   │   ├── dfs(null) → return -1          ← 节点 2 没有右子节点，返回 -1
    │   │   └── return -1                      ← 节点 2 处理完毕，未找到
    │   │
    │   ├── leftRes = -1                       ← 接收节点 2 的结果：没找到
    │   ├── 💥 --k (k=1 变为 0)                 ← 遍历到节点 3 (第 3 小的数)
    │   ├── 🎯 k == 0 命中！                    ← 找到目标了！
    │   └── return 3 ✅                         ← 直接返回节点的值：3
    │
    ├── leftRes = 3                            ← 节点 5 接收到左子树的结果为 3
    ├── ✂️ leftRes != -1 成立！                 ← 触发剪枝逻辑 (答案在左子树中找到了)
    │                                           (不再执行 --k，也不再递归寻找节点 6)
    └── return 3 ✅                             ← 向上层返回结果
 
返回最终结果: 3 ✅
```

### 💡 图解亮点解析

1. **深度优先深入**：程序会像探针一样，最先一直下钻到最左下角的节点 `1`（BST 中最小的节点）。
2. **`--k` 发生的位置**：注意看，`--k` 发生的时间点永远在**左子树返回之后，右子树递归之前**。这就是标准的中序遍历。
3. **剪枝操作 (`✂️`)**：当节点 `3` 返回正确答案 `3` 给节点 `5` 时，节点 `5` 发现 `leftRes != -1`，它立刻就知道“答案在我的左子树里已经找到了”，于是它**直接跳过了自己的右子树**（节点 `4` 和 `6` 根本没有被访问），大大提升了效率。

## 知识点

BST 核心性质：

- 左子树所有节点 < 根节点 < 右子树所有节点
- 中序遍历 = 有序序列（升序）
- 查找、插入、删除平均 O(log n)，最坏 O(n)

## 复杂度

- **Time**: O(H + K)，其中 $H$ 是树的高度。最开始找到最小的节点（最左下角）需要走 $O(H)$ 步，然后还需要再遍历 $k$ 个节点。最好的情况下是 $O(\log N + k)$，如果树退化成链表则是 $O(N)$。
- **Space**: O(H)，递归调用栈的最大深度等于树的高度。在最坏情况下（树退化成链表）是 $O(N)$，在平衡树的情况下是 $O(\log N)$。
