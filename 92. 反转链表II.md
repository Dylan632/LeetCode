## 思路（必填）
### 问题分析
  - 题目要求反转链表中从位置 left 到 right 的节点
  - 需要保持反转区间外的节点顺序不变
  - 只能使用一次遍历，空间复杂度 O(1)
### 算法选择
  - 使用头插法反转链表，这是反转链表的标准方法
  - 引入虚拟头节点简化边界情况处理（避免 left=1 的特殊判断）
  - 使用双指针思想：p0 指向反转区间前一个节点，cur 遍历反转区间
### 实现步骤
  1. 创建虚拟头节点：dummy 指向 head，统一处理 left=1 的情况
  2. 定位反转区间前节点：p0 移动到第 left-1 个节点（需要反转的区间的前一个节点）
  3. 反转链表区间：使用头插法，逐个反转 left 到 right 的节点
  4. 重新连接链表：
    - p0.next.next = cur：反转后的尾节点连接到后续节点
    - p0.next = pre：前节点连接到反转后的头节点
  5. 返回结果：dummy.next 为新链表的头节点

## 关键点
>[!Tip]
> - 虚拟头节点技巧：避免单独处理 left=1 的边界情况
> - 反转后重新连接：反转完成后需要正确连接链表的两端
> - 反转次数计算：right - left + 1 精确控制反转的节点数

## 代码

```java
class Solution {  
    public ListNode reverseBetween(ListNode head, int left, int right) {  
        ListNode dummy = new ListNode(0, head);  
        ListNode p0 = dummy;  
        for (int i = 0; i < left - 1; i++) {  
            p0 = p0.next;  
        }  
  
        ListNode pre = null;  
        ListNode cur = p0.next;  
        for (int i = 0; i < right - left + 1; i++) {  
            ListNode nxt = cur.next;  
            cur.next = pre;  
            pre = cur;  
            cur = nxt;  
        }  
  
        p0.next.next = cur;  
        p0.next = pre;  
        return dummy.next;  
    }  
}

```

## 知识点
### 核心反转循环（局部反转）

```Java
ListNode pre = null;
ListNode cur = p0.next; // cur 指向即将要反转的第一个节点
for (int i = 0; i < right - left + 1; i++) {
    ListNode nxt = cur.next; // 1. 暂存下一个节点，防止链表断裂丢失
    cur.next = pre;          // 2. 核心操作：当前节点调头指向前一个节点
    pre = cur;               // 3. pre 指针前移
    cur = nxt;               // 4. cur 指针前移
}
```

- 这是一段最标准的链表反转模板。
- 循环的次数正好是需要反转的节点个数（`right - left + 1`）。
- 当这个 `for` 循环结束时，发生了三件事：
    1. 中间部分的指针已经全部倒序了。
    2. `pre` 停留在了反转段的**最后一个节点**（也就是反转后的**新头节点**）。
    3. `cur` 停留在了反转段之后的**第一个未参与反转的节点**。
