## 思路（必填）
### 问题分析
  - 输入：已排序的非递减数组 nums，目标值 target
  - 输出：target 在数组中的起始和结束位置，不存在则返回 `[-1, -1]`
  - 要求：时间复杂度 O(log n)
### 算法选择
  - 选择二分查找：因为数组已排序，可以利用有序性快速定位
  - 使用开区间二分：避免边界处理出错，循环不变量更清晰
### 实现步骤
  1. 查找起始位置：用 lowerBound(nums, target) 找第一个 >= target 的位置
  2. 验证存在性：检查该位置是否真的等于 target（可能 > target）
  3. 查找结束位置：用 lowerBound(nums, target + 1) 找第一个 > target 的位置，减 1 得到最后一个 target

## 关键点
>[!Tip] 
>  - 循环不变量：始终保持 `nums[left] < target` 和 `nums[right] >= target`。最后循环结束时，left指向< target的最后一个数，right指向>= target的第一个数，开区间不包含任何数，此时right就是我们要的答案。
 >- 巧妙转化：通过找第一个 > target 的位置来间接得到最后一个 target
 > - 开区间技巧：初始 left = -1, right = n，避免边界讨论


## 代码

```java
class Solution {  
    public int[] searchRange(int[] nums, int target) {  
        int start = lowerBound(nums, target);  
        if (start == nums.length || nums[start] != target) {  
            // 花括号 `{}` 是 Java 数组初始化的标准语法，用于直接指定数组元素值。
            return new int[]{-1, -1}; // nums 中没有 target        
        }  
        // 如果 start 存在，那么 end 必定存在  
        int end = lowerBound(nums, target + 1) - 1;  
        return new int[]{start, end};  
    }  
  
	// lowerBound 返回最小的满足 nums[i] >= target 的下标 i        
	// 如果数组为空，或者所有数都 < target，则返回 nums.length        
	// 要求 nums 是非递减的，即 nums[i] <= nums[i + 1]    
	private int lowerBound(int[] nums, int target) {  
        int left = -1;  
        int right = nums.length; // 开区间 (left, right)        
        while (left + 1 < right) { // 区间不为空  
            // 循环不变量：  
            // nums[left] < target  
            // nums[right] >= target            
            int mid = left + (right - left) / 2;  
            if (nums[mid] >= target) {  
                right = mid; // 范围缩小到 (left, mid)            
            } else {  
                left = mid; // 范围缩小到 (mid, right)            
            }  
        }  
        // 循环结束后 left+1 = right        
        // 此时 nums[left] < target 而 nums[right] >= target        
        // 所以 right 就是第一个 >= target 的元素下标  
        return right;  
    }  
}
```

## 知识点

> [!Question] 如何理解 `end = lowerBound(nums, target + 1) - 1` 这段代码？
> 
> 要想找到 ≤target 的最后一个数，无需单独再写一个二分。我们可以先找到这个数的右边相邻数字，也就是 >target 的第一个数。在所有数都是整数的前提下，>target 等价于 ≥target+1，这样就可以复用我们已经写好的二分函数了，即 `lowerBound(nums, target + 1)`，算出这个数的下标后，将其减一，就得到 ≤target 的最后一个数的下标。

> [!Question] 如果 ≥target+1 的第一个数不存在怎么办？
> 
> 这说明数组中的数都 ≤target。如果数组中有 target，那么数组的最后一个数（下标 n−1）就是 target（因为数组是递增的）。同时，`lowerBound(nums, target + 1)` 在这种情况下会返回 n，减一得到 n−1，这正是我们要计算的下标。

> [!Question]  当数组中不存在 `≥target+1` 的元素时，为什么`lowerBound(nums, target + 1)` 在这种情况下会返回 n。：
> - 初始 `right = n`（数组长度）
> - 循环结束时，`right` 就是第一个 `≥target` 的元素下标
> - 如果全部元素都 `< target+1`，`right` 会一直保持在 `n`，不会向左移动
> 
> 所以返回 `n` 表示“答案在数组末尾之外”，即不存在符合条件的元素。

> [!Question] 为什么要写 `left + (right - left) / 2`？
> 
> 在面试或者实际场景中，你不一定知道输入的数组有多长，万一数组长度达到 int 最大值，`left + right` 可能会发生加法溢出。当然，如果只看本题的数据范围，写 `(left + right) / 2` 也可以。对于 Python 来说，由于没有溢出这个概念，所以可以直接相加。

> [!Question] 怎么判断我写的是哪一种二分？
> 
> 看 while 循环的条件：
> - `left <= right` → 闭区间
> - `left < right` → 半闭半开区间
> - `left + 1 < right` → 开区间

> [!Question] 对于闭区间写法，为什么 `nums[mid] >= target` 的时候要写 `right = mid - 1`？此时的 mid 不是有可能是答案吗？这样写不会错过答案吗？
> 
> 答案在区间外面，不在区间里面。如果觉得答案在区间里面的话，请思考这个问题：闭区间循环结束后 `left > right`，区间 `[left, right]` 是空的，什么也没有，难道答案在空区间里面吗？

> [!Question] 我看到一种二分的写法，在二分的过程中，额外记录答案的值，你对此怎么看？
> 
> 喜欢这种写法的同学，推荐写开区间二分，因为开区间二分 if 条件成立时更新的是哪个变量，最后返回的也就是哪个变量。这和记录答案的做法如出一辙。

> [!Question] 关于开区间二分，如何理解 −1 和 n 这两个下标？
> 
> 可以假设 `nums[-1] = -∞` 以及 `nums[n] = ∞`，此时 nums 仍然是有序的。在这种情况下，`nums[-1] < target`，所以 −1 是红色；`nums[n] >= target`，所以 n 是蓝色。这个思想可以推广到二分答案，见本文末尾的二分题单。

### 总结

| 需求            | 写法                          | 如果不存在  |
| ------------- | --------------------------- | ------ |
| ≥x 的第一个元素的下标  | `lowerBound(nums, x)`       | 结果为 n  |
| >x 的第一个元素的下标  | `lowerBound(nums, x+1)`     | 结果为 n  |
| <x 的最后一个元素的下标 | `lowerBound(nums, x) − 1`   | 结果为 −1 |
| ≤x 的最后一个元素的下标 | `lowerBound(nums, x+1) − 1` | 结果为 −1 |
