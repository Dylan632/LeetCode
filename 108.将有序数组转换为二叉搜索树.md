## 题目
![[108.png]]
**提示：**

- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 按 **严格递增** 顺序排列
## 思路（必填）

1. 二叉搜索树的特性是：**左 < 根 < 右**。
2. 有序数组的特性也是：左边的数小，右边的数大。
3. **平衡**意味着：左右两边的节点数量要差不多，高度差不能超过 1。
4. **怎么才能最平衡？** 答案就是：**永远选最中间的那个数当根节点！**
5. 每次都取**中间**，这保证了分给左边和右边的数字个数最多只相差1个（这就叫二分）。
  
### 算法选择：
采用分治法 + 递归。升序数组天然对应 BST的中序遍历结果，选取数组中间元素作为根节点，可以保证左右两侧元素数量尽量均等，从而实现平衡。

### 实现步骤：
1. 取当前区间 `[left, right]` 的中间位置 `mid`
2. 用 `nums[mid]` 创建当前子树的根节点
3. 递归地用 `[left, mid-1]` 构建左子树
4. 递归地用 `[mid+1, right]` 构建右子树
5. 当 l`eft > right` 时区间为空，返回 `null`
## 关键点
> [!Tip] 中间元素的选取是平衡的关键。选中点后自然将数组分成大小接近的两半，递归深度为 O(log n)，保证了树的平衡性。当数组长度为偶数时，选偏左或偏右的中点都可以，对应不同的合法答案。

## 代码

```java
class Solution {  
    public TreeNode sortedArrayToBST(int[] nums) {  
        return buildBST(nums, 0, nums.length - 1);  
    }  
    private TreeNode buildBST(int[] nums, int left, int right) {  
        if (left > right) {  
            return null;  
        }  
        int mid = left + (right - left) / 2;  
        // 前序
        TreeNode root = new TreeNode(nums[mid]);  
        root.left = buildBST(nums, left, mid - 1);  
        root.right = buildBST(nums, mid + 1, right);  
        return root;  
    }  
}
```

## 知识点
以 `nums = [-10, -3, 0, 5, 9]` 为例：


读图方式：从最内层的 null 开始往回看，每个节点拿到左右子树的返回值后挂到自己身上，最终在根节点 0 处组装出完整的树。

### 递归构建左右子树（魔法发生的地方）
```Java
	root.left = buildBST(nums, left, mid - 1);  
	root.right = buildBST(nums, mid + 1, right);  
```

这是全篇最烧脑，但也最巧妙的地方：

**构建左子树 (`root.left`)**：既然 `mid` 位置的数字已经被提拔当老大了，那么 `mid` 左边的所有数字（范围从 `left` 到 `mid - 1`）都比老大小。所以，我们再次呼叫施工队 `buildBST`，把左边这半截数组丢给它，让它建好一棵树，然后**挂在老大的左手上**。
**构建右子树 (`root.right`)**：同理，`mid` 右边的数字（范围从 `mid + 1` 到 `right`）都比老大大。把右边这半截丢给施工队，建好之后**挂在老大的右手上**。

## 复杂度

- Time: O(n)
- Space: O(log n)
n 为数组长度，递归栈深度 log n（平衡树）
