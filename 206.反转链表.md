## 题目

![[206.png]]

## 思路

1. 首先定义一个 `cur` 指针，指向头结点，再定义一个 `pre` 指针，初始化为 `null`。然后就要开始反转了，首先要把 `cur.next` 节点用 `nxt` 指针保存一下，也就是保存一下这个节点。
2. 为什么要保存一下这个节点呢，因为接下来要改变 `cur.next` 的指向了，将 `cur.next` 指向 `pre`，此时已经反转了第一个节点了。
3. 将 `pre` 移动到 `cur`，把 `cur` 移动到 `nxt`。
4. 接下来，就是循环走如下代码逻辑了，继续移动 `pre` 和 `cur` 指针。
5. 最后，`cur` 指针已经指向了 `null`，循环结束，链表也反转完毕了。此时我们 `return pre` 就可以了，`pre` 指针就指向了新的头结点。

## 关键点

> [!TIP] 必须先用 `nxt` 保存 `cur.next`，否则断开后就找不到下一个节点了。

## 实现步骤

1. 初始化 `pre = null`（反转后尾节点指向 `null`），`cur = head`
2. 每一轮循环做四件事：
    - `nxt = cur.next` — 先保存下一个节点（断开前必须保存）
    - `cur.next = pre` — 核心：把箭头反过来
    - `pre = cur` — 前驱前移
    - `cur = nxt` — 当前前移
3. 循环结束时 `cur == null`，`pre` 指向原链表尾节点，即新头节点

## 代码

### 解法一：双指针迭代

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode pre = null;
        ListNode cur = head;
        while (cur != null) {
            ListNode nxt = cur.next;
            // 反转：核心操作！当前节点指向前驱
            cur.next = pre;
            // 两个指针都向前移一位
            pre = cur;
            cur = nxt;
        }
        return pre;
    }
}
```

### 解法二：递归

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        if (head == null || head.next == null) {
            return head;
        }
        ListNode last = reverseList(head.next);
        head.next.next = head;
        head.next = null;
        return last;
    }
}
```

## 知识点

用树形图重新演示递归过程（链表 `1 → 2 → 3 → null`）：

```text
reverseList(1)                                   链表: 1 → 2 → 3 → null
├── last = reverseList(2)                        递归深入
│   ├── last = reverseList(3)                    继续递归
│   │   └── head.next == null → return 3         base case! 3 就是新头节点
│   │
│   │   回到 head=2, last=3
│   │   此时链表: 1 → 2 → 3 → null
│   │
│   ├── head.next.next = head    (3.next = 2)    让 3 指回 2
│   │   此时链表: 1 → 2 ⇄ 3                     2和3互指，成环！
│   │
│   ├── head.next = null         (2.next = null) 断开 2→3，解除环
│   │   此时链表: 1 → 2 ← 3
│   │                ↓
│   │               null
│   │
│   └── return 3                                 ✅ 返回 3
│
│   回到 head=1, last=3
│   此时链表: 1 → 2 ← 3
│                ↓
│               null
│
├── head.next.next = head        (2.next = 1)    让 2 指回 1
│   此时链表: 1 ⇄ 2 ← 3                         1和2互指，成环！
│
├── head.next = null             (1.next = null) 断开 1→2，解除环
│   此时链表: null ← 1 ← 2 ← 3
│
└── return 3                                     ✅ 返回 3，反转完成！

最终: 3 → 2 → 1 → null
```

---

### 递归核心规律

每一层回溯都做完全相同的两件事：

```text
head.next.next = head    让后继指回我（产生环）
head.next = null         断开我指向后继的箭头（解除环）

反转前:  head  →  head.next       有去无回
反转后:  head  ←  head.next       箭头掉头
          ↓
         null                     我变成尾巴（暂时）
```

`last`（即 3）从 base case 开始被逐层原封不动地传回来，自始至终没变过——它就是新的头节点。

> [!NOTE] 为什么只返回一个节点 `3`，却能得到整条链表？
> 表面上只返回了一个节点（Node 3），但实际上它带回了整个链表。好比手里拿着一根绳子的**一头**递给你，虽然只给了你“绳头”，但其实整根绳子都被你牵走了。
>
> 在链表中，**“持有头节点” = “持有整个链表”**。所以虽然 `last` 只是一个指针，指向节点 `3`，但因为节点之间已经连好了，**返回 `3` 就等于返回了 `3 → 2 → 1` 这一整串数据。**

## 复杂度

|          | 双指针迭代 | 递归       |
| -------- | ---------- | ---------- |
| **Time** | O(n)       | O(n)       |
| **Space**| O(1)       | O(n)       |

- 迭代：只用了 `pre`、`cur`、`nxt` 三个指针，空间 O(1)
- 递归：递归调用栈深度为 n，空间 O(n)
