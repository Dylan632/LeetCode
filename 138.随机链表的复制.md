## 思路（必填）

### 算法选择

使用交错链表技巧（空间 O(1)）

- 优点：空间复杂度更优
- 原理：利用原节点的 next 指针临时存储新节点的位置

### 实现步骤

1. 复制节点：遍历原链表，在每个原节点后插入复制节点
   - 形成：原节点1 → 复制节点1 → 原节点2 → 复制节点2 → ...
2. 设置 random：遍历交错链表，利用 `cur.random.next` 找到对应的复制节点
   - 关键：`cur.next.random = cur.random.next`
3. 分离链表：遍历交错链表，将原节点和复制节点分离成两个独立链表
   - 同时恢复原链表结构

## 关键点

> [!Tip]
> - 交错链表技巧：巧妙利用 next 指针建立原节点与新节点的映射关系
> - random 指针映射：`cur.random.next` 就是 `cur.random` 对应的复制节点
> - 恢复原链表：分离时需要恢复原链表，避免破坏原数据结构

## 代码

```java
class Solution {
    public Node copyRandomList(Node head) {
        if (head == null) return null;
        Node cur = head;
        // 复制每个节点，把新节点直接插到原节点的后面
        while (cur != null) {
            Node tmp = new Node(cur.val);
            tmp.next = cur.next;
            cur.next = tmp;
            cur = tmp.next;
        }
        // 遍历交错链表中的原链表节点
        for (cur = head; cur != null; cur = cur.next.next) {
            if (cur.random != null) {
                // 要复制的 random 是 cur.random 的下一个节点
                cur.next.random = cur.random.next;
            }
        }
        // 把交错链表分离成两个链表
        Node dummy = new Node(0);
        Node tail = dummy;
        for (cur = head; cur != null; cur = cur.next, tail = tail.next) {
            // 新节点
            Node copy = cur.next;
            // 把新节点插在 tail 的后面，构建新的链表
            tail.next = copy;
            // 恢复原链表
            cur.next = copy.next;
        }
        return dummy.next;
    }
}
```
