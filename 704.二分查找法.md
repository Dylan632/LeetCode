## 思路（必填）

我们定义 target 是在一个在<font color="#ff0000">左闭右闭</font>的区间里，**也就是`[left, right]` （这个很重要、非常重要）**。
区间的定义这就决定了二分法的代码应该如何写，**因为定义target在`[left, right]`区间，所以有如下两点：**
- while (left <= right) 要使用 <= ，因为<font color="#ff0000">left == right是有意义的</font>，所以使用 <=
- `if (nums[middle] > target)` right 要赋值为 middle - 1，因为<font color="#ff0000">当前这个nums[middle]一定不是target</font>，那么接下来要查找的左区间结束下标位置就是 middle - 1

## 关键点
二分查找的核心痛点在于**循环边界（`<=` 还是 `<`）** 以及**区间缩减（`middle` 还是 `middle - 1`）**。 因为我们使用了 `left = 0, right = nums.length - 1`（两端都闭的区间），所以要确保即便只有一个元素时（`left == right`）代码依然能够判断它，故必须用 `while (left <= right)`。同理，比较过 `middle` 之后，新的区间就不应再包含 `middle` 了，故必须 `+1` 或 `-1`。

## 代码

```java
class Solution {
    public int search(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        
        // 修正3：如果你想做边界检查，应该比 target，而不是 middle
        if (target < nums[0] || target > nums[nums.length - 1]) {
            return -1;
        }
        while (left <= right) {
            // 修正2：必须在循环里更新 middle！
            int middle = left + ((right - left) >> 1);
            
            if (nums[middle] == target) {
                return middle;
            } else if (nums[middle] > target) {
                right = middle - 1;
            } else {
                left = middle + 1;
            }
        } // while 循环在这里结束
        // 修正1：return -1 必须放在循环外面，作为保底
        return -1;
    }
}
```
