## 思路（必填）
### 问题分析
题目要求在一个 `m x n` 的网格中寻找从左上角走到右下角的独特路径数量。约束条件是每次只能“向下”或“向右”移动。这意味着到达网格中任意坐标 `(i, j)` 的路径总数，必然等于到达它上方点 `(i-1, j)` 的路径数加上到达它左方点 `(i, j-1)` 的路径数。
### 算法选择
采用**动态规划 (Dynamic Programming)**。常规解法会开辟一个 `m x n` 的二维数组，但由于我们在推导当前行的状态时，其实只依赖“上一行”和“当前行左侧”的数据，因此利用**状态压缩技巧**，用一个一维数组就能在迭代中反复刷新覆盖，以此节省空间。
### 实现步骤
1. **初始化**：创建一个大小为 `n + 1` 的一维数组 `dp`。为了启动第一步的状态，令 `dp[1] = 1`（相当于人为制造了一个起点的虚拟左侧，供起点吸收）。
2. **状态转移**：外层循环遍历行，内层循环遍历列。每次运算 `dp[j + 1] += dp[j]` 时：
    - 赋值前的 `dp[j + 1]` 中存的是上一行（上方）传递下来的值。
    - `dp[j]` 中存的是刚计算出的当前行前一个元素（左侧）传递过来的值。
    - 两者相加覆盖原位置，完美完成了 $dp[i][j] = dp[i-1][j] + dp[i][j-1]$ 的逻辑。
3. **返回结果**：当双重循环结束，数组最后一位 `dp[n]` 就累积了到达网格终点的所有路径总数。

## 关键点
>[!Tip] 
>最惊艳的地方在于将二维状态转移方程 `dp[i][j] = dp[i-1][j] + dp[i][j-1]` 完美折叠成了 `dp[j] += dp[j - 1]`，极大降低了空间复杂度。

## 代码

```java
class Solution {  
    public int uniquePaths(int m, int n) {  
        int[] dp = new int[n + 1];  
        dp[1] = 1;  
        for (int i = 0; i < m; i++) {  
            for (int j = 0; j < n; j++) {  
                // 状态转移方程（压缩前：dp[i][j] = dp[i-1][j] + dp[i][j-1]）  
                // 等号右侧的 dp[j + 1] 代表上方格子传下来的路径数  
                // 等号右侧的 dp[j] 代表左侧格子传过来的路径数  
                dp[j + 1] += dp[j];  
            }  
        }  
        // 遍历结束后，数组最后一位即为到达右下角的总路径数  
        return dp[n];  
    }  
}
```

## 知识点


## 复杂度

- Time: O(mn)
- Space: O(n)
需要双层循环遍历网格 `m * n` 次；只使用了一个长度为 `n` 的一维数组记录状态。
