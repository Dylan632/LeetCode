## 思路（必填）
### 问题分析
题目要求计算所有新鲜橘子变腐烂的最短时间。每次一分钟，腐烂的橘子会向上下左右四个方向传染。如果最后还有橘子无法腐烂，返回 -1。
### 算法选择
这是一个典型的**多源广度优先搜索 (Multi-source BFS)** 问题。因为可能有多个橘子同时一开始就是腐烂的，它们会像水波纹一样同时向外扩散。
### 实现步骤
1. **初始化**：遍历整个二维网格，统计新鲜橘子的总数 `fresh`，并将所有初始腐烂橘子的坐标加入集合 `q` 作为 BFS 的第一层"波源"。
2. **层序扩散**：当还有新鲜橘子且队列不为空时，开始按分钟（层数）循环。每次循环代表过去了一分钟 (`ans++`)。
3. **状态更新**：遍历当前层 `tmp` 里的所有腐烂橘子，向四个方向探索。遇到新鲜橘子，就将其变为腐烂状态，放入新的队列 `q` 中，并让 `fresh` 减一。
4. **结果判定**：BFS 结束后，检查 `fresh` 是否大于 0。如果大于 0，说明存在无法被感染的死角，返回 -1；否则返回记录的分钟数 `ans`。

## 关键点
>[!Tip] 
>在 `while` 循环条件中同时判断 `fresh > 0`。这样在最后一分钟最后一个橘子被感染后，循环会立刻结束，不会再空转一圈去检查有没有新邻居，这是保证 `ans` 不需要再减一的关键巧思。

## 代码

```java
class Solution {  
    private static final int[][] DIRECTIONS = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};  
  
    public int orangesRotting(int[][] grid) {  
        int m = grid.length;  
        int n = grid[0].length;  
        int fresh = 0;  
        List<int[]> q = new ArrayList<>();  
        for (int i = 0; i < m; i++) {  
            for (int j = 0; j < n; j++) {  
                if (grid[i][j] == 1) {  
                    fresh++;  
                } else if (grid[i][j] == 2) {  
                    q.add(new int[]{i, j});  
                }  
            }  
        }  
        int ans = 0;  
        while (fresh > 0 && !q.isEmpty()) {  
            ans++;  
            List<int[]> tmp = q;  
            q = new ArrayList<>();  
            for (int[] pos : tmp) {  
                for (int[] d : DIRECTIONS) {  
                    int i = pos[0] + d[0];  
                    int j = pos[1] + d[1];  
                    if (0 <= i && i < m && 0 <= j && j < n && grid[i][j] == 1) {  
                        fresh--;  
                        grid[i][j] = 2;  
                        q.add(new int[]{i, j});  
                    }  
                }  
            }  
        }  
        return fresh > 0 ? -1 : ans;  
    }  
}
```

## 知识点
### 方向
```java
private static final int[][] DIRECTIONS = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
```
当前站在网格的 `(i, j)` 位置上，代码中的 `DIRECTIONS = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}}` 分别对应：

- **`{-1, 0}`**：行号 `i - 1`，列号 `j` 不变  **向上走**
- **`{1, 0}`**：行号 `i + 1`，列号 `j` 不变  **向下走**
- **`{0, -1}`**：行号 `i` 不变，列号 `j - 1`  **向左走**
- **`{0, 1}`**：行号 `i` 不变，列号 `j + 1`  **向右走**
```
         上 (i-1, j)
               ↑
左 (i, j-1) ← (i, j) → 右 (i, j+1)
               ↓
           下 (i+1, j)
```

### 四周腐烂
```java
for (int[] d : DIRECTIONS) {  
    int i = pos[0] + d[0];  
    int j = pos[1] + d[1];  
}                
```
这个 `for` 循环（增强 for 循环）的作用就是：**把 `DIRECTIONS` 里面的四个方向挨个拿出来，每次拿出一个，就执行一次大括号里面的加法。**

### 橘子腐烂过程
![[994_1.png]]
1. 统计所有初始就腐烂的橘子的位置，加到列表 q 中，现在 `q=[(0,0)]`。
2. 初始化答案 ans=0。模拟橘子腐烂的过程，不断循环，直到没有新鲜橘子，或者 q 为空。
3. 答案加一，在第 ans=1 分钟，遍历 q 中橘子的四方向相邻的新鲜橘子，把这些橘子腐烂，q 更新为这些橘子的位置，现在 `q=[(0,1),(1,0)]`。
4. 答案加一，在第 ans=2 分钟，遍历 q 中橘子的四方向相邻的新鲜橘子，把这些橘子腐烂，q 更新为这些橘子的位置，现在 `q=[(0,2),(1,1)]`。
5. 答案加一，在第 ans=3 分钟，遍历 q 中橘子的四方向相邻的新鲜橘子，把这些橘子腐烂，q 更新为这些橘子的位置，现在 `q=[(2,1)]`。
6. 答案加一，在第 ans=4 分钟，遍历 q 中橘子的四方向相邻的新鲜橘子，把这些橘子腐烂，q 更新为这些橘子的位置，现在 `q=[(2,2)]`。
7. 由于没有新鲜橘子，退出循环。

为了判断是否有永远不会腐烂的橘子（如示例 2），我们可以统计初始新鲜橘子的个数 fresh。在 BFS 中，每有一个新鲜橘子被腐烂，就把 fresh 减一，这样最后如果发现 fresh>0，就意味着有橘子永远不会腐烂，返回 −1。

代码实现时，在 BFS 中要将 `grid[i][j]=1` 的橘子修改成 2（或者其它不等于 1 的数），这可以保证每个橘子加入 q 中至多一次。如果不修改，我们就无法知道哪些橘子被腐烂过了，比如示例 1 中 (0,1) 去腐烂 (1,1)，而 (1,1) 在此之后又重新腐烂 (0,1)，如此反复，程序就会陷入死循环。

### 答疑

> [!Question] 如果代码不在 `while` 中判断 fresh>0，会发生什么？
> 
> 会在腐烂完所有新鲜橘子后，多循环一次。这会导致 ans 比实际多 1。（新的腐烂的橘子还需要再向四周探索，但四周已经没有好的橘子了，所以它只是在"空转"，发现无法腐烂其他的新的橘子，才结束循环）

## 复杂度

- Time: O(mn)
- Space: O(mn)
其中 m 和 n 分别为 grid 的行数和列数。
