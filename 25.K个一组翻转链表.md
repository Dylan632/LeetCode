## 思路（必填）
### 算法选择
迭代式翻转。先统计链表长度以确定需要翻转的组数，然后逐组用经典的“三指针翻转”手法处理。
### 实现步骤
1. 统计长度：遍历链表得到 len，后续用 len >= k 判断剩余节点是否够一组
2. 创建哨兵：dummy 指向 head，groupPrev 初始指向 dummy，追踪每组的前驱节点
3. 组内翻转：经典三指针（prev/cur/temp），循环 k 次后 prev 指向翻转后的组头，cur 指向下一组的头
4. 重新连接（核心四步，顺序不能乱）：
    - ListNode groupTail = groupPrev.next — 保存翻转前的组头（现在是组尾）
    - groupPrev.next.next = cur — 组尾的 next 接上下一组头节点
    - groupPrev.next = prev — 前驱的 next 接上翻转后的新组头
    - groupPrev = groupTail — 前驱指针移动到当前组尾，为下一组做准备

## 关键点
>[!Tip] groupPrev.next.next = cur 必须在 groupPrev.next = prev 之前执行，因为会覆盖groupPrev.next，之后就无法再通过 groupPrev.next 找到组尾了，groupTail 的保存也必须在修改指针之前，否则引用丢失

## 代码

```java
class Solution {  
    public ListNode reverseKGroup(ListNode head, int k) {  
        if (k <= 1 || head == null) return head;  
        // 统计节点长度  
        int len = 0;  
        for (ListNode cur = head; cur != null; cur = cur.next) {  
            len++;  
        }  
  
        ListNode dummy = new ListNode(0, head);  
        ListNode groupPrev = dummy; // 当前待翻转组的前驱节点  
        ListNode prev = null;  
        ListNode cur = head;  
  
        // k个一组处理翻转  
        for (; len >= k; len -= k) {  
            prev = null; // 每组翻转前重置  
            for (int i = 0; i < k; i++) {  
                ListNode temp = cur.next;  
                cur.next = prev;  
                prev = cur;  
                cur = temp;  
            }  
            // 将翻转后的组接回链表  
            ListNode groupTail = groupPrev.next; // 翻转前的组头，现在是组尾  
            groupPrev.next.next = cur;           // 组尾接上下一组的头  
            groupPrev.next = prev;               // 前驱接上翻转后的组头  
            groupPrev = groupTail;               // 更新前驱为当前组尾  
        }  
        return dummy.next;  
    }  
}
```

## 复杂度

- Time: O(n)——两次遍历（一次统计长度，一次翻转）
- Space: O(1)——仅用常量指针变量
