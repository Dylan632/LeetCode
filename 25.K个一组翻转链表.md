## 思路（必填）
### 算法选择
迭代式翻转。先统计链表长度以确定需要翻转的组数，然后逐组用经典的“三指针翻转”手法处理。
### 实现步骤
1. 统计长度：遍历链表得到 n，后续用 n >= k 判断剩余节点是否够一组
2. 创建哨兵：dummy 指向 head，p0 初始指向 dummy，追踪每组的前驱节点
3. 组内翻转：经典三指针（pre/cur/nxt），循环 k 次后 pre 指向翻转后的组头，cur 指向下一组的头
4. 重新连接（核心四步，顺序不能乱）：
    - ListNode nxt = p0.next — 保存翻转前的组头（现在是组尾）
    - p0.next.next = cur — 组尾的 next 接上下一组头节点
    - p0.next = pre — 前驱的 next 接上翻转后的新组头
    - p0 = nxt — 前驱指针移动到当前组尾，为下一组做准备

## 关键点
>[!Tip] 重新连接的顺序至关重要
> - `p0.next.next = cur` 必须在 `p0.next = pre` 之前执行
> - 因为 `p0.next = pre` 会覆盖 `p0.next`
> - 覆盖后就无法再通过 `p0.next` 找到组尾了
> - `nxt` 的保存也必须在修改指针之前，否则引用丢失

## 代码

```java
class Solution {  
    public ListNode reverseKGroup(ListNode head, int k) {  
        // 统计节点个数  
        int n = 0;  
        for (ListNode cur = head; cur != null; cur = cur.next) {  
            n++;  
        }  
  
        ListNode dummy = new ListNode(0, head);  
        ListNode p0 = dummy;  
        ListNode pre = null;  
        ListNode cur = head;  
  
        // k 个一组处理  
        for (; n >= k; n -= k) {  
            for (int i = 0; i < k; i++) { // 同 92 题  
                ListNode nxt = cur.next;  
                cur.next = pre; // 每次循环只修改一个 next                
                pre = cur;  
                cur = nxt;  
            }  
            // 将翻转后的组接回链表  
            ListNode nxt = p0.next;  // 翻转前的组头，现在是组尾
            p0.next.next = cur;      // 组尾接上下一组的头 
            p0.next = pre;           // 前驱接上翻转后的组头
            p0 = nxt;                // 更新前驱为当前组尾
        }  
        return dummy.next;  
    }  
}
```

## 复杂度

- Time: O(n)——两次遍历（一次统计长度，一次翻转）
- Space: O(1)——仅用常量指针变量
