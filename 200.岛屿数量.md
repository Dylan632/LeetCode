## 思路（必填）
### 问题分析
  - 给定一个由 “1”（陆地）和 “0”（水域）组成的二维网格，计算岛屿的数量
  - 岛屿定义：被水域包围的陆地，水平或垂直方向相连的陆地算作同一个岛屿
### 算法选择
  - DFS（深度优先搜索）：适合连通性问题，可以从一个陆地格子出发，探索所有相连的陆地
  - 原地标记：通过修改原数组标记已访问节点，无需额外空间（除递归栈外）
### 实现步骤
  1. 遍历网格中的每个格子
  2. 当遇到 “1” 时，说明找到了一个新的岛屿
  3. 调用 DFS 从该格子出发，将所有相连的陆地标记为已访问（改为 “2”）
  4. 岛屿计数器 ans++
  5. 继续遍历，后续遇到的 “1” 一定是新的岛屿（因为旧的岛屿已被标记）

## 关键点
> [!Tip]
> - 插旗标记：将访问过的陆地改为 “2”，避免重复访问和无限递归
> - 边界检查：在 DFS 递归前检查是否越界或非陆地
> - 四方向搜索：上下左右四个方向搜索相连的陆地

## 代码

```java
class Solution {  
    public int numIslands(char[][] grid) {  
        int ans = 0;  
        for (int i = 0; i < grid.length; i++) {  
            for (int j = 0; j < grid[0].length; j++) {  
                if (grid[i][j] == '1') { // 找到了一个新的岛  
                    dfs(grid, i, j); // 把这个岛插满旗子，这样后面遍历到的 '1' 一定是新的岛  
                    ans++;  
                }  
            }  
        }  
        return ans;  
    }  
  
    private void dfs(char[][] grid, int i, int j) {  
        // 出界，或者不是 '1'，就不再往下递归  
        if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length || grid[i][j] != '1') {  
            return;  
        }  
        grid[i][j] = '2'; // 插旗！避免来回横跳无限递归  
        dfs(grid, i, j - 1); // 往左走  
        dfs(grid, i, j + 1); // 往右走  
        dfs(grid, i - 1, j); // 往上走  
        dfs(grid, i + 1, j); // 往下走  
    }  
}
```

## 知识点

> [!Question] 为什么不会重复计数？
> 
> 因为一旦发现一个岛，就立刻用 DFS 把整座岛全部标记成 “2”，之后再遍历到这些格子时，就不会再把它当成新岛。

> [!Question] 二叉树的递归和网格图的递归有何区别？
> 
> 列表总结如下。
> 
> |        | 二叉树         | 网格图          |
> | ------ | ----------- | ------------ |
> | 递归入口   | 根节点         | 岛屿第一行最左边的陆地  |
> | 递归方向   | 左儿子和右儿子     | 左右上下的相邻陆地    |
> | 递归边界   | 空节点（或者叶节点）  | 出界、遇到水或者旗子   |

> [!Question] 如何理解递归？
> 
> 递归的思想是，假设你是一家公司的老板，你不需要万事亲力亲为，而是拆解问题，交给下属去做。对于这题来说，就是从岛屿的某个位置登陆插旗，然后一分为四，把探索整个岛屿的任务交给其他人去处理，自己只需处理好第一步就行。

> [!Question] 我可以先判断 `grid[i][j]` 是否等于 1，再判断 (i,j) 是否出界吗？
> 
> 不行，比如 i = -1，获取 `grid[i][j]` 就下标越界了。要先保证下标在范围中，再去获取数组元素值。

> [!Question] 我可以把访问过的 `grid[i][j]` 改成 0 吗？
> 
> 可以。相当于把访问过的位置变成水。其实，改成除了 1 以外的任何值都行。

> [!Question] 我可以把这行代码 `grid[i][j] = '2'` 写在 dfs 的最后一行吗？
> 
> 不行。这样写会先执行递归，比如从左上角 (0,0) 向右移动到 (0,1)，内部继续递归，从 (0,1) 又向左移动到 (0,0)，导致反复横跳，无限递归。

> [!Question] DFS 中能否只考虑往右走和往下走？
> 
> 不行，比如蚊香形的岛屿，在探索的时候必须具备左右上下走的能力。


## 复杂度

- Time: O(m × n)，每个格子最多访问一次
- Space: O(m × n)，最坏情况下递归深度为整个网格大小
