## 思路（必填）
### 问题分析
常规的二分查找要求数组完全有序。而在**旋转排序数组**中，数组被从中间某个点劈开并互换了位置（例如 `[0,1,2,4,5,6,7]` 变成 `[4,5,6,7,0,1,2]`）。

**解题破局点：** 将数组从中间 `mid` 劈开后，**必定有一半是完全有序的**，另一半是包含旋转点的无序区间。我们只需要：

1. 找出哪一半是有序的
2. 判断目标值 `target` 是否落在这个“有序区间”内
3. 如果在，就缩小范围到这个有序区间；如果不在，就去另一半找


### 算法选择
- 使用改进的二分查找，因为原数组有序，旋转后仍保持局部有序性
- 每次可以将搜索范围缩小一半，满足 O(log n) 要求
### 实现步骤

1. 边界检查：处理空数组和 null 情况
2. 二分查找框架：计算 mid，判断是否找到目标
3. 判断有序区间：旋转数组必有一半是有序的
   - 如果 `nums[left] <= nums[mid]`，则左半部分有序
   - 否则右半部分有序
4. 缩小搜索范围：
   - 在有序区间内：判断目标是否在该区间，决定搜索方向
   - 不在有序区间内：搜索另一半
5. 返回结果：找到返回索引，否则返回 -1

## 关键点

> [!Tip] 核心要点
> - 旋转数组的重要性质：每次二分后，至少有一半是有序的
> - 通过判断有序区间，确定目标值可能存在的位置
> - 正确处理边界条件（`<=` 和 `<` 的使用）

## 代码

```java
class Solution {
    public int search(int[] nums, int target) {
        if (nums == null || nums.length == 0) {
            return -1;
        }
        int left = 0;
        int right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                return mid;
            }
            if (nums[left] <= nums[mid]) {
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            } else {
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
}
```

## 知识点
**记忆口诀总结：** 劈两半 -> 找有序的那一半 -> 判断 `target` 在不在有序的这一半 -> 在就留下，不在就去另一半。

```java
    if (nums[left] <= target && target < nums[mid])
else {  // 右边有序
    if (nums[mid] < target && target <= nums[right])
```

### 判断“哪边有序”时为什么是 `<=` ？
```java
if (nums[left] <= nums[mid])
```
这里必须带等号，原因是：当区间很小时，可能出现 `left == mid`。比如只剩两个数：`[left, right]`，此时 `mid` 可能就等于 `left`。如果你写成：`nums[left] < nums[mid]`，那会因为两者相等而判错。
所以这里要记：
**判断哪边有序时，为了兼容 left == mid / mid == right，要用 <=。**

## 复杂度

- 时间复杂度：O(log n),因为每次循环都会排除一半的元素
- 空间复杂度：O(1),只使用了几个指针变量
