## 题目
![[33.png]]
**提示：**

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 中的每个值都 **独一无二**
- 题目数据保证 `nums` 在预先未知的某个下标上进行了旋转
- `-10^4 <= target <= 10^4`

## 思路（必填）
### 问题分析
常规的二分查找要求数组完全有序。而在**旋转排序数组**中，数组被从中间某个点劈开并互换了位置（例如 `[0,1,2,4,5,6,7]` 变成 `[4,5,6,7,0,1,2]`）。

**解题破局点：** 将数组从中间 `mid` 劈开后，**必定有一半是完全有序的**，另一半是包含旋转点的无序区间。我们只需要：
1. 找出哪一半是有序的
2. 判断目标值 `target` 是否落在这个“有序区间”内
3. 如果在，就缩小范围到这个有序区间；如果不在，就去另一半找
### 算法选择
- 使用改进的二分查找，因为原数组有序，旋转后仍保持局部有序性
- 每次可以将搜索范围缩小一半，满足 O(log n) 要求
### 实现步骤

1. 边界检查：处理空数组和 null 情况
2. 二分查找框架：计算 mid，判断是否找到目标
3. 判断有序区间：旋转数组必有一半是有序的
   - 如果 `nums[left] <= nums[mid]`，则左半部分有序
   - 否则右半部分有序
4. 缩小搜索范围：
   - 在有序区间内：判断目标是否在该区间，决定搜索方向
   - 不在有序区间内：搜索另一半
5. 返回结果：找到返回索引，否则返回 -1

## 关键点

> [!Tip] 核心要点
> - 旋转数组的重要性质：每次二分后，至少有一半是有序的
> - 通过判断有序区间，确定目标值可能存在的位置
> - 正确处理边界条件（`<=` 和 `<` 的使用）

## 两次二分
```java
class Solution {  
    public int search(int[] nums, int target) {  
        int n = nums.length;  
        int i = findMin(nums);  
        if (target > nums[n - 1]) { // target 只可能在第一段  
            return lowerBound(nums, -1, i, target); // 开区间 (-1, i)
        }  
        // target 只可能在第二段  
        // 由于此时 target <= nums[n-1]，所以 lowerBound 中的循环结束后，right < n 一定成立，无需判断 right == n        
        return lowerBound(nums, i - 1, n, target); // 开区间 (i-1, n)    
    }  
  
    // 153. 寻找旋转排序数组中的最小值（返回的是下标）  
    private int findMin(int[] nums) {  
        int n = nums.length;  
        int left = -1;  
        int right = n - 1; // 开区间 (-1, n-1)        
        while (left + 1 < right) { // 开区间不为空  
            int mid = left + (right - left) / 2;  
            if (nums[mid] < nums[n - 1]) {  
                right = mid;  
            } else {  
                left = mid;  
            }  
        }  
        return right;  
    }  
  
    // 有序数组中找 target 的下标  
    private int lowerBound(int[] nums, int left, int right, int target) {  
        while (left + 1 < right) { // 开区间不为空  
            // 循环不变量：  
            // nums[right] >= target  
            // nums[left] < target            
            int mid = left + (right - left) / 2;   
            if (nums[mid] >= target) {  
                right = mid; // 范围缩小到 (left, mid)            
            } else {  
                left = mid; // 范围缩小到 (mid, right)        
            }  
        }  
        return nums[right] == target ? right : -1;  
    }  
}
```
### 问题分析：
  - 题目要求在旋转排序数组中查找目标值
  - 数组原本升序排列，在某一点旋转，形成两个有序段 
  - 要求时间复杂度 O(log n)

### 算法选择：
  - <font color="#de7802">选择两次二分查找：第一次找旋转点（min的索引i），第二次找目标值</font>
  - 为什么不用线性扫描？因为线性扫描是 O(n)，不满足题目要求
  - 为什么不用一次二分？虽然可以，但两次二分思路更清晰，且都是 O(log n)

### 实现步骤：
  1. 找旋转点（最小值索引）：通过比较 `nums[mid]` 和 `nums[n-1]` 判断 `mid` 在哪一段
  2. 确定搜索区间：根据 `target` 和 `nums[n-1]` 的关系，判断 `target` 在哪一段
  3. 二分查找：在确定的区间内进行标准二分查找

### 关键点：
  - 开区间设计：使用开区间 `(left, right)` 避免边界判断，循环条件统一为 `left + 1 < right`
  - 旋转点判断：通过与最后一个元素比较，判断 `mid` 在第一段还是第二段
  - 区间选择：`target > nums[n-1]` 说明在第一段，否则在第二段
  
### 知识点详解：

  1. 开区间二分查找

  - 定义：搜索区间为 `(left, right)`，不包含左右边界
  - 优势：
    - 循环条件统一：`left + 1 < right`
    - 避免边界判断：不需要考虑 `left <= right` 还是 `left < right`
    - 终止条件明确：循环结束时 `left + 1 == right`，区间内只有一个元素
  - 适用场景：所有二分查找问题，特别是需要找边界的情况

  2. 旋转数组的性质

  - 旋转后形成两段有序数组
  - 第一段所有元素 > 第二段所有元素
  - 旋转点就是最小值的索引

  3. 循环不变量

  - 在循环过程中始终保持成立的性质
  - 本题中的不变量：`nums[right] >= target` 且 `nums[left] < target`
  - 用途：证明算法正确性，帮助理解代码逻辑
### 📝 笔记：为什么找 Target 时必须用 `>` 而不能用 `>=`？

**判断条件：`if (target > nums[n - 1])` 去左区间，否则去右区间。**

- **核心定位**：`nums[n-1]` 是数组的末尾元素，它**永远属于右区间**。
- **左区间特征**：由于数组被旋转，左区间的所有元素都**严格大于**右区间，因此也**严格大于** `nums[n-1]`。
- **用 `>=` 的致命错误**：如果写成 `target >= nums[n - 1]`，当要找的目标刚好是最后一个元素（即 `target == nums[n-1]`）时，代码会误判它在**左区间**。
- **结果**：去左区间找一个右区间的数，永远找不到，错误地返回 `-1`。

**一句话总结**：等于 `nums[n-1]` 的值只可能在右区间，必须用严格大于（`>`）把 == 的情况逼退到右区间的逻辑中去。

### 灵茶讲解
首先 [[153.寻找旋转排序数组中的最小值]]，找到 `nums` 的最小值的下标 `i`。

根据旋转排序数组的定义，下标在 `[0, i-1]` 中的元素都比下标在 `[i, n-1]` 中的元素大（注意题目保证 `nums` 没有重复元素）。特别地，如果 `i=0`，那么 `nums` 就是严格递增数组，没有发生旋转。

根据这一性质，分类讨论：

如果 `target > nums[n-1]`，那么 `target` 只可能在子数组 `[0, i-1]` 中。由于子数组 `[0, i-1]` 是递增的，我们可以在 `[0, i-1]` 中二分查找 `target`，<font color="#de7802">也就是开区间(-1, i)中</font>。
如果 `target <= nums[n-1]`，那么 `target` 只可能在子数组 `[i, n-1]` 中。由于子数组 `[i, n-1]` 是递增的，我们可以在 `[i, n-1]` 中二分查找 `target`，<font color="#de7802">也就是开区间(i - 1, n)中</font>。
注意上述讨论兼容 `i=0` 的情况：

如果 `target > nums[n-1]`，由于 `nums` 是递增的，所以 `target` 比 `nums` 中的每个数都要大，所以 `nums` 不存在 `target`。顺带一提，此时代码二分查找中的 `nums[right] == target` 相当于 `nums[0] == target`，这一定是 false。
如果 `target <= nums[n-1]`，那么在 `[i, n-1]` 中二分也就是在 `[0, n-1]` 中二分。


## 代码

```java
class Solution {
    public int search(int[] nums, int target) {
        if (nums == null || nums.length == 0) {
            return -1;
        }
        int left = 0;
        int right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                return mid;
            }
            if (nums[left] <= nums[mid]) {
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            } else {
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
}
```

**记忆口诀总结：** 劈两半 -> 找有序的那一半 -> 判断 `target` 在不在有序的这一半 -> 在就留下，不在就去另一半。

### 判断“哪边有序”时为什么是 `<=` ？
```java
if (nums[left] <= nums[mid])
```
这里必须带等号，原因是：当区间很小时，可能出现 `left == mid`。比如只剩两个数：`[left, right]`，此时 `mid` 可能就等于 `left`。如果你写成：`nums[left] < nums[mid]`，那会因为两者相等而判错。
所以这里要记：
**判断哪边有序时，为了兼容 `left == mid` / `mid == right`，要用 `<=`。**

## 复杂度

- 时间复杂度：O(log n),因为每次循环都会排除一半的元素
- 空间复杂度：O(1),只使用了几个指针变量
