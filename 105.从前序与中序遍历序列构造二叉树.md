## 题目
![[105.png]]
**提示:**

- `1 <= preorder.length <= 3000`
- `inorder.length == preorder.length`
- `-3000 <= preorder[i], inorder[i] <= 3000`
- `preorder` 和 `inorder` 均 **无重复** 元素
- `inorder` 均出现在 `preorder`
- `preorder` **保证** 为二叉树的前序遍历序列
- `inorder` **保证** 为二叉树的中序遍历序列

## 思路(必填)
### 问题分析
题目提供二叉树的前序遍历(Preorder)和中序遍历(Inorder)结果，要求我们还原这棵二叉树。已知树中没有重复元素
### 算法选择
树的还原问题天然适合使用**递归(分治法)**。因为任何一棵树及其子树都可以被分解为 “根节点” + “左子树” + “右子树”。为了快速定位根节点在中序遍历中的位置，使用 **HashMap** 以空间换时间，避免在数组中线性查找，是该题性能优化的核心。
### 实现步骤
1. 建立索引表：遍历中序数组 `inorder`,将元素值与其索引存入 `HashMap`。
2. 确立根节点：在 `dfs` 方法中,前序遍历区间的第一个元素 `preorder[preL]` 就是当前子树的根。
3. 划分左右子树：从 `HashMap` 中查出根节点在中序遍历中的位置 `index.get(preorder[preL])`，该位置减去中序起始位置 `inL` 即为左子树的节点个数 `leftSize`。
4. 递归切割：利用 `leftSize`，在 `preorder` 数组中严格划分出左子树和右子树的边界索引。向下递归生成左右孩子,最后将它们挂载到 `new TreeNode` 上返回。

## 关键点
>[!Tip] 
严格维护区间定义的语义。此代码选用的“左闭右开”即 `[start, end)` 技巧十分经典:数组长度为 `end - start`,划分时 `[start, start + length)` 极为清晰,逻辑严密。
## 代码

```java
class Solution {  
    public TreeNode buildTree(int[] preorder, int[] inorder) {  
        int n = preorder.length;  
        Map<Integer, Integer> index = new HashMap<>(n, 1);  
        for (int i = 0; i < n; i++) {  
            index.put(inorder[i], i);  
        }  
        // 左闭右开区间  
        return dfs(0, n, 0, preorder, index);  
    }  
  
    private TreeNode dfs(int preL, int preR, int inL, int[] preorder, Map<Integer, Integer> index) {  
        if (preL == preR) {  
            return null;  
        }  
        // 左子树大小  
        int leftSize = index.get(preorder[preL]) - inL;  
        TreeNode left = dfs(preL + 1, preL + 1 + leftSize, inL, preorder, index);  
        TreeNode right = dfs(preL + 1 + leftSize, preR, inL + 1 + leftSize, preorder, index);  
        return new TreeNode(preorder[preL], left, right);  
    }  
}
```

## 知识点
前序遍历性质：节点按照 `[ 根节点 | 左子树 | 右子树 ]` 排序。
中序遍历性质： 节点按照 `[ 左子树 | 根节点 | 右子树 ]` 排序。

以题目示例为例：

前序遍历划分 `[ 3 | 9 | 20 15 7 ]`
中序遍历划分 `[ 9 | 3 | 15 20 7 ]`
根据以上性质，可得出以下推论：

前序遍历的首元素 为 树的根节点 node 的值。
在中序遍历中搜索根节点 node 的索引 ，可将 中序遍历 划分为 `[ 左子树 | 根节点 | 右子树 ]` 。
根据中序遍历中的左(右)子树的节点数量，可将 前序遍历 划分为 `[ 根节点 | 左子树 | 右子树 ]` 。
![[105_1.png]]

通过以上三步，可确定 三个节点：1.树的根节点、2.左子树根节点、3.右子树根节点。

### 💡 看图关键点解析:

1. **`preL == preR` 的拦截**:仔细看叶子节点(如 9、15、7)向下调用时,由于 `preL + 1` 和 `preL + 1 + leftSize` 算出来的值刚好相等,触发了 `if (preL == preR) return null;`。这证明了**左闭右开区间**在处理边界时比较优雅。
2. **`leftSize` 的作用**：它是切割前序数组的“手术刀”。比如在节点 20 时,算出 `leftSize = 1`,说明前序数组 `[20, 15, 7]` 中,紧跟在 `20` 后面的 `1` 个元素(也就是 `15`)是归属左子树的,剩下的部分(`7`)统统归属右子树。

## 复杂度

- Time: O(n)
- Space: O(n)
$n$ 为树中的节点个数。整体时间主要消耗在哈希表构建和树的递归遍历上。空间主要消耗在哈希表和系统调用栈
