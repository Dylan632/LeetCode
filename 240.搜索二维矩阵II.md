## 题目
![[240.png]]
**提示：**

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= n, m <= 300`
- `-10^9 <= matrix[i][j] <= 10^9`
- 每行的所有元素从左到右升序排列
- 每列的所有元素从上到下升序排列
- `-10^9 <= target <= 10^9`
## 思路（必填）
### 问题分析
矩阵的特点是：每行的元素从左到右升序排列，每列的元素从上到下升序排列。我们需要在 O(m+n) 的时间复杂度内判断目标值是否存在。
### 算法选择
选择 **Z 字形查找（模拟二叉搜索树）**。不能单纯使用整体二分查找，因为行与行之间没有首尾相连的严格递增关系。
### 实现步骤
1. 将搜索起点定在矩阵的**右上角** `matrix[0][matrix[0].length - 1]`（即代码中的 `i = 0, j = matrix[0].length - 1`）。
2. 将当前位置的元素与 `target` 比较：
    - 如果等于 `target`，直接返回 `true`。
    - 如果当前元素**小于** `target`：因为当前元素已经是当前行中最大的了，所以 `target` 不可能在当前行，只能在下方，因此行指针下移（`i++`）。
    - 如果当前元素**大于** `target`：因为当前元素已经是当前列中最小的了，所以 `target` 不可能在当前列，只能在左侧，因此列指针左移（`j--`）。
3. 当指针越界（`i >= matrix.length` 或 `j < 0`）时结束循环，说明矩阵中不存在该目标值，返回 `false`。


## 关键点
>[!Tip] 
>解题的突破口在于**选对起点**。如果从左上角出发，向右和向下都是递增，遇到比目标值小的数时无法决定该往哪个方向走。而从右上角（或左下角）出发，可以将其视为一棵“二叉搜索树”的根节点（左边比它小，下边比它大），每次比较都能稳定排除一行或一列。

## 代码

```java
class Solution {  
    public boolean searchMatrix(int[][] matrix, int target) {  
        int i = 0;  
        int j = matrix[0].length - 1;  
        while (i < matrix.length && j >= 0) {  
            if (matrix[i][j] == target) {  
                return true;  
            }  
            if (matrix[i][j] < target) {  
                i++;  
            } else {  
                j--;  
            }  
        }  
        return false;  
    }  
}
```

## 复杂度

- Time: O( m + n )
- Space: O(1)
m和n分别为矩阵的行数和列数。每次循环必定排除一行或一列，最多循环 m + n 次。只使用了常数级别的额外指针空间
