## 思路（必填）
### 问题分析
  - 给定一个整数数组 nums 和滑动窗口大小 k
  - 窗口从数组最左侧滑动到最右侧，每次移动一位
  - 需要返回每个窗口中的最大值
### 算法选择
  选择单调队列（单调递减的双端队列），原因：
  - 暴力解法每个窗口遍历 k 个元素，时间复杂度 O(n×k)
  - 优先队列虽然可以快速获取最大值，但移除窗口外元素困难
  - 单调队列可以 O(1) 获取最大值，且每个元素最多入队出队各一次，总时间 O(n)
### 实现步骤
  1. 维护单调性：新元素入队前，移除队列中所有小于它的元素（它们永远不会成为最大值）
  2. 存储下标：队列存储元素下标而非值，便于判断元素是否离开窗口
  3. 移除过期元素：队首元素下标 < 窗口左端点时，说明已离开窗口，移除
  4. 记录答案：窗口形成后（left >= 0），队首即为当前窗口最大值

## 关键点
>[!Tip]
>1. 新元素进来时，把队尾所有比它小或等于它的元素删掉，因为它们以后不可能成为最大值。
>2. 窗口滑动时，如果队首下标已经不在窗口里，就删掉。
>3. 队列始终单调递减，所以队首就是当前窗口最大值。


## 代码

```java
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    public int[] maxSlidingWindow(int[] nums, int k) {  
        int n = nums.length;  
        int[] ans = new int[n - k + 1]; // 窗口个数  
        Deque<Integer> q = new ArrayDeque<>();  
  
        for (int i = 0; i < n; i++) {  
            // 1. 右边入  
            while (!q.isEmpty() && nums[q.getLast()] <= nums[i]) {  
                q.removeLast(); // 维护 q 的单调性  
            }  
            // 注意保存的是下标，这样下面可以判断队首是否离开窗口  
            q.addLast(i);  
  
            // 2. 左边出  
            int left = i - k + 1; // 窗口左端点  
            if (q.getFirst() < left) { // 队首离开窗口  
                q.removeFirst();  
            }  
            // 3. 在窗口左端点处记录答案  
            if (left >= 0) {  
                // 由于队首到队尾单调递减，所以窗口最大值就在队首  
                ans[left] = nums[q.getFirst()];  
            }  
        }  
        return ans;  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点

1. **滑动窗口**
   - 适用场景：处理连续子数组/子串问题
   - 关键：维护窗口内的状态，避免重复计算
2. **单调队列**
   - 特性：队列内元素保持单调递增或递减
   - 应用：滑动窗口最大/最小值、求窗口内第 k 大元素
   - 操作：入队时移除破坏单调性的元素
3. **双端队列**
   - 特点：两端都可以插入和删除
   - Java 实现：ArrayDeque（基于数组，性能优于 LinkedList）

## 复杂度

- Time: O(n)
