## 思路（必填）

### 问题分析

- 目标：找到字符串 `s` 中最长的回文子串
- 回文特性：关于中心对称
  - 奇数长度：中心是一个字符（如 `"aba"`，中心是 `'b'`）
  - 偶数长度：中心是两个字符之间（如 `"abba"`，中心在两个 `'b'` 之间）

### 算法选择

- **中心扩展法**：
  - 遍历每个可能的中心位置
  - 从中心向两边扩展，直到不满足回文条件
  - 记录最长回文子串的位置
- **优势**：
  - 空间复杂度 O(1)，优于动态规划 O(n²)
  - 代码简洁，实际运行更快（常数因子小）
  - 思路直观，易于理解

### 实现步骤

1. 边界处理：空字符串或 null 直接返回
2. 遍历中心：对每个位置 `i`，尝试两种中心：
   - 单中心 `(i, i)`：对应奇数长度回文
   - 双中心 `(i, i+1)`：对应偶数长度回文
3. 中心扩展：从中心向两边扩展，直到字符不等或越界
4. 更新答案：比较当前回文长度与最大长度，更新边界
5. 返回结果：根据最终边界截取子串

## 关键点

> [!Tip]
> - 双重中心处理：必须同时考虑奇偶长度
> - 边界计算技巧：
>   - 长度 = `right - left - 1`（因为退出循环时已经多扩展了一次）
>   - 起始位置 = `i - (len - 1) / 2`（推导得出）
> - 时间复杂度：虽然有嵌套循环，但总体仍是 O(n²)

## 代码（多维动态规划）

```java
//leetcode submit region begin(Prohibit modification and deletion)
class Solution {
    public String longestPalindrome(String s) {
        if (s == null || s.length() < 2) {
            return s;
        }
        int strLen = s.length();
        int maxStart = 0;
        int maxEnd = 0;
        int maxLen = 1;

        boolean[][] dp = new boolean[strLen][strLen];
        for (int right = 1; right < strLen; right++) {
            for (int left = 0; left < right; left++) {
                if (s.charAt(left) == s.charAt(right) && (right - left <= 2 || dp[left + 1][right - 1])) {
                    dp[left][right] = true;
                    if (right - left + 1 > maxLen) {
                        maxLen = right - left + 1;
                        maxStart = left;
                        maxEnd = right;
                    }
                }
            }
        }
        return s.substring(maxStart, maxEnd + 1);
    }
}
//leetcode submit region end(Prohibit modification and deletion)
```

### 优化的代码（中心扩展算法）

```java
//leetcode submit region begin(Prohibit modification and deletion)
class Solution {
    public String longestPalindrome(String s) {
        if (s == null || s.length() < 1) {
            return "";
        }
        int start = 0;
        int end = 0;
        for (int i = 0; i < s.length(); i++) {
            int len1 = expandAroundCenter(s, i, i);
            int len2 = expandAroundCenter(s, i, i + 1);
            int len = Math.max(len1, len2);
            if (len > end - start) {
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }
        return s.substring(start, end + 1);
    }

    private int expandAroundCenter(String s, int left, int right) {
        int L = left;
        int R = right;
        while (L >= 0 && R < s.length() && s.charAt(L) == s.charAt(R)) {
            L--;
            R++;
        }
        return R - L - 1;
    }
}
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点

### 1. 为什么返回 `R - L - 1`

函数 `expandAroundCenter` 做的事是：

- 从中心开始扩散
- 只要左右相等，就继续往外走
- 一旦不满足条件，就停下来

关键点在于：**当 `while` 结束时，`L` 和 `R` 已经走过头了**

也就是说：

- 真正回文串的左边界不是 `L`
- 真正回文串的右边界也不是 `R`

而是：`L + 1` 到 `R - 1`

所以返回的就是 `(R - 1) - (L + 1) + 1 = R - L - 1`

### 2. 为什么 `start = i - (len - 1) / 2`

这个公式是用来算：**最长回文串的左边界**

这里的 `i` 表示"当前中心左侧的位置"。

- 对奇数回文，中心就是 `i`
- 对偶数回文，中心在 `i` 和 `i+1` 中间，但代码里还是拿 `i` 当参考点

所以公式要同时兼容奇数和偶数。

**为什么奇数时是 `(len - 1) / 2`？**

因为奇数回文长这样：左边一半 + 中心 + 右边一半

比如长度 5：

- 中心左边有 2 个字符
- 中心右边也有 2 个字符

所以从中心往左走的步数是：`(len - 1) / 2`

### 3. 为什么 `end = i + len / 2`

这个公式是算：**最长回文串的右边界**

**为什么奇数时 `len / 2` 也成立？**

因为 Java 整数除法会自动向下取整。

例如：

- `3 / 2 = 1`
- `5 / 2 = 2`
- `7 / 2 = 3`

这正好就是奇数回文从中心向右走的步数。

### 4. 为什么这两个公式能统一奇偶

因为它选取的参考点很巧妙：

- 奇数回文：`i` 就是正中心
- 偶数回文：`i` 是"中间偏左"的那个位置

于是：

- 奇数长度 `len = 2k + 1`
	- 左边界：`i - (len - 1) / 2 = i - k`
	- 右边界：`i + len / 2 = i + k`

- 偶数长度 `len = 2k`

此时中心在 `i` 和 `i+1` 中间。

左边界应该是：`i - k + 1`

看公式：

```
i - (len - 1) / 2
= i - (2k - 1) / 2
= i - (k - 1)
= i - k + 1
```

对。

右边界应该是：`i + k`

看公式：

```
i + len / 2
= i + k
```

也对。
