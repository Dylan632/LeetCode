## 题目

![[74.png]]

**提示：**

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= m, n <= 100`
- `-10^4 <= matrix[i][j], target <= 10^4`

## 思路

### 问题分析

- 题目要求在有序二维矩阵中查找目标值
- 矩阵特性：每行从左到右递增，每行第一个元素大于上一行最后一个元素
- 这意味着将矩阵按行展开后，整体是一个有序的一维数组

### 算法选择

- 选择二分查找，因为矩阵可视为有序一维数组
- 时间复杂度从暴力解法 O(m×n) 优化到 O(log(m×n))

### 实现步骤

1. 虚拟展开：将二维矩阵 `[m][n]` 虚拟展开为一维数组 `[0, m×n-1]`
2. 索引映射：一维索引 `mid` 映射到二维坐标 (`mid / n`, `mid % n`)
   - `mid / n`：整除得到行号（每行有 n 个元素）
   - `mid % n`：取余得到列号（当前行的第几个元素）
3. 二分查找：在虚拟一维数组上执行标准二分查找
4. 区间设计：使用 `left = -1` 和 `right = m*n` 的开区间设计，避免边界遗漏

## 关键点

> [!Tip]
> - ✨ 索引映射公式：`matrix[mid / n][mid % n]` 是核心技巧
> - ✨ 区间设计：`left = -1` 保证不遗漏索引 0，`right = m*n` 保证不遗漏最后一个元素

## 代码

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;
        int left = 0;
        int right = m * n - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            int x = matrix[mid / n][mid % n];
            if (x == target) {
                return true;
            }
            if (x < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return false;
    }
}
```

## 知识点

由于矩阵的每一行是递增的，且每行的第一个数大于前一行的最后一个数，如果把矩阵每一行拼在一起，我们可以得到一个递增数组。

![[74_1.png]]

例如示例 1，三行拼在一起得：

`a = [1, 3, 5, 7, 10, 11, 16, 20, 23, 30, 34, 60]`

由于这是一个有序数组，我们可以用二分查找判断 target 是否在 matrix 中。

代码实现时，并不需要真的拼成一个长为 `m * n` 的数组 `a`，而是将 `a[i]` 转换成矩阵中的行号和列号。例如示例 1，`i = 9` 对应的 `a[i] = 30`，由于矩阵有 `n = 4` 列，所以 `a[i]` 在 `i / n = 2` 行，在 `i % n = 1` 列。

一般地，有：`a[i] = matrix[i / n][i % n]`

## 复杂度

- **时间复杂度**：O(log(m × n))，其中 m 和 n 分别为 matrix 的行数和列数
- **空间复杂度**：O(1)
