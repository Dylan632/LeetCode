## 思路（必填）
### 算法选择
  - 排序 + 贪心：先按左端点排序，再遍历合并
  - 为什么排序？排序后只需比较当前区间与上一个合并区间，无需两两比较
  - 为什么贪心？每次合并都取最大右端点，保证局部最优
### 实现步骤
  1. 排序：按左端点升序排列，保证后续处理有序
  2. 遍历：逐个处理每个区间
  3. 合并判断：
    - 若当前左端点 ≤ 上一个合并区间的右端点 → 重叠，更新右端点
    - 否则 → 不重叠，添加新区间
  4. 返回：将 List 转换为数组

## 关键点
>[!Tip] 排序是核心优化点
>- 排序将 O(n²) 降为 O(n log n)
>- 合并时只更新右端点，左端点保持不变（因为已排序）
>- `Math.max` 保证右端点取最大值，处理包含关系（如 `[1,4]` 和 `[2,3]`）

## 代码

```java
import java.util.ArrayList;  
  
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    public int[][] merge(int[][] intervals) {  
        // 按照左端点从小到大排序  
        Arrays.sort(intervals, (p, q) -> Integer.compare(p[0], q[0]));  
        List<int[]> ans = new ArrayList<>();  
        for (int[] p : intervals) {  
            int m = ans.size();  
            // 可以合并：当前区间左端点 ≤ 上一个合并区间的右端点  
            if (m > 0 && p[0] <= ans.get(m-1)[1]) {  
                // 更新右端点最大值  
                ans.get(m - 1)[1] = Math.max(ans.get(m - 1)[1], p[1]);  
            } else { // 不相交，无法合并  
                // 新的合并区间  
                ans.add(p);  
            }  
        }  
        return ans.toArray(new int[0][]);  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

##  代码执行跟踪
```java
  // 初始状态
  intervals = [[1,3], [2,6], [8,10], [15,18]]
  ans = []

  // 第1次循环: p = [1,3]
  m = 0
  m > 0 为 false，执行 else 分支
  ans = [[1,3]]

  // 第2次循环: p = [2,6]
  m = 1
  m > 0 为 true，p[0]=2 <= ans.get(0)[1]=3，执行合并
  ans.get(0)[1] = max(3, 6) = 6
  ans = [[1,6]]

  // 第3次循环: p = [8,10]
  m = 1
  m > 0 为 true，p[0]=8 <= ans.get(0)[1]=6 为 false，执行 else
  ans = [[1,6], [8,10]]

  // 第4次循环: p = [15,18]
  m = 2
  m > 0 为 true，p[0]=15 <= ans.get(1)[1]=10 为 false，执行 else
  ans = [[1,6], [8,10], [15,18]]

  // 返回结果
  return [[1,6], [8,10], [15,18]]
```
## 复杂度

- Time: O(n log n)，排序 O(n log n) + 遍历 O(n)，结果数组存储 O(n)
- Space: O(n)
