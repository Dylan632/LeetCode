## 思路（必填）
### 问题分析：
  - 找出未排序数组中缺失的第一个正整数
  - 要求时间复杂度 O(n)，空间复杂度 O(1)
  - 数组中可能包含负数、零、重复元素
### 算法选择
  选择原地哈希算法：
  - 利用数组本身作为哈希表，将数值映射到索引
  - 数值 x 应该放在索引 x-1 的位置（例如：3 放在索引 2）
  - 遍历一次数组，将每个符合条件的元素交换到正确位置
  - 再遍历一次，找出第一个位置不匹配的数
### 实现步骤
  1. 第一次遍历：原地重排数组
    - 对于每个位置 i，如果 nums[i] 是 [1, n] 范围内的正整数
    - 且它不在正确位置（nums[i] != i+1）
    - 且目标位置还不是正确的值（避免重复元素）
    - 则交换到正确位置 nums[i]-1
  2. 第二次遍历：查找缺失的正整数
    - 遍历数组，找到第一个 nums[i] != i+1 的位置
    - 返回 i+1 即为缺失的最小正整数
  3. 边界处理：
    - 如果所有位置都正确，返回 n+1

## 关键点

> [!Tip]
> - 为什么用 while 循环：交换后位置 i 的新元素可能仍需继续交换
> - 为什么判断 `nums[nums[i]-1] != nums[i]`：避免重复元素导致死循环
> - 时间复杂度证明：每个元素最多被交换一次到正确位置，总体 O(n)

## 代码

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int len = nums.length;
        
        for (int i = 0; i < len; i++) {
            while (nums[i] > 0 && nums[i] <= len && nums[nums[i] - 1] != nums[i]) {
                // 满足在指定范围内、并且没有放在正确的位置上，才交换
                // 例如：数值 3 应该放在索引 2 的位置上
                swap(nums, nums[i] - 1, i);
            }
        }

        // [1, -1, 3, 4]
        for (int i = 0; i < len; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        // 都正确则返回数组长度 + 1
        return len + 1;
    }

    private void swap(int[] nums, int index1, int index2) {
        int tmp = nums[index1];
        nums[index1] = nums[index2];
        nums[index2] = tmp;
    }
}
```

## 知识点

### 1.while vs if：为什么必须用 while 循环？

**核心目标：** 将数字 `x` 放到索引 `x-1` 的位置上（例如数字 1 放到索引 0，数字 2 放到索引 1）。

**`while` 循环的作用是：“持续检查被交换回来的新数字”。**

当你把当前位置 `i` 上不听话的数字（比如 `A`）扔回它该去的地方时，会从那个地方换回来一个新的数字（比如 `B`）。

- 如果用 `if`：换回来的 `B` 就直接被留在了位置 `i`，程序走向下一个索引，**导致 `B` 失去了被检查和归位的机会**。
- 如果用 `while`：程序会“钉”在当前索引 `i` 不走，继续审视刚换回来的 `B`。如果 `B` 也不在正确位置且符合条件，就继续把 `B` 扔到该去的地方，直到换回来的数字不需要/不能再交换为止。

#### 经典推演示例：`[3, 4, -1, 1]`

当遍历到索引 `i = 1` 时（当前数字是 `4`）：

1. **第一次 `while` 检查：** `4` 应该去索引 `3`。
    - **动作：** 将 `4` 与索引 `3` 处的 `1` 交换。
    - **数组变为：** `[-1, 1, 3, 4]`

2. **第二次 `while` 检查（关键！）：** 此时 `i = 1` 没变，但数字变成了换回来的 `1`。`1` 应该去索引 `0`。
    - **动作：** 将 `1` 与索引 `0` 处的 `-1` 交换。
    - **数组变为：** `[1, -1, 3, 4]`

3. **第三次 `while` 检查：** 此时 `i = 1` 处的数字变成了 `-1`，不满足 `nums[i] > 0` 的条件，循环终于结束。
    - **动作：** 结束 `while`，执行外层循环的 `i++`。

**一句话总结：** 原地哈希的核心就是”一个萝卜一个坑”，`while` 保证了在前往下一个坑之前，当前坑里要么坐着正确的萝卜，要么坐着我们无法处理的废弃萝卜（如负数或超界数）。

### 2.为什么只在 1 到 n 之间处理？

这里 `n` 是数组的长度 (`nums.length`)。理由基于一个数学事实：

**在一个长度为 n 的数组中，缺失的第一个正数一定在 [1, n+1] 这个范围内。**

- **最好的情况**：数组里正好包含了 1, 2, 3, ..., n 这 n 个数。那么缺失的第一个正数就是 **n+1**。
- **其他情况**：只要数组里缺了 1 到 n 之间的任何一个数，那么缺失的第一个正数就一定 **≤ n**。

**结论**：我们根本不需要关心大于 n 的数（比如 n+5, 1000 等），因为它们对寻找”最小的缺失正数”没有影响。如果 1 到 n 都齐了，答案自然是 n+1；如果没齐，答案就在 1 到 n 之中。

## 复杂度

- Time: O(n)，其中 n 是 nums 的长度。虽然我们写了个二重循环，但每次交换都会把一个学生换到正确的座位上，所以总交换次数至多为 n，所以内层循环的总循环次数是 O(n) 的，所以时间复杂度是 O(n)。
- Space: O(1)
