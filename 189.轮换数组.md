## 思路

### 算法选择

三次反转法，核心思想：**旋转 = 整体反转 + 局部反转**

### 实现步骤

![[189_1.png]]

假设 `0 ≤ k < n`（对于 `k ≥ n` 的情况，`k %= n` 即可转换）。

设 `nums = A + B`，其中 A 是前 `n - k` 个元素，B 是后 `k` 个元素。例如 `A = [1,2,3,4]`，`B = [5,6,7]`。

**目标**：把 `A + B` 变成 `B + A`

**三次反转**：

1. 整体反转 `nums`：得到 `rev(B) + rev(A)`
2. 反转 `rev(B)`：得到 `B`
3. 反转 `rev(A)`：得到 `A`

最终结果：`B + A` ✓


## 关键点

>[!Tip] 核心要点
>- **反转顺序**：必须先整体反转，再局部反转
>- **边界处理**：`k` 可能大于 `n`，需要取模
>- **双指针技巧**：reverse 方法使用双指针交换元素，实现 O(1) 空间复杂度

## 代码

```java
  
//leetcode submit region begin(Prohibit modification and deletion)  
class Solution {  
    public void rotate(int[] nums, int k) {  
        int n = nums.length;  
        k %= n; // 轮转 k 次等于轮转 k % n 次  
        reverse(nums, 0, n - 1);  
        reverse(nums, 0, k - 1);  
        reverse(nums, k, n - 1);  
    }  
  
    private void reverse(int[] nums, int i, int j) {  
        while (i < j) {  
            int temp = nums[i];  
            nums[i++] = nums[j];  
            nums[j--] = temp;  
        }  
    }  
}  
//leetcode submit region end(Prohibit modification and deletion)
```

## 知识点
### 反转数组的下标变化公式

假设我们有一个数组（或子数组），我们要反转它在下标区间 $[L, R]$ 内的元素。

- 在这个区间里，最左边是 $L$，最右边是 $R$。
- 假设某个元素现在的下标是 $x$（且 $L \le x \le R$）。
- 这个元素距离左边界 $L$ 的距离是 $x - L$。
- 反转之后，它应该被换到距离右边界 $R$ 同等距离的位置上，也就是 $R - (x - L)$。

化简一下这个式子：

$$R - (x - L) = L + R - x$$

**这就是反转操作的“万能公式”：反转后的新下标 = 左边界 + 右边界 - 旧下标。**

（你也可以这么记：在对称的反转操作中，元素**反转前后的下标之和，永远等于区间的左右边界之和**，即 $x_{old} + x_{new} = L + R$）。
