## 题目

![[98.png]]

**提示：**

- 树中节点数目范围在 `[1, 10^4]` 内
- `-2^31 <= Node.val <= 2^31 - 1`

## 思路

### 问题分析

- 验证一棵二叉树是否为合法的二叉搜索树（BST）
- BST 定义：左子树所有节点值 < 根节点值 < 右子树所有节点值
- 关键约束：必须验证整个子树，而不仅是直接子节点

### 算法选择

- 中序遍历 + 前驱值跟踪是本题最优解
- BST 的中序遍历结果必然是严格递增序列
- 只要验证遍历过程中每个节点值严格大于前一个节点值即可

### 实现步骤

1. 递归左子树：先验证左子树是否合法
2. 验证当前节点：与记录的前驱值 pre 比较，必须严格递增
3. 更新前驱值：将当前节点值赋给 pre
4. 递归右子树：验证右子树是否合法
5. 剪枝优化：任一步骤失败立即返回 false

## 关键点

> [!Tip]
> - 使用 Long.MIN_VALUE 初始化，兼容 int 范围的所有节点值
> - 使用 <= 而非 <，保证严格递增（BST 定义不允许重复值）
> - 前驱值必须验证整个子树的最大值（左侧）或最小值（右侧），而非仅验证直接父子关系

## 代码

```java
class Solution {
    private long pre = Long.MIN_VALUE;
    public boolean isValidBST(TreeNode root) {
        // 1. 终止条件（Base Case）
        // 如果节点为空，说明已经遍历到了叶子节点的下一层，空树被认为是合法的 BST
        if (root == null) {
            return true;
        }
        // 2. 遍历左子树（左）
        // 递归判断左子树是否为合法的 BST。
        // 如果左子树不合法，直接返回 false，不需要再继续判断了（剪枝操作）。
        if (!isValidBST(root.left)) {
            return false;
        }
        // 3. 处理当前节点（中）
        // 核心逻辑：当前节点的值必须严格大于上一个节点的值 (pre)
        // 如果当前节点的值 <= pre，说明破坏了严格递增的规律，不是合法的 BST
        if (root.val <= pre) {
            return false;
        }
        // 检查通过后，将当前节点的值赋给 pre，作为下一个待访问节点的前驱节点值
        pre = root.val;
        // 4. 遍历右子树（右）
        // 左子树和当前节点都验证通过了，最后递归验证右子树是否合法，并返回结果
        return isValidBST(root.right);
    }
}
```

## 知识点

```
初始状态: pre = -∞ (Long.MIN_VALUE)

isValidBST(5)
├── 左: isValidBST(1)
│   ├── 左: isValidBST(null) → true
│   ├── 中: 检查 1 <= -∞ 不成立 ✅，更新 pre = 1
│   ├── 右: isValidBST(null) → true
│   └── ✅ 左中右均合法，返回 true
│
├── 中: 检查 5 <= 1 不成立 ✅，更新 pre = 5
│
└── 右: isValidBST(4)
    ├── 左: isValidBST(3)
    │   ├── 左: isValidBST(null) → true
    │   ├── 中: 检查 3 <= 5 成立 ❌ (注意：当前访问节点是3，但 pre 已经是根节点传过来的5了！)
    │   └── ❌ 破坏递增规律，返回 false (触发剪枝，不继续遍历 3 的右孩子)
    │
    └── ❌ 因为左子树 isValidBST(3) 返回 false，节点 4 的判断直接终止，不再检查 4 和 6
    
└── ❌ 整体返回 false
```

## 复杂度

- Time: O(n)，其中 n 为二叉搜索树的节点个数。
- Space: O(n)，最坏情况下，二叉搜索树退化成一条链（注意题目没有保证它是**平衡**树），因此递归需要 O(n) 的栈空间。
