## 思路（必填）
### 问题分析
设计一个数据结构，支持动态添加数字并实时查询中位数。中位数是有序序列的中间值（长度为偶数时取中间两数平均值）。
### 算法选择
  选择双堆法，因为：
  - 堆可以在 O(log n) 时间内插入元素
  - 堆顶可以在 O(1) 时间内访问极值
  - 用两个堆分别维护“较小的一半”和“较大的一半”，即可快速得到中位数
### 实现步骤
  1. 左堆（大顶堆）：存储较小半部分，堆顶是这部分的最大值
  2. 右堆（小顶堆）：存储较大半部分，堆顶是这部分的最小值
  3. 维护平衡：
    - 元素数量相等时：新元素先入右堆，再从右堆取出最小值加入左堆 → 左堆多一个
    - 元素数量不等时：新元素先入左堆，再从左堆取出最大值加入右堆 → 恢复平衡
  4. 查询中位数：
    - 左堆元素多：返回左堆顶（奇数个元素）
    - 两堆相等：返回两堆顶平均值（偶数个元素）

## 关键点
>[!Tip]
>- 始终保证 left.size() >= right.size()，使得中位数要么在左堆顶，要么是两堆顶的平均值
>- 元素插入时通过“先入一端，再取顶移到另一端”的方式，保证两堆元素有序且中位数分界

## 代码

```java
class MedianFinder {  
    private final PriorityQueue<Integer> left = new PriorityQueue<>((a, b) -> b - a);  
    private final PriorityQueue<Integer> right = new PriorityQueue<>();  
  
    public void addNum(int num) {  
        if (left.size() == right.size()) {  
            right.offer(num);  
            left.offer(right.poll());  
        } else {  
            left.offer(num);  
            right.offer(left.poll());  
        }  
    }  
  
    public double findMedian() {  
        if (left.size() > right.size()) {  
            return left.peek();  
        }  
        return (left.peek() + right.peek()) / 2.0;  
    }  
}
```

## 知识点

### 中位数定义
比如现在有 6 个数：1,5,6,2,3,4，要计算中位数，可以把这 6 个数从小到大排序，得到 1,2,3,4,5,6，中间 3 和 4 的平均值 3.5 就是中位数。
> 中位数可将数值集合划分为相等的两部分。
中位数把这 6 个数均分成了左右两部分：
- `left = [1,2,3]`（较小的一半）
- `right = [4,5,6]`（较大的一半）
中位数来自 left 中的最大值，以及 right 中的最小值。
### 双堆结构
默认 `a - b` 小顶堆，构造大顶堆的时候需要使用比较器，使用 `b - a`。
```java
// 初始化小顶堆 
Queue<Integer> minHeap = new PriorityQueue<>(); 
// 初始化大顶堆（使用 lambda 表达式修改 Comparator 即可） 
Queue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
```
![[295_1.png]]
#### 1. `left` (大顶堆) - 负责左半边（较小的数字）
- **作用**：在一堆较小的数字中，快速找到**最大**的那一个（紧贴切口左侧）。
- **Java 实现**：`new PriorityQueue<>((a, b) -> b - a)`
    - **语法解析 `(a, b) -> b - a`**：这是 Java 的 Lambda 表达式，用于自定义比较器（Comparator）。
    - Java 默认是升序（小顶堆，`a - b`）。当我们改为 `b - a` 时，数值越大的元素反而会被判定为优先级更高，从而被推到堆顶，完美实现**降序（大顶堆）**。
#### 2. `right` (小顶堆) - 负责右半边（较大的数字）
- **作用**：这里装的虽然都是整体偏大的数字，但为了拿到切口右侧的那个数，我们需要在一堆大数字中找**最小**的那个老幺。
- **Java 实现**：`new PriorityQueue<>()`（默认即为小顶堆）。

### 核心约束
随着 addNum 不断添加数字，需要满足两个要求：
1. **大小平衡**：<font color="#de7802">left 和 right 的大小尽量相等。规定：奇数个元素时，left 比 right 多 1 个。</font>
2. **元素有序**：<font color="#de7802">left 的所有元素 ≤ right 的所有元素。</font>
满足这两个要求后，就可以用 left 的最大值和 right 的最小值计算中位数。
### 分类讨论
**情况一：left.size() == right.size()**
- 添加大数（如 7）：加到 right，然后 right 最小值移到 left
- 添加小数（如 0）：加到 left
合并操作：先加到 right，再从 right 弹出最小值加到 left。
**情况二：left.size() > right.size()**
- 添加大数（如 7）：加到 right
- 添加小数（如 0）：加到 left，然后 left 最大值移到 right
合并操作：先加到 left，再从 left 弹出最大值加到 right。
### `offer` vs `poll`
- **`offer(num)`**：将元素**插入**队列（堆）。相比于 `add()`，`offer` 在面对容量限制时更为温和，插入失败会返回 false 而不是抛出异常崩溃。
- **`poll()`**：将堆顶元素**弹出并获取**（即从堆里拿出来并删掉）。

## 复杂度

- Time: O(log n)
- Space: O(1)
