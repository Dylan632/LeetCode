## 题目
![[543.png]]

## 思路（必填）

1. 递归求每个节点的深度（后序遍历，先左右再自己）
2. 在每个节点处，计算 leftHeight + rightHeight，这就是经过该节点的最长路径
3. 用全局变量记录所有节点中的最大路径值
4. 返回 max(left, right) + 1 作为该节点的深度

## 算法选择
后序遍历（DFS）。关键洞察——任意路径都有一个“最高点”（拐点），该路径 = 左子树深度 + 右子树深度。所以只需在求深度的过程中，对每个节点都计算一次“经过该节点的最长路径”，取全局最大值。

## 关键点
> [!Tip] 
> 把“求直径”巧妙转化为“求深度”的附产品，一次遍历解决两个问题。

## 代码

```java
class Solution {  
    int maxDiameter = 0;  
  
    public int diameterOfBinaryTree(TreeNode root) {  
        maxDepth(root);  
        return maxDiameter;  
    }  
  
    private int maxDepth(TreeNode node) {  
        if (node == null) {  
            return 0;  
        }  
        //后序遍历  
        int leftHeight = maxDepth(node.left);  
        int rightHeight = maxDepth(node.right);  
        maxDiameter = Math.max(maxDiameter, leftHeight + rightHeight);  
        return Math.max(leftHeight, rightHeight) + 1;  
    }  
}
```

## 知识点
🌳 递归图

以如下二叉树为例：

```
        1
       / \
      2   3
     / \
    4   5
```

递归调用过程（后序遍历：左 → 右 → 根）：

```

  maxDepth(1)
  ├── maxDepth(2)
  │   ├── maxDepth(4)
  │   │   ├── maxDepth(null) → return 0
  │   │   └── maxDepth(null) → return 0
  │   │   → left=0, right=0
  │   │   → maxDiameter = max(0, 0+0) = 0
  │   │   → return max(0,0)+1 = 1
  │   │
  │   └── maxDepth(5)
  │       ├── maxDepth(null) → return 0
  │       └── maxDepth(null) → return 0
  │       → left=0, right=0
  │       → maxDiameter = max(0, 0+0) = 0
  │       → return max(0,0)+1 = 1
  │
  │   → left=1, right=1
  │   → maxDiameter = max(0, 1+1) = 2  ✨ 更新！
  │   → return max(1,1)+1 = 2
  │
  └── maxDepth(3)
      ├── maxDepth(null) → return 0
      └── maxDepth(null) → return 0
      → left=0, right=0
      → maxDiameter = max(2, 0+0) = 2  （不变）
      → return max(0,0)+1 = 1

  → left=2, right=1
  → maxDiameter = max(2, 2+1) = 3  ✨ 更新！
  → return max(2,1)+1 = 3（深度，但不再使用）

  最终结果：maxDiameter = 3
```
递归状态汇总表：

| 节点 | leftH | rightH | leftH+rightH | maxDiameter更新 | 返回深度 |
|------|-------|--------|--------------|----------------|----------|
| 4    | 0     | 0      | 0            | 0              | 1        |
| 5    | 0     | 0      | 0            | 0              | 1        |
| 2    | 1     | 1      | 2            | 0 → 2 ✨       | 2        |
| 3    | 0     | 0      | 0            | 2（不变）       | 1        |
| 1    | 2     | 1      | 3            | 2 → 3 ✨       | 3        |

直径 = 3，对应路径为 4 → 2 → 1 → 3（或 5 → 2 → 1 → 3），经过 3 条边。

## 复杂度

- Time: O(n)
- Space: O(h)，调用 maxDepth，h 最坏为 n（退化链表），最优为 log n（平衡树）
