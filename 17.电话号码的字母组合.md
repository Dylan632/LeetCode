## 题目
![[17.png]]

## 思路（必填）
### 问题分析
给定数字字符串（2-9），返回所有可能的字母组合。每个数字对应电话按键上的多个字母，需要找出所有组合。
### 算法选择
回溯法（DFS）。因为需要枚举所有可能的组合，回溯是最自然的解法。
### 实现步骤
1. 建立数字到字母的映射表 MAPPING
2. 使用 path 数组记录当前路径（已选的字母）
3. DFS 递归：对每个位置的数字，遍历其对应的所有字母
4. 当路径长度等于数字串长度时，生成结果字符串加入答案
5. 回溯：通过复用 path 数组，无需显式撤销操作（直接覆盖即可）

## 关键点
>[!Tip] 
>  - 原地修改 path 数组，避免频繁创建字符串对象
  >- `digits[i] - '0'` 技巧将字符数字转为整数索引

## 代码

```java
class Solution {
    // 索引刚好对应电话按键上的数字
    private static final String[] MAPPING = new String[]{"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
    public List<String> letterCombinations(String digits) {
        int n = digits.length();
        if (n == 0) {
            // 处理边界情况：输入为空字符串，直接返回空列表
            return List.of();
        }
        List<String> ans = new ArrayList<>();
        char[] path = new char[n];
        dfs(0, ans, path, digits.toCharArray());
        return ans;
    }

    private void dfs(int i, List<String> ans, char[] path, char[] digits) {
        if (i == digits.length) {
            ans.add(new String(path));
            return;
        }
        String letters = MAPPING[digits[i] - '0'];
        for (char c : letters.toCharArray()) {
            path[i] = c;
            dfs(i + 1, ans, path, digits);
        }
    }
}
```

## 知识点
### 💡 核心思路

传统的字符串组合回溯通常使用 `StringBuilder` 或 `List`，并在递归返回时显式调用 `deleteCharAt` 或 `removeLast` 来“撤销选择”。

但这道题存在一个特殊性质：**结果的长度是固定的（等于输入数字的长度）**。因此，我们可以通过初始化一个**定长字符数组 (`char[] path`)**利用递归深度作为索引，**直接覆盖**旧字符，从而省去显式的撤销步骤，大幅提升代码运行效率。

### 灵魂发问一：什么时候可以“直接覆盖”？什么时候必须“显式撤销”？

这是回溯算法中最核心的底层逻辑区分。决定用哪种方式，取决于**路径容器的数据结构**和**路径长度的固定性**。

|**维度**|**🟢 直接覆盖 (Implicit Backtrack)**|**🔴 显式撤销 (Explicit Backtrack)**|
|---|---|---|
|**适用场景**|结果长度**固定**且预知（如本题、固定长度密码生成）|结果长度**不固定**（如求子集、组合总和）|
|**数据结构**|**定长数组**（`char[]`, `int[]`）|**动态容器**（`List`, `StringBuilder`）|
|**核心操作**|`path[depth] = val;` (利用深度索引排他性)|`list.add(val); dfs(...); list.removeLast();`|
|**全局状态**|无需维护全局 `visited` 数组|常配合全局 `visited` 数组且必须重置状态|
|**性能表现**|**极高**（无对象创建和动态扩容/缩容开销）|**一般**（涉及容器底层的元素移动）|

**结论**：如果题目最终生成的组合/排列长度是固定的，首选“定长数组 + 深度索引直接覆盖”的套路。

### 灵魂发问二：没有撤销操作，递归到底是怎么执行的？（以 `digits = “23”` 为例）

很多时候我们觉得必须“撤销”，是怕旧数据污染了下一次的拼接。但“覆盖法”的精髓就在于：**旧数据会被当成“脏数据”，并在进入对应层级时被无情盖住，根本没机会影响最终结果。**

我们可以把定长数组 `path` 想象成一块**只有 2 个格子的黑板**：
- 输入 `“23”` 对应字母集：`2 -> “abc”`, `3 -> “def”`。

**执行推演**：

1. **一路向下**：
    - 第一层（填第 0 格）：拿到 `2`，挑出字母 `'a'`，写在第 0 格。黑板：`['a', ' ']`
    - 第二层（填第 1 格）：拿到 `3`，挑出字母 `'d'`，写在第 1 格。黑板：`['a', 'd']`
    - 触发终止条件，记录结果 `“ad”`。退回上一层。
2. **体验直接覆盖的魔力**：
    - 退回第二层，`3` 的字母还没用完，挑出下一个字母 `'e'`。
    - **不需要擦掉 'd'**，直接在第 1 格用力写上 `'e'`。黑板变成：`['a', 'e']`。
    - 触发终止条件，记录结果 `“ae”`。继续覆盖生成 `“af”`。
3. **“脏数据”无需理会**：
    - 第二层执行完毕，退回第一层。
    - `2` 的字母挑出下一个 `'b'`，写在第 0 格。此时黑板变成：`['b', 'f']`。（这里的 `'f'` 就是上一次遗留的脏数据）。
    - 带着 `['b', 'f']` 进入第二层。再次遍历 `3` 的字母 `“def”`。
    - 拿到 `'d'` 时，直接写在第 1 格。脏数据 `'f'` 瞬间被盖住，黑板变成：`['b', 'd']`。生成正确结果 `“bd”`。

**结论**：你的执行逻辑保证了旧数据一定会被新数据正确覆盖，因此无需显式“擦除”。

## 复杂度

- Time: $O(n \cdot 4^n)$，其中 $n$ 为 digits 的长度。最坏情况下每次需要枚举 4 个字母，递归次数为一个满四叉树的节点个数，那么一共会递归 $O(4^n)$ 次（等比数列和），再算上加入答案时复制 path 需要 $O(n)$ 的时间，所以时间复杂度为 $O(n \cdot 4^n)$。
- Space: $O(n)$，返回值的空间不计入。
